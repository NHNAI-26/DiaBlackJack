실제 착수·구현·검증 결과는 `Docs/stage-progression-progress-log.md`에 누적한다.

## 1. 계획 목적

완료된 단일 전투 코어 루프를 수정 범위가 작은 방식으로 재사용하여, 고정 경로의 여러 전투를 하나의 런으로 연결한다. 이 계획은 코어 루프 단계와 별개인 독립 작업 계획이다.

## 2. 완료 목표

```text
런 시작
→ 일반 전투
→ 플레이어 영혼 유지
→ 다음 일반 전투
→ 최종 보스 전투
→ 런 승리 또는 중간 런 패배
→ 런 재시작
```

검증용 경로는 일반 전투 2개와 최종 보스 전투 1개로 구성한다. 경로의 실제 콘텐츠 수량은 이 작업의 확정 대상이 아니다.

## 3. 작업 범위

### 3.1 포함

- 런 상태와 플레이어 지속 상태
- 스테이지 정의 목록과 현재 진행 인덱스
- 일반 전투·최종 보스 전투 생성
- 전투 승패 결과의 진행 상태 반영
- 플레이어 영혼·덱 구성 유지
- 다음 스테이지와 런 재시작 입력
- 최소 진행도 UI
- EditMode 및 실제 씬 흐름 검증

### 3.2 제외

- 보상, 휴식, 상점, 사건과 분기 지도
- 카드·계약·유물 콘텐츠
- 적 고유 행동과 보스 패턴
- 저장·불러오기
- 최종 아트·연출

## 5. 작업별 실행 조건

### SP-00 — 기준 확정과 회귀 기준 기록

**할 일**

- `rule.md`, 전체 기획서와 신규 기획·개발 명세를 대조한다.
- 현재 작업 트리와 Unity MCP 연결 상태를 확인한다.
- 기존 코어 루프 EditMode 27개 통과 상태를 기준선으로 기록한다.
- `CoreLoopTest` 씬은 회귀 확인용으로 유지하고 진행 전용 씬을 분리한다.

**완료 증거**

- 기준 문서 간 충돌 없음
- 구현 대상 파일과 수정 금지 범위 확정
- 기존 27개 테스트 통과

### SP-01 — 런·스테이지 순수 상태 기반

**할 일**

- 스테이지 정의와 경로 유효성 검사를 작성한다.
- 플레이어 영혼·덱 구성 런 상태를 작성한다.
- 진행 상태와 유효 입력을 작성한다.
- 잘못된 경로, 중복 처리와 유효하지 않은 입력 테스트를 먼저 만든다.

**완료 증거**

- SP-U01, SP-U03, SP-U04, SP-U08~SP-U11 통과
- Unity 오브젝트 없이 진행 상태 검증 가능

**실행 결과: 완료**

- 순수 상태·검증 파일 7개와 별도 EditMode 테스트 어셈블리 작성
- 신규 테스트 13/13, 기존 포함 전체 40/40 통과
- 전투 생성·UI·씬 변경 없이 SP-01 범위 유지

### SP-02 — 전투 코어 루프 연결

**할 일**

- `SoulPool`에 현재 영혼을 전달하는 호환 생성 경로를 추가한다.
- 기존 생성자의 최대 영혼 시작 동작을 보존한다.
- 진행 상태에서 스테이지별 `CoreLoopBattle`을 생성한다.
- 승리 시 남은 영혼을 저장하고 다음 적을 새로 만든다.
- 패배와 최종 보스 승리를 런 종료 상태로 연결한다.

**완료 증거**

- SP-U02, SP-U05~SP-U07, SP-U12 통과
- 기존 코어 루프 27개 테스트 회귀 통과

### SP-03 — 최소 진행 UI와 전용 씬

**할 일**

- `StageProgressionViewModel`과 Presenter를 작성한다.
- 현재 스테이지, 전체 수, 전투 종류와 런 결과를 표시한다.
- `NEXT STAGE`, 전투 행동, `RESTART RUN` 입력을 상태별로 연결한다.
- `StageProgressionTest` 씬을 만들고 코어 루프 임시 UI와 결합한다.

**완료 증거**

- 유효 상태에서만 각 입력 활성화
- View가 진행 규칙을 계산하지 않음
- 씬 누락 스크립트·깨진 참조 0건

### SP-04 — 전체 흐름 검증과 기록

**할 일**

- SP-U01~SP-U12와 SP-F01~SP-F05를 실행한다.
- 일반전 2개와 최종 보스까지 실제 흐름을 확인한다.
- 중간 패배와 양쪽 런 재시작을 확인한다.
- 기존 코어 루프 27개 테스트를 함께 실행한다.
- AI 활용 기술서, 팀 역할 기술서와 진행 기록을 갱신한다.

**완료 증거**

- 신규 테스트와 기존 테스트 전체 통과
- 런 승리·패배 수동 확인
- 씬 문제 0건
- 게임 관련 Console Error·Exception 0건

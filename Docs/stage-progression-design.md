# 런·스테이지 진행 시스템 기획서

> 프로젝트: DiaBlackJack  
> 문서 책임자: 이천서  
> 버전: v0.1  
> 상태: SP-01 반영 기준안
> 최종 갱신: 2026-07-19

## 1. 문서 목적

완료된 단일 전투 시스템과 별도로, 여러 전투를 하나의 런으로 연결하는 독립된 **런·스테이지 진행 시스템**을 정의한다.

기준 문서는 `Docs/rule.md`와 `Docs/game-design-document.md`다. 두 문서에 없는 수치나 화면 흐름은 임시 기획 결정으로 구분한다.

## 2. 작업 목표

플레이어가 런을 시작하고 고정된 순서의 전투 스테이지를 진행해 최종 보스를 쓰러뜨리거나, 영혼이 0이 되어 패배할 때까지의 흐름을 검증한다.

```text
런 시작
→ 현재 스테이지 진입
→ 기존 전투 코어 루프 실행
→ 전투 승리: 플레이어 상태 보존 후 다음 스테이지
→ 전투 패배: 런 패배
→ 최종 보스 승리: 런 승리
→ 런 재시작
```

## 3. 코어 루프와의 경계

| 구분 | 책임 |
| --- | --- |
| 전투 코어 루프 | 한 전투 안의 카드 배분, 행동, 라운드 판정, 영혼 피해와 전투 승패 |
| 런·스테이지 진행 | 현재 스테이지, 플레이어의 지속 상태, 다음 적 생성, 런 승패와 런 재시작 |

진행 시스템은 전투 규칙을 다시 계산하지 않는다. 전투의 `PlayerVictory` 또는 `PlayerDefeat` 결과만 받아 진행 상태를 바꾼다. 전투 코어 루프도 현재 스테이지 번호나 최종 보스 여부를 알지 않는다.

## 4. 용어

- **런**: 시작부터 최종 보스 승리 또는 플레이어 패배까지의 전체 세션.
- **스테이지**: 하나의 전투 또는 비전투 노드. 이번 구현에서는 전투 노드만 다룬다.
- **전투 스테이지**: 기존 전투 코어 루프를 한 번 실행하는 진행 단위.
- **최종 스테이지**: 승리 시 다음 스테이지 대신 런 승리로 전환되는 보스 전투.
- **런 상태**: 스테이지 사이에도 유지되는 플레이어 영혼, 최대 영혼과 덱 구성.

## 5. 최초 구현 범위

### 5.1 포함 범위

- 고정 선형 전투 경로
- 일반 전투와 최종 보스 전투 구분
- 현재 스테이지와 전체 스테이지 수 표시
- 전투 승리 후 다음 스테이지 진입
- 플레이어 현재 영혼과 최대 영혼 유지
- 플레이어 덱 구성 유지 및 전투 시작 시 새 덱으로 재구성·섞기
- 스테이지마다 새 적 영혼과 적 덱 생성
- 플레이어 패배와 최종 보스 승리에 따른 런 종료
- 런 전체 재시작

### 5.2 제외 범위

- 분기 지도와 경로 선택
- 휴식, 상점, 악마 제단, 사건
- 전투 보상과 덱 추가·제거·강화 UI
- 적별 카드 효과와 보스 고유 패턴
- 저장·불러오기와 메타 성장
- 최종 아트, 사운드와 연출

제외 범위에 필요한 범용 노드 프레임워크나 빈 화면을 미리 만들지 않는다.

## 6. 최초 검증 경로

전체 프로토타입 권장 경로는 기획서의 다음 구성을 유지한다.

`튜토리얼 일반전 → 일반전 → 휴식 → 일반전 → 엘리트 → 보상/휴식 → 보스`

이번 작업에서는 진행 구조만 빠르게 검증하기 위해 아래의 축소 경로를 사용한다.

`일반 전투 1 → 일반 전투 2 → 최종 보스 전투`

축소 경로의 3개 노드는 콘텐츠 확정 수량이 아니라 테스트용 임시 기획 결정이다. 경로는 코드에 개별 분기로 고정하지 않고 순서가 있는 스테이지 정의 목록으로 제공한다.

## 7. 상태 유지 규칙

### 7.1 런 시작

- 플레이어 최대 영혼과 현재 영혼은 12로 시작한다.
- 플레이어는 현재 기본 숫자 덱 구성을 사용한다.
- 현재 스테이지는 첫 번째 노드다.
- 이전 전투 결과와 런 종료 상태는 남기지 않는다.

### 7.2 스테이지 승리

- 플레이어의 남은 영혼과 최대 영혼을 런 상태에 반영한다.
- 플레이어 덱의 구성은 유지한다.
- 전투 중 손패·뽑을 더미·버린 더미의 위치 상태는 폐기한다.
- 다음 전투 시작 시 유지된 덱 구성을 합쳐 새로 섞는다.
- 적의 영혼, 덱, 손패와 전투 상태는 새로 만든다.

### 7.3 스테이지 패배

- 플레이어 영혼이 0이면 즉시 런 패배다.
- 현재 스테이지 번호를 증가시키지 않는다.
- 같은 런에서 전투만 다시 시작하는 기능은 제공하지 않는다.

### 7.4 최종 보스 승리

- 최종 스테이지 승리 시 런 승리다.
- 존재하지 않는 다음 스테이지를 만들지 않는다.
- 런 재시작 입력만 허용한다.

## 8. 화면과 입력

최소 화면에는 다음 정보만 추가한다.

- `STAGE 1 / 3` 형식의 진행도
- 일반 전투 또는 최종 보스 표시
- 스테이지 승리 후 `NEXT STAGE` 입력
- 런 승리·런 패배 결과
- 런 종료 후 `RESTART RUN` 입력

스테이지 승리 직후 자동으로 다음 전투를 시작하지 않는다. 플레이어가 결과와 유지된 영혼을 확인한 뒤 다음 스테이지로 이동하도록 한 번의 확인 입력을 둔다.

## 9. 완료 조건

- 런 시작 시 첫 번째 전투가 정상적으로 시작된다.
- 일반 전투 승리 후에만 진행도가 한 칸 증가한다.
- 스테이지 사이에 플레이어 영혼이 유지된다.
- 새 스테이지에서 적과 전투 임시 상태가 초기화된다.
- 플레이어 패배 시 진행도가 증가하지 않고 런 패배가 표시된다.
- 최종 보스 승리 시 런 승리가 표시된다.
- 런 재시작 시 영혼 12, 첫 스테이지와 새 전투로 복귀한다.
- 기존 코어 루프 EditMode 테스트가 모두 계속 통과한다.
- 진행 시스템 자동 테스트와 실제 씬 흐름이 통과한다.
- 게임 관련 Unity Console Error·Exception이 없다.

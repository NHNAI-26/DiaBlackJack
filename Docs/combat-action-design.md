> 최종 갱신: 2026-07-22

## 1. 문서 목적

전투 중 플레이어가 선택할 수 있는 행동과 패배 피해를 현행 규칙으로 통일한다. 이번 개정의 핵심은 **폴드 삭제**, **체인지의 전투 내 누적 비용화**, **스탠드 비교 패배와 버스트의 피해 분리**다.

기준 문서는 `Docs/rule.md`와 `Docs/game-design-document.md`다. 이 문서는 해당 결정을 전투 행동 단위로 구체화하며, 기존 v0.1의 폴드 및 라운드당 1회 체인지 규칙을 대체한다.

## 3. 설계 의도

폴드는 낮은 피해로 라운드를 중단하는 별도 버튼이었지만, 초심자에게 행동 수를 늘리고 스탠드의 역할과 겹쳤다. 따라서 폴드를 삭제하고, 비교 패배의 피해를 영혼 1로 낮춰 **스탠드가 위험을 멈추는 방어적 선택**까지 담당하게 한다.

체인지는 공개 정보로 인해 비공개 카드가 추론되었을 때 이를 뒤집는 최후의 수단이다. 무료 1회는 기본적인 안전망으로 남기되, 이후 사용은 영혼을 점점 더 요구한다. 이 구조는 체인지의 존재감을 유지하면서도 거짓말 탐지기와 총처럼 상대 패를 읽거나 압박하는 수단의 가치를 보존한다.

## 4. 플레이어 행동 구성

### 4.1 히트

- 공개 카드를 한 장 받는다.
- 숫자 합계가 21을 초과하면 즉시 버스트한다.
- 버스트 시 플레이어는 영혼 2를 잃고 라운드가 끝난다.

### 4.2 스탠드

- 현재 패로 더 이상 카드를 받지 않겠다고 선언한다.
- 양쪽의 행동이 끝나면 합계를 비교한다.
- 플레이어가 비교에서 패배하면 영혼 1을 잃는다.
- 별도의 폴드 행동은 없으며, 불리한 패를 더 진행하지 않는 역할은 스탠드가 맡는다.

### 4.3 카드 사용

- 보유한 카드의 조건과 효과에 따라 사용한다.
- 카드 효과가 플레이어를 버스트시키면 영혼 2 피해를 적용한다.
- 카드 효과의 세부 처리 순서는 카드 사용 기획 문서를 따른다.

### 4.4 체인지

- 플레이어 차례에 현재 비공개 카드를 교체하는 상황 행동이다.
- 한 전투 안에서 여러 번 사용할 수 있다.
- 완료된 체인지 1회마다 플레이어 차례 1회를 소비한다.
- 후보를 공개한 뒤에는 취소할 수 없다.

### 4.5 계약

- 일반 카드 덱과 분리된 악마 카드 덱을 선택해 전투 도중 제시되는 악마와 계약하는 상황 행동이다.
- 별도의 중앙 계약 오브젝트는 사용하지 않는다.
- 영혼 1을 지불하고 악마 카드 덱 위 3장 중 1장과 계약한다.
- 하나의 전투 스테이지에서 원칙적으로 1회만 사용할 수 있다.
- 카드·유물·악마 효과가 추가 계약을 명시한 특별한 상황만 횟수 제한의 예외로 처리한다.
- 지속 효과와 대가의 세부 처리는 악마 계약 기획을 따른다.

## 5. 체인지 상세 규칙

### 5.1 사용 조건

- 전투가 진행 중이고 플레이어 차례여야 한다.
- 플레이어 손에 비공개 카드가 정확히 1장 있어야 한다.
- 기존 비공개 카드를 제외하고 후보 2장을 확보할 수 있어야 한다.
- 다른 후보 선택이나 라운드 판정을 처리 중이면 사용할 수 없다.
- 현재 영혼이 체인지 비용보다 많아야 한다. 비용이 0이어도 영혼이 최소 1은 있어야 시작할 수 있다.

### 5.2 누적 비용

| 이번 전투에서 이미 완료한 체인지 | 다음 체인지 비용 |
| ---: | ---: |
| 0회 | 영혼 0 |
| 1회 | 영혼 1 |
| 2회 | 영혼 2 |
| 3회 | 영혼 3 |
| n회 | 영혼 n |

- 비용은 체인지 시작 시 확정하고 지불한다.
- 비용 지불 후 영혼이 0이 되거나 영혼이 부족하면 체인지를 시작할 수 없으며 카드와 상태는 바뀌지 않는다.
- 완료 횟수와 다음 비용은 라운드가 바뀌어도 같은 전투 안에서 유지된다.
- 전투가 끝나고 다음 전투가 시작되면 완료 횟수와 비용을 0으로 초기화한다.

### 5.3 처리 순서

1. 현재 비용과 사용 조건을 확인한다.
2. 비용을 지불하고 기존 비공개 카드를 앞면으로 공개한다.
3. 공개한 기존 비공개 카드를 버린 카드 더미로 이동한다.
4. 일반 덱에서 후보 카드 2장을 뽑아 플레이어에게만 공개한다.
5. 플레이어가 후보 1장을 선택한다.
6. 선택한 카드를 새 비공개 카드로 손에 넣는다.
7. 선택하지 않은 후보를 버린 카드 더미로 이동한다.
8. 이번 전투의 완료 횟수를 1 증가시킨다.
9. 플레이어 차례를 종료하고 적 차례로 진행한다.

기존 비공개 카드를 먼저 공개하는 것은 체인지가 상대의 추론을 완전히 지우는 행동이 아니라, 정보를 노출하는 대가를 감수하고 새 비공개 카드를 확보하는 행동임을 명확히 하기 위해서다.

### 5.4 정보 공개

- 기존 비공개 카드의 앞면은 체인지 시 양쪽 모두에게 공개된다.
- 후보 2장의 숫자와 식별 정보는 플레이어에게만 보인다.
- 적 정책에는 후보, 선택 결과와 새 비공개 카드의 숫자를 전달하지 않는다.
- 선택된 카드는 다시 비공개 상태로 놓는다.
- 유효하지 않은 후보 입력은 카드 이동이나 완료 횟수 증가 없이 선택 상태를 유지한다.

## 6. 라운드 피해와 종료

| 상황 | 피해 대상 | 영혼 피해 | 처리 |
| --- | --- | ---: | --- |
| 플레이어 스탠드 후 합계 비교 패배 | 플레이어 | 1 | 라운드 종료 |
| 플레이어 숫자 합계 버스트 | 플레이어 | 2 | 즉시 라운드 종료 |
| 플레이어 카드 효과 버스트 | 플레이어 | 2 | 즉시 라운드 종료 |
| 적의 합계 비교 패배 또는 버스트 | 적 | 1 | 라운드 종료 |

- 피해는 한 라운드에 정확히 한 번만 적용한다.
- 영혼이 0이 되면 해당 전투에서 패배하고 런 패배 처리로 연결한다.
- 정확히 21, 동점, 특수 카드에 의한 별도 판정은 상위 규칙 문서를 따른다.

## 7. 적 행동 규칙

- 적은 폴드를 사용하지 않는다.
- 적의 기본 판단 후보는 히트와 스탠드이며, 카드 사용과 계약은 개별 적 프로필에서 허용될 때만 추가한다.
- 적에게 체인지를 허용할지는 별도 밸런스 검토 전까지 미정이다.
- 적은 플레이어의 새 비공개 카드와 체인지 후보를 직접 알 수 없다.

## 8. 화면과 입력

- 화면 하단에는 항상 사용하는 `HIT`, `STAND`만 둔다.
- `FOLD` 버튼은 표시하지 않는다.
- `CHANGE`는 플레이어 비공개 카드 근처의 오브젝트형 입력으로 둔다.
- 체인지 입력에는 다음 비용을 항상 표시한다. 첫 사용은 `CHANGE 0`, 이후에는 `CHANGE 1`, `CHANGE 2`처럼 갱신한다.
- 영혼이 부족하면 체인지 입력을 비활성화하고 부족 사유를 표시한다.
- 체인지 후보 선택 중에는 다른 전투 행동 입력을 잠그고 후보 2장만 선택할 수 있게 한다.
- 체인지 후보를 확인한 뒤 취소하는 입력은 제공하지 않는다.
- 계약은 별도 악마 카드 덱 오브젝트를 선택해 시작하며, 중앙 계약 오브젝트나 상시 계약 버튼은 제공하지 않는다.

## 9. 콘텐츠 영향

- 루시퍼는 계약 즉시 무작위 악마 카드 5개 중 1개를 선택한다. 루시퍼의 대가로 영혼 1개를 잃으며, 선택한 악마 카드의 효과와 대가도 모두 적용한다.
- 독극물이 발동하면 `지금 즉시 스탠드한다` 또는 `영혼 3개를 잃는다` 중 하나를 선택한다. 현재 영혼이 3개보다 적다면 영혼 지불 선택 시 보유 영혼을 모두 잃는다. 영혼 지불을 선택한 뒤 해당 라운드에서 승리하면 영혼 5개를 회복한다.
- 폴드를 전제로 한 유물·튜토리얼·대사·UI는 삭제하거나 스탠드 비교 패배 규칙에 맞게 재설계한다.

## 10. 구현 현황과 이관 항목

폴드 삭제, 전투별 체인지 완료 횟수·누적 비용, 비용 지불 후 최소 영혼 1 유지와 기존 비공개 카드 공개 폐기는 현재 전투 코드에 이관되어 있다. 새로 확정한 계약 구조와 독극물·루시퍼 개정안은 다음 구현 작업에서 반영해야 한다.

- 일반 카드 덱과 분리된 플레이어 악마 카드 덱 상태 추가
- 전투 스테이지당 계약 완료 횟수 1회 제한과 다음 스테이지 초기화
- 카드·유물·악마 효과가 명시한 추가 계약 예외 처리
- 악마 카드 덱 오브젝트 선택, 후보 3장 표시와 1장 선택 입력 연결
- 중앙 계약 오브젝트 없이 악마 카드 덱에서 계약을 시작하도록 화면 갱신
- 독극물의 즉시 스탠드·영혼 지불·승리 회복 처리와 `스탠드할 수 없음` 효과의 우선순위 확정
- 루시퍼의 무작위 악마 카드 5개 후보 생성, 영혼 1 상실과 선택한 악마의 효과·대가 연속 처리
- 단독 전투와 런 전투의 계약 횟수·영혼·지속 효과 동기화 및 회귀 검증

## 11. 완료 조건

- 플레이어와 적의 행동 후보, UI, 판정에서 폴드가 존재하지 않는다.
- 첫 체인지는 무료이고, 이후 비용이 같은 전투에서 1씩 증가한다.
- 체인지 완료 횟수는 라운드 사이에 유지되고 다음 전투에서 초기화된다.
- 영혼이 부족한 유료 체인지는 아무 상태 변화 없이 거부된다.
- 현재 영혼이 비용과 같아 지불 후 0이 되는 체인지도 아무 상태 변화 없이 거부된다.
- 기존 비공개 카드는 공개 후 버려지고, 후보 중 선택한 카드만 새 비공개 카드가 된다.
- 후보 확인 이후 취소할 수 없으며 완료된 체인지는 차례를 소비한다.
- 스탠드 후 비교 패배는 영혼 1, 숫자·카드 효과 버스트는 영혼 2를 정확히 한 번 적용한다.
- 일반 카드 덱과 악마 카드 덱이 별도 상태와 오브젝트로 존재한다.
- 계약은 전투 스테이지당 원칙적으로 1회만 승인되고 명시적 추가 계약 효과만 예외로 동작한다.
- 계약은 중앙 버튼이 아니라 악마 카드 덱 오브젝트에서 시작한다.
- 단독 전투와 런 전투에서 같은 규칙이 작동한다.
- 관련 자동 테스트가 통과하고 게임 관련 Unity Console Error·Exception이 없다.

## 1. 문서 목적

이 문서는 일반 전투 진입 전 서로 다른 상대 후보 2명을 비교·선택하는 화면과, 선택한 적의 공개 추론 정보·엘리트 신뢰도·보스 구간 및 강행동 예고를 실제 전투 화면에 표시하는 기준을 확정한다.

적 프로필·AI·전투 생성·보상 경계는 EP-06 완료 결과를 재사용한다. 이번 작업은 그 결과를 플레이어가 보고 선택하고 전투 중 이해할 수 있게 만드는 사용자 접점이다.

최종 기획 승인과 상대 선택·전투 UI 통합 책임자는 이천서다.

## 4. 플레이 흐름

```text
런 시작
  → 상대 후보 2명 생성
  → 후보 정보 비교
  → 후보 1명 지정
  → 선택 확정
  → 선택 키로 실제 스테이지 전투 생성
  → 전투 중 등급별 AI 정보 표시
  → 승리 시 카드 보상
  → 다음 일반 전투라면 새 후보 2명 선택
  → 최종 보스는 선택 없이 고정 진입
```

현재 프로토타입의 이벤트 스테이지는 HONG의 별도 작업이다. EUI는 일반 전투 2회와 최종 보스 전투에서 필요한 선택·표시 계약을 제공하며 이벤트 상태를 임의로 만들지 않는다.

## 5. 상대 후보 생성 규칙

### 5.1 후보 풀

- 일반 후보: `gunslinger`, `cultist`, `trickster`
- 엘리트 후보: `enforcer`
- 최종 보스 `final-boss`는 일반 상대 선택 풀에서 제외한다.
- 후보 데이터는 `EnemyCombatProfileCatalog.Default.Previews`만 사용한다.
- UI나 진행 코드가 이름·영혼·보상 등급을 별도로 복제하지 않는다.

### 5.2 한 제안의 구성

- 항상 서로 다른 후보 2명을 제시한다.
- 허용 조합은 `일반+일반` 또는 `일반+엘리트`다.
- `엘리트+엘리트`, 같은 프로필 2개와 보스 포함 조합은 금지한다.
- 후보 표시 순서는 결정적 난수로 섞는다.
- 선택은 화면 위치가 아니라 안정적인 `ProfileKey`로 전달한다.

### 5.3 엘리트 등장 임시안

규칙 원문은 엘리트 등장 확률을 확정하지 않았다. 프로토타입 기본값은 **상대 제안당 35%**로 둔다.

- 35% 판정 성공: 엘리트 1명과 일반 적 1명
- 실패: 서로 다른 일반 적 2명
- 테스트에서는 0%·100%를 주입해 두 조합을 결정적으로 검증한다.
- 실제 플레이 로그가 쌓이면 첫 번째·두 번째 전투의 확률을 별도로 조정할 수 있다.
- 서로 다른 선택 화면 사이에서 같은 프로필이 다시 등장하는 것은 1차 범위에서 허용한다.

### 5.4 재생성과 확정

- 같은 선택 화면에서 후보 재추첨은 제공하지 않는다.
- 후보를 클릭하면 강조 대상만 바뀌고 실제 진행 상태는 변경되지 않는다.
- `CONFIRM OPPONENT` 입력에 성공한 순간에만 선택을 확정한다.
- 확정 뒤 취소·되돌리기는 1차 범위에서 제공하지 않는다.

## 6. 상대 선택 화면

각 후보는 다음 정보만 표시한다.

| 정보 | 출처 | 표시 예시 |
| --- | --- | --- |
| 이름 | `EnemyProfilePreview.DisplayName` | 총잡이 |
| 등급 | `EnemyProfilePreview.Grade` | NORMAL / ELITE |
| 최대 영혼 | `EnemyProfilePreview.MaximumSoul` | SOUL 5 |
| 핵심 성향 | `EnemyProfilePreview.Summary` | 스탠드 방해와 강제 드로우 중심 |
| 예상 보상 | `EnemyProfilePreview.ExpectedRewardTier` | NORMAL / HIGH-GRADE |

다음 정보는 선택 화면에 표시하지 않는다.

- 적 덱의 정확한 카드 순서
- 정책 키와 내부 점수 가중치
- 카드 물리 ID
- 다음 행동 또는 선택형 카드의 내부 판단
- 플레이어에게 공개되지 않은 전투 정보

### 6.1 선택 화면 와이어프레임

```text
DEVIL BLACKJACK RUN
STAGE 1 / 3
CHOOSE OPPONENT

┌──────────────────────┐  ┌──────────────────────┐
│ GUNSLINGER           │  │ ENFORCER             │
│ NORMAL · SOUL 3      │  │ ELITE · SOUL 5       │
│ 숫자 추측 중심       │  │ 스탠드 방해 중심     │
│ NORMAL REWARD        │  │ HIGH-GRADE REWARD    │
│       SELECT         │  │       SELECT         │
└──────────────────────┘  └──────────────────────┘

SELECTED: ENFORCER
[ CONFIRM OPPONENT ]
```

## 7. 전투 중 적 정보 표시

### 7.1 공통 정보

모든 프로필 전투는 다음 정보를 표시한다.

- 적 이름과 등급
- 현재 영혼 / 최대 영혼
- 기존 공개 패·비공개 카드·공개 합계
- 프로필 성향 요약
- 현재 등급에 맞는 AI 정보 패널

독립 `CoreLoopTest`처럼 프로필 키가 없는 전투는 기존 `ENEMY` 제목을 유지하고 전략 정보 패널을 `NO PROFILE INFORMATION`으로 표시한다. 임의 프로필을 추정하지 않는다.

### 7.2 일반 적

총잡이·광신도·사기꾼은 플레이어 비공개 카드에 대한 상위 추론 숫자 3개와 정수 백분율을 표시한다.

```text
INFERENCE
7  30%
8  25%
5  15%
```

- 확률은 AI가 실제 사용하는 공개 구성 기반 계산과 같은 결과여야 한다.
- 합계 100% 전체 목록을 모두 노출하지 않고 상위 3개만 표시한다.
- 동률이면 낮은 숫자를 먼저 표시한다.
- 비공개 카드가 없거나 추론할 수 없으면 `INFERENCE UNAVAILABLE`을 표시한다.
- 정확한 다음 행동은 표시하지 않는다.

### 7.3 엘리트

집행관은 상위 추론 숫자 2개와 신뢰도만 표시한다.

```text
LIKELY NUMBERS  4 · 7
CONFIDENCE      MEDIUM
```

- 원시 확률은 표시하지 않는다.
- 신뢰도 구간은 기존 `낮음/보통/높음` 모델을 사용한다.
- 해머·나이프의 내부 점수와 실제 다음 행동은 표시하지 않는다.

### 7.4 최종 보스

최종 보스는 숫자 목록 대신 현재 구간, 추론 방향, 신뢰도와 강행동 예고 범주를 표시한다.

| 항목 | 표시 값 |
| --- | --- |
| 구간 | SURVIVAL / PRESSURE / EXECUTION |
| 추론 방향 | LOW / BALANCED / HIGH / UNKNOWN |
| 신뢰도 | LOW / MEDIUM / HIGH |
| 강행동 예고 | NONE / NUMBER GUESS / FORCED DRAW |

`NUMBER GUESS`는 선언 숫자를, `FORCED DRAW`는 카드 ID나 실제 성공 확률을 공개하지 않는다. 예고가 사라지거나 무효화되면 다음 화면 갱신에서 즉시 `NONE`으로 바뀐다.

## 8. 상태와 입력 규칙

EUI는 진행 상태에 `OpponentSelection`을 추가한다.

```text
NotStarted
  → OpponentSelection
  → InBattle
  → RewardSelection
  → StageCleared
  → OpponentSelection 또는 고정 보스 InBattle
  → RewardSelection
  → RunVictory
```

- `OpponentSelection`에서는 전투 행동·보상·다음 스테이지 입력을 모두 거절한다.
- 선택 확정 처리 중에는 후보·확정 버튼을 잠근다.
- 알 수 없는 키, 이전 제안 ID와 현재 후보에 없는 키는 상태 변경 없이 거절한다.
- 전투 생성이 실패하면 선택 상태와 같은 후보를 유지해 재시도 또는 오류 확인이 가능해야 한다.
- 선택 성공 뒤에만 전투 씬으로 이동한다.
- 최종 보스는 후보 생성 없이 기존 고정 프로필로 바로 전투를 시작한다.

## 9. 화면 전환 원칙

| 현재 상태 | 화면 | 성공 입력 | 다음 화면 |
| --- | --- | --- | --- |
| `NotStarted` | `StageTest` | START RUN | 첫 상대 선택 화면 |
| `OpponentSelection` | `StageTest` | 후보 선택 | 같은 화면, 강조만 변경 |
| `OpponentSelection` | `StageTest` | CONFIRM | `CoreLoopTest` |
| `InBattle` | `CoreLoopTest` | 전투 종료 | `StageTest` 보상 화면 |
| `StageCleared` | `StageTest` | NEXT STAGE | 상대 선택 또는 보스 전투 |
| `RunVictory/RunDefeat` | `StageTest` | RESTART | 첫 상대 선택 화면 |

현재 View가 `OnGUI`를 사용하므로 1차 범위는 새 씬이나 직렬화 필드를 요구하지 않는다.

## 10. 가독성과 접근성

- 색상만으로 일반·엘리트·보스를 구분하지 않고 텍스트 등급을 함께 표시한다.
- 선택된 후보는 테두리·`SELECTED` 문구와 버튼 상태를 함께 사용한다.
- 신뢰도와 보스 예고는 색상과 영문 범주를 함께 표시한다.
- 1280×720과 1920×1080에서 후보 카드·확정 버튼·전투 행동 버튼이 겹치지 않아야 한다.
- 긴 성향 요약은 후보 카드 안에서 줄바꿈하며 버튼 영역을 밀어내지 않는다.
- 1차 프로토타입 버튼 문구는 기존 화면과 맞춰 영문 대문자를 사용한다. 정식 한국어 현지화는 별도 작업이다.

## 12. 완료 조건

- 일반 전투마다 서로 다른 후보 2명이 표시된다.
- 한 화면에 엘리트가 최대 1명이며 보스는 나오지 않는다.
- 미리보기 정보와 선택 뒤 실제 적·영혼·보상이 일치한다.
- 확정 전에는 진행·전투 상태가 바뀌지 않는다.
- 일반 적은 상위 3개 확률, 엘리트는 상위 2개와 신뢰도, 보스는 구간·방향·신뢰도·예고만 표시한다.
- 플레이어 비공개 값·덱 순서·정확한 다음 행동이 UI 데이터에 포함되지 않는다.
- 선택→전투→보상→다음 선택과 보스→런 승리 흐름이 반복 실행된다.
- 기존 고정 프로필·프로필 없는 전투가 회귀 없이 유지된다.
- 전체 EditMode, 실제 두 씬과 Console 검증이 통과한다.
- AI 활용과 이천서의 실제 담당 기록이 갱신된다.

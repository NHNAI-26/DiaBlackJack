## 1. 문서 목적

이 문서는 `rule.md`의 카드 사용 규칙을 현재 전투 구조에서 바로 구현할 수 있는 범위로 구체화한다. 폴드는 현행 행동에서 삭제되었으며, 폴드에 의존하던 독극물과 루시퍼는 각각 즉시 스탠드 선택과 영혼 1 대가로 개정한다.

이번 작업은 플레이어가 일반 카드의 효과를 직접 사용하는 경험을 완성하는 데 집중한다. 자동 발동 카드, 악마 계약, 특수 카드와 적의 자율 카드 사용은 같은 효과 처리 기반을 재사용할 수 있지만 이번 완료 조건에는 포함하지 않는다.

규칙 원문에 없는 처리 순서나 모호한 카드 문구는 프로토타입 임시 결정으로 명시한다. 최종 기획 승인과 변경 결정은 이천서가 담당한다.

## 4. 작업 범위

### 4.1 포함 범위

- 카드 정의와 전투 중 카드 인스턴스의 사용 가능·처리 중·사용 완료 상태
- 공개 카드 사용 및 비공개 카드 공개 후 사용
- 카드별 사용 조건과 유효 대상 검사
- 카드 효과를 위한 숫자·카드 선택
- 효과 처리 중 카드 공개, 드로우, 버림, 비공개 카드 교체, 강제 히트와 직접 버스트
- 카드 효과로 발생한 즉시 라운드 종료와 영혼 피해
- 효과 종료 후 적 차례 또는 다음 플레이어 차례 연결
- 최소 전투 화면의 카드 사용·선택·결과 표시
- 런 전투에서 카드 사용 결과와 지속 영혼 동기화
- 규칙, 표시, 세션, 진행 연결의 자동·수동 검증

### 4.2 제외 범위

- 자동 발동 카드 5종
- 악마 계약과 악마 패시브에 의한 카드 효과 변경
- 사탄의 권능, 오망성 등 특수 카드
- 적이 스스로 카드 사용 행동을 고르는 AI
- 카드 보상, 강화, 삭제와 덱 편집 UI
- 카드 애니메이션, 최종 일러스트, 사운드와 현지화 마감
- 범용 스크립팅 언어, 문자열 기반 효과 해석기와 외부 데이터 포맷
- 저장 파일 버전 변경

제외 항목은 카드 사용 작업의 완료를 막지 않는다. 다만 신규 구조가 해당 기능을 불필요하게 차단해서는 안 된다.

## 5. 공통 카드 사용 규칙

### 5.1 사용 조건

다음 조건을 모두 만족해야 카드를 사용할 수 있다.

1. 전투가 진행 중이고 현재 상태가 플레이어 행동 가능 상태다.
2. 선택한 카드가 플레이어 손에 실제로 존재한다.
3. 카드 정의가 `수동 사용` 유형이다.
4. 해당 카드가 아직 사용되지 않았고 다른 효과를 처리 중이지 않다.
5. 카드별 필수 대상, 덱 잔량과 선택지가 존재한다.
6. 런 전투라면 진행 상태가 `InBattle`이다.

검사 실패는 `false`를 반환하며 카드 공개, 사용 표시, 카드 이동, 차례 변경을 포함한 어떤 상태도 바꾸지 않는다.

### 5.2 사용 처리 순서

1. 공통 조건과 카드별 사전 조건을 모두 검사한다.
2. 비공개 사용 카드라면 공개한다.
3. 사용 카드를 `처리 중`으로 잠근다.
4. 필요한 선택지를 플레이어에게 제시한다.
5. 카드별 효과를 정해진 순서대로 실행한다.
6. 각 효과 명령 뒤 버스트·라운드·전투 종료를 확인한다.
7. 전투가 계속되면 사용 카드를 `사용 완료`로 표시한다.
8. 효과가 완전히 끝났으면 차례를 소비하고 적 차례를 실행한다.
9. 적이 이미 스탠드 상태라면 기존 규칙에 따라 플레이어에게 차례가 돌아올 수 있다.

효과가 라운드나 전투를 끝냈다면 8~9단계를 실행하지 않는다. 사용 카드가 효과 중 버려져도 해당 인스턴스의 사용 완료 상태는 유지되며, 덱에서 다시 뽑힐 때 초기화한다.

### 5.3 공개 정보와 선택 취소

- 비공개 카드는 사용 승인 시 공개된다. 공개 후에는 다시 숨기지 않는다.
- 덱 위 카드를 확인하는 효과는 카드 소유자에게만 실제 숫자를 표시한다.
- 상대 비공개 값은 효과 판정 계층에서만 비교하고 UI나 일반 표시 모델에 노출하지 않는다.
- 사용이 승인된 뒤에는 취소할 수 없다. 확인한 정보만 얻고 행동을 되돌리는 악용을 막기 위한 결정이다.
- UI에서 카드 설명을 열거나 사용 가능 여부를 확인하는 행위는 사용 승인이 아니며 자유롭게 취소할 수 있다.

### 5.4 사용 상태 수명

- `사용 가능`: 손에 들어온 수동 카드의 초기 상태
- `처리 중`: 사용이 승인되어 효과나 선택이 끝나지 않은 상태
- `사용 완료`: 이번에 손에 들어온 뒤 효과를 이미 사용한 상태

카드가 버린 더미와 덱을 거쳐 다시 손에 들어올 때 `사용 가능`으로 초기화한다. 단순히 공개·비공개 상태가 바뀌거나 손 안에서 위치가 바뀌는 것만으로는 초기화하지 않는다.

## 6. 1차 카드별 확정 규칙

### 6.1 수정 구슬 — 숫자 5

#### 사용 조건

- 플레이어 덱에서 카드 2장을 확보할 수 있어야 한다.

#### 처리

1. 덱 위 카드 2장을 덱에서 임시로 분리해 플레이어만 확인한다.
2. `가져오지 않음`, 첫 번째 카드, 두 번째 카드 중 하나를 선택한다.
3. 선택한 카드가 있으면 공개 상태로 플레이어 손에 추가한다.
4. 선택하지 않은 카드는 덱 위로 되돌린다.
5. 되돌아간 카드끼리의 기존 상대 순서는 유지한다.
6. 아무 카드도 고르지 않으면 다음 드로우 순서는 효과 사용 전과 같다.
7. 공개 카드가 추가되었다면 플레이어 버스트 여부를 즉시 확인한다.

수정 구슬 자체는 손에 남고 사용 완료 상태가 된다. 이번 범위에는 자동 발동 카드가 덱에 없으므로 선택 카드의 자동 효과는 처리하지 않는다.

### 6.2 위협용 해머 — 숫자 6

위협용 해머는 **상대의 공개 카드 1장을 제거하는 공격 카드**다. 자신의 공개 카드를 비용으로 버리게 한 이전 세부 기획은 폐기한다.

#### 사용 조건

- 상대에게 선택 가능한 공개 카드가 한 장 이상 있어야 한다.
- 상대가 스탠드 상태라면 상대에게 비공개 카드가 정확히 한 장 있고 상대 덱이 한 장을 제공할 수 있어야 한다.

#### 처리

1. 플레이어가 상대의 공개 카드 한 장을 선택해 버린다.
2. 상대가 스탠드 상태가 아니면 효과를 끝낸다.
3. 상대가 스탠드 상태라면 스탠드를 취소한다.
4. 상대의 기존 비공개 카드를 공개하지 않고 버린다.
5. 상대 덱에서 새 카드 한 장을 비공개 상태로 받게 한다.
6. 교체 결과 상대가 버스트 조건을 만족하면 즉시 라운드를 끝낸다.

해머 자체는 별도 문구가 없으므로 플레이어의 손에 남고 사용 완료 상태가 된다. 상대가 스탠드 상태인데 비공개 카드 교체를 완전하게 수행할 수 없다면 사용 자체를 승인하지 않는다.

### 6.3 리볼버 — 숫자 7·8

#### 사용 조건

- 상대에게 비공개 카드가 정확히 한 장 있어야 한다.

기본 라운드 규칙상 상대 비공개 카드는 항상 한 장이며 두 장 이상인 정상 상태는 없다. 다중 비공개 카드를 위한 별도 선택·추측 규칙은 지원하지 않고, 비공개 카드가 없으면 대상 없음으로 사용을 거절한다.

#### 처리

1. 플레이어에게 1~10 중 숫자 하나를 선택하게 한다.
2. 선택한 숫자를 상대 비공개 카드 숫자와 내부적으로 비교한다.
3. 같으면 상대를 카드 효과 버스트로 처리한다.
4. 다르면 추가 효과 없이 끝낸다.

비공개 카드 숫자는 비교를 위해 공개하지 않는다. 성공 UI는 선택 숫자가 맞았다는 사실만, 실패 UI는 틀렸다는 사실만 표시한다. 직접 버스트는 일반 적 버스트와 동일하게 적 영혼 1 피해를 적용하며 카드 문구에 별도 피해가 있을 때만 추가 피해를 적용한다.

### 6.4 보위 나이프 — 숫자 9·10

#### 사용 조건

- 상대 공개 카드 합이 16 이하여야 한다.
- 상대 덱에서 카드 한 장을 확보할 수 있어야 한다.

#### 처리

1. 상대가 덱에서 카드 한 장을 공개 상태로 강제 드로우한다.
2. 상대 전체 카드 합이 버스트라면 즉시 라운드를 끝낸다.
3. 버스트가 아니라면 상대의 선택 정책이 추가 카드를 유지할지 버릴지 결정한다.
4. 현재 단순 적 정책은 버스트하지 않은 강제 드로우 카드를 유지한다.
5. 향후 적 행동 AI가 추가되면 같은 선택 정책 경계를 교체한다.

원문의 선택권을 보존하기 위해 유지·버림 결정은 별도 정책으로 둔다. 이번 프로토타입에서 항상 유지하는 것은 임시 AI 결정이며 카드 규칙의 영구 변경이 아니다.

## 7. 효과 중 승패와 우선순위

효과 처리 중에는 다음 순위를 적용한다.

1. 명시적 카드 이동이나 수치 변경을 한 명령 단위로 완료한다.
2. 직접 버스트 효과를 판정한다.
3. 양측 전체 카드 합의 숫자 버스트를 판정한다.
4. 버스트가 있으면 즉시 라운드 피해와 전투 종료를 처리한다.
5. 전투가 계속될 때만 다음 명령이나 선택으로 진행한다.
6. 모든 효과가 끝났을 때만 적 차례로 넘긴다.

같은 명령에서 양측이 동시에 버스트하면 전체 게임 기획서의 결정대로 효과를 발생시킨 쪽이 패배한다. 1차 카드에는 동시 버스트 명령이 없지만 공통 처리 기준에는 포함한다.

## 8. 화면과 입력

### 8.1 평상시

- 플레이어 카드마다 숫자, 이름, 공개 여부, 사용 가능 여부를 표시한다.
- 사용할 수 있는 카드는 `USE` 입력을 제공한다.
- 사용 완료 카드는 `USED`로 표시하고 입력을 비활성화한다.
- 조건이 맞지 않는 카드는 비활성화하되, 최소 디버그 UI에는 짧은 사유를 표시한다.
- 비공개 카드는 플레이어 자신의 카드이므로 숫자와 효과 설명을 볼 수 있다. 상대 화면에는 계속 `?`로 표시한다.

### 8.2 효과 선택 중

- 히트, 스탠드, 체인지, 계약과 다른 카드 사용 입력을 모두 잠근다.
- 한 번에 하나의 질문과 선택지만 표시한다.
- 수정 구슬은 두 카드와 `가져오지 않음`을 표시한다.
- 해머는 버릴 수 있는 상대의 공개 카드만 표시한다.
- 리볼버은 1~10 숫자 선택을 표시한다.
- 처리 중에는 재시작을 제외한 별도 취소 입력을 제공하지 않는다.

### 8.3 결과 피드백

- 사용 카드 이름과 핵심 결과를 한 줄 전투 로그로 남긴다.
- 비공개 정보는 로그에 기록하지 않는다.
- 카드 효과로 버스트하면 일반 히트 버스트와 구분되는 원인을 표시한다.
- 런 승리·패배 화면은 기존 진행 시스템 표시를 그대로 사용한다.

## 10. 완료 조건

다음을 모두 만족하면 카드 사용 시스템의 현행 기획 이관을 완료한다. 위협용 해머는 2026-07-22에 상대 공개 카드를 버리는 현행 규칙으로 코드·적 AI·GameScene 표시·자동 테스트 이관을 마쳤다. 독극물과 루시퍼는 별도 후속 구현 범위다.

- 일반 수동 카드 4종이 문서의 조건과 처리 순서대로 작동한다.
- 잘못된 상태·카드·선택은 전투 상태를 바꾸지 않는다.
- 비공개 카드 사용은 공개 이후 취소할 수 없다.
- 카드 한 장은 다시 뽑기 전까지 한 번만 사용된다.
- 선택 중 다른 전투 행동이 실행되지 않는다.
- 효과 중 버스트가 즉시 라운드와 전투 결과에 반영된다.
- 독립 전투와 런 전투에서 같은 카드 사용 결과가 나온다.
- 런 전투 종료 시 지속 영혼과 진행 상태가 정확히 동기화된다.
- 카드 효과가 상대 비공개 정보를 표시 모델과 로그에 노출하지 않는다.
- 신규·기존 EditMode 테스트, 두 테스트 씬과 실제 화면 흐름이 통과한다.
- AI 활용 기술서와 이천서의 실제 담당 기록을 단계마다 갱신한다.

### 12.5 폴드 의존 카드의 확정 대체안

#### 독극물 — 숫자 2, 자동 발동

발동 시 다음 중 하나를 선택한다.

- 지금 즉시 스탠드한다.
- 영혼 3개를 잃는다. 현재 영혼이 3개보다 적다면 현재 보유한 영혼을 모두 잃는다.

영혼을 지불하고 해당 라운드에서 승리하면 영혼 5개를 회복한다. 회복은 최대 영혼을 넘을 수 없다. 즉시 스탠드와 스탠드 불가 효과가 동시에 적용될 때의 우선순위는 후속 공통 판정 설계에서 확정한다.

#### 루시퍼 — 악마 카드

계약 즉시 무작위 악마 카드 5개 중 하나를 선택해 계약한다. 루시퍼의 대가로 영혼 1개를 잃으며, 선택한 악마의 능력과 대가도 모두 적용한다. 후보 출처, 루시퍼 자신과 중복 악마의 등장 여부는 악마 카드 덱 초기 구성과 함께 확정한다.

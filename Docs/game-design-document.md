## 1. 프로젝트 개요

| 항목 | 내용 |
| --- | --- |
| 프로젝트명 | 데블랙잭 |
| 영문 가제 | DiaBlackJack |
| 장르 | 1인용 블랙잭 기반 덱 빌딩 로그라이트 |
| 플랫폼 | PC |
| 플레이 인원 | 1인 |
| 목표 플레이 시간 | 1런 15~20분 |
| 개발 규모 | 3인, 2주 프로토타입 |
| 핵심 대상 | 카드 게임 규칙을 빠르게 이해하고, 확률 추론과 고위험 선택을 즐기는 플레이어 |

### 1.1 한 문장 정의

> 영혼을 체력과 화폐로 사용하며, 카드 효과와 악마 계약으로 상대의 블랙잭을 붕괴시키는 1대1 덱 빌딩 로그라이트.

### 1.2 플레이어 판타지

플레이어는 악마에게 납치되어 지옥의 블랙잭 대회에 참가한다. 정직하게 21을 노릴 수도 있지만, 총기·흉기·마법·악마의 힘으로 상대의 패를 조작하고 강제로 버스트시킬 수도 있다. 승리를 위해 자신의 생명인 영혼까지 거래해야 한다.

### 1.3 핵심 재미

1. **숫자와 효과의 이중 선택**: 좋은 합계를 만들지, 불리한 숫자를 감수하고 강력한 효과를 사용할지 판단한다.
2. **제한된 정보의 심리전**: 공개 정보로 비공개 카드를 추론하고, 적 AI가 자신을 어떻게 추정하는지 역이용한다.
3. **영혼의 이중 용도**: 생존 자원인 영혼을 성장에 소비하는 긴장감을 만든다.
4. **규칙 파괴의 쾌감**: 악마 계약으로 평범한 블랙잭 규칙을 일시적으로 뒤집는다.

## 2. 제품 목표와 범위

### 2.1 프로토타입의 검증 목표

- 카드 숫자와 전투 효과를 함께 고려하는 선택이 실제로 재미있는가?
- 적의 비공개 카드 확률을 추론하는 과정이 이해 가능하고 유용한가?
- 영혼을 체력과 계약 비용으로 함께 쓰는 구조가 의미 있는 위험 선택을 만드는가?
- 한 전투가 일반 2~4분, 엘리트 4~6분, 보스 6~8분 안에 끝나는가?
- 첫 5분 안에 히트, 스탠드, 카드 사용, 추론, 계약, 버스트, 보상을 모두 경험할 수 있는가?

### 2.2 필수 구현 범위(MVP)

- 상대 선택 전투 2회, 전투 후 이벤트 2회, 최종 보스
- 일반 적 3종, 엘리트 1종, 보스 1종
- 히트, 스탠드, 카드 사용, 체인지, 계약
- 일반 카드 1~10 기본 세트 및 핵심 효과 카드
- 악마 4종 우선 구현: 사탄, 벨페고르, 마몬, 레비아탄
- 영혼 피해, 덱·버린 카드 더미 순환, 에이스 계산
- 공개 정보만 사용하는 적 AI와 추론 확률 UI
- 전투 보상: 카드 3장 제시 후 1장 선택 또는 건너뛰기

### 2.3 후순위 또는 제외 범위

- 메타 성장, 복수 캐릭터와 복수 지역
- 복잡한 카드 강화 트리와 다중 악마 계약
- 바포메트와 모략 카드 묶음
- 모든 자동 발동 카드와 고급 보스 전용 규칙

일정 위험이 발생하면 후순위 항목부터 제외한다. 단, AI 추론 UI와 영혼 계약은 게임의 차별점이므로 축소하더라도 제거하지 않는다.

## 3. 게임 진행 구조

### 3.1 전체 흐름

1. 플레이어는 영혼 12와 기본 덱으로 런을 시작한다.
2. 지도 없이 `전투 → 전투 후 이벤트 → 전투 → 전투 후 이벤트 → 보스`의 고정된 순서를 진행한다.
3. 일반 전투 진입 전 서로 다른 상대 후보 2명 중 한 명을 선택한다. 후보 중 엘리트는 최대 1명만 등장한다.
4. 전투에서 적 영혼이 0이 될 때까지 여러 블랙잭 라운드를 반복한다.
5. 전투에서 승리하면 카드 보상과 골드를 획득한다. 강한 적일수록 더 많은 골드를 주며 엘리트는 일반 적보다 더 많은 골드를 확정 지급한다.
6. 다음 전투가 남아 있다면 상점 또는 무작위 사건으로 이동한다. 첫 개발에서는 상점만 등장한다.
7. 모든 전투 승리 후 카드 보상 3장 중 1장을 선택하거나 보상을 받지 않고 건너뛴다. 플레이어 영혼은 자동 회복되지 않는다.
8. 최종 보스를 처치한 경우에도 보상 선택 또는 건너뛰기를 완료한 뒤 런 승리로 확정한다. 플레이어 영혼이 0이 되면 보상 없이 런이 종료된다.

### 3.2 정식 런 경로

`상대 2명 중 선택 → 전투 → 카드·골드 보상 → 상점/사건 → 상대 2명 중 선택 → 전투 → 카드·골드 보상 → 상점/사건 → 보스`

분기 지도는 구현하지 않는다. 경로 선택 대신 전투마다 제시되는 상대 2명 중 한 명을 고르는 방식으로 전투의 위험과 보상을 선택한다. 일반 전투 후보 조합은 `일반+일반` 또는 `일반+엘리트`만 허용하며, 엘리트 2명이 동시에 제시되지는 않는다. 엘리트의 등장 확률과 보상 차이는 플레이 테스트 후 조정한다.

전투 후 이벤트는 상점 또는 무작위 사건 중 하나를 결정한다. 휴식은 상점의 골드 소비 서비스로 통합한다. 첫 개발에서는 사건을 제외하고 두 이벤트 슬롯 모두 상점으로 고정하며, 이후 등장 가중치와 중복 규칙을 확정한다.

### 3.3 세션 목표

| 구간 | 목표 시간 | 학습/경험 |
| --- | ---: | --- |
| 첫 전투 | 3분 이내 | 히트, 스탠드, 카드 효과, 버스트 |
| 두 번째 전투 | 누적 5분 이내 | AI 추론과 첫 악마 계약 |
| 중반 | 6~12분 | 덱 변화, 영혼 보존과 소비의 갈등 |
| 보스 | 6~8분 | 빌드 검증과 고유 규칙 대응 |

## 4. 전투 규칙

### 4.1 전투 단위

- **런**: 시작부터 최종 보스 승리 또는 플레이어 패배까지.
- **스테이지**: 전투·보상·상점·무작위 사건 등 하나의 진행 단위.
- **라운드**: 양측이 두 장을 받은 뒤 한쪽이 버스트하거나 양측이 스탠드할 때까지.
- **차례**: 플레이어나 적이 행동 하나를 선택하는 단위.

### 4.2 라운드 시작

- 양측은 자신의 덱에서 카드 2장을 뽑는다.
- 각각 1장은 공개, 1장은 비공개로 놓는다.
- 적의 비공개 카드는 플레이어와 AI 의사결정 로직 모두 직접 열람할 수 없다.
- 최초 배분에서는 자동 발동 효과가 발동하지 않는다.
- 플레이어가 먼저 행동한다. 보스의 선공 변경은 프로토타입에서 제외한다.

### 4.3 플레이어 행동

| 행동 | 규칙 |
| --- | --- |
| 히트 | 덱 맨 위 카드 1장을 공개 카드로 추가한다. |
| 스탠드 | 이번 라운드의 추가 행동을 종료한다. 카드 효과로 취소될 수 있다. |
| 카드 사용 | 공개 카드 1장의 효과를 발동한다. 비공개 카드는 공개한 뒤 사용할 수 있다. |
| 체인지 | 기존 비공개 카드를 공개해 버리고 덱 위 2장을 혼자 확인해 1장을 새 비공개 카드로 선택한다. 같은 전투의 체인지 비용은 `0→1→2→3…` 영혼으로 증가하며 다음 전투에서 초기화한다. 비용 지불 후 영혼이 최소 1 남아야 한다. |
| 계약 | 일반 카드 덱과 분리된 악마 카드 덱을 선택하고 영혼 1을 지불해 위 3장 중 1장과 계약한다. 전투 스테이지당 원칙적으로 1회만 사용할 수 있다. |

각 행동은 기본적으로 차례 하나를 소모한다. 카드 문구가 명시한 경우에만 추가 행동이나 즉시 처리를 허용한다.

### 4.4 카드 합계와 에이스

- 전투 모델은 공개 합계와 비공개 카드를 포함한 전체 합계를 각각 계산한다.
- 히트·강제 히트·카드 효과 도중의 중간 버스트는 공개 합계만 사용한다.
- 양쪽 모두 스탠드한 최종 승부에서는 비공개 카드를 공개하고 전체 합계를 사용한다.
- 에이스는 1 또는 11 중 합계가 21 이하에서 가장 커지는 값을 자동 선택한다.
- 악마 계약이 생성한 특수 카드에 숫자가 명시되어 있으면 그 숫자를 합계에 포함한다. 숫자가 없는 계약 상태·카운터·표식은 합계에 포함하지 않는다.

### 4.5 버스트 정의

다음 중 하나라도 발생하면 해당 대상은 버스트한다.

- 라운드 진행 중 공개 합계가 21을 초과한다.
- 양쪽 스탠드 후 비공개 카드를 포함한 최종 합계가 21을 초과한다.
- 카드 또는 계약 효과가 대상을 버스트시킨다.
- 해당 라운드에 적용되는 특수 패배 조건을 충족한다.

버스트한 대상은 즉시 라운드에서 패배한다. 양측이 같은 효과 처리 중 동시에 버스트하면 효과를 발생시킨 쪽이 패배한 것으로 처리한다. 이 규칙은 무한 연쇄와 판정 모호성을 막기 위한 프로토타입 결정이다.

### 4.6 양측 스탠드 후 승패

1. 비공개 카드를 모두 공개한다.
2. 21을 초과한 대상은 비교에서 제외한다.
3. 21 이하에서 합계가 21에 가까운 쪽이 승리한다.
4. 양측 모두 21을 초과했다면 21과의 차이가 작은 쪽이 승리한다.
5. 합계가 같으면 플레이어 승리로 처리한다.

### 4.7 영혼 피해

| 결과 | 플레이어 피해 | 적 피해 |
| --- | ---: | ---: |
| 플레이어 스탠드 후 합계 비교 패배 | 1 | 0 |
| 플레이어 버스트 | 2 | 0 |
| 적 버스트 | 0 | 1 |
| 일반 합계 비교 승리 | 0 | 1 |
| 정확히 21로 플레이어 승리 | 0 | 2 |
| 카드·계약 효과 | 효과 명시값 | 효과 명시값 |

폴드 삭제 뒤 안전한 손실 선택은 스탠드가 담당한다. 플레이어가 스탠드 후 합계 비교에서 패배하면 영혼 1, 숫자 초과 또는 카드 효과로 버스트하면 영혼 2를 잃는다. 적은 버스트와 합계 비교 패배 모두 영혼 1을 잃는다.

### 4.8 라운드 종료와 덱 순환

- 버스트 또는 양측 스탠드 중 하나가 발생하면 라운드가 끝난다.
- 사용한 카드와 라운드 종료 시 보유 카드는 버린 카드 더미로 이동한다.
- 덱이 소진되면 버린 카드 더미를 섞어 새 덱으로 사용한다.
- 적을 처치하면 적 덱은 폐기되고, 플레이어의 덱과 버린 카드 더미는 다음 전투 시작 시 합쳐 섞는다.
- 자동 발동 카드는 효과 처리 후 버린다. 지속 카드만 공개 영역에 남는다.

## 5. 영혼과 성장

### 5.1 영혼 설계

- 시작 영혼: 12
- 최대 영혼: 기본 12. 유물이나 보상으로만 증가 가능.
- 적 영혼: 일반 2~3, 강한 일반 3~4, 엘리트 4~5, 보스 6~8.
- 전투 종료 시 자동 회복 없음.
- 영혼 0 도달 즉시 런 패배.

영혼을 1 소모하는 계약은 현재 생존 가능성을 직접 낮춘다. 계약 확인창에는 계약 후 남는 영혼과 예상 위험도를 항상 표시한다.

### 5.2 전투 보상

적을 쓰러뜨리면 카드 보상 3장을 제시한다. 플레이어는 다음 중 하나를 선택한다.

- 제시된 카드 3장 중 1장을 현재 런 덱에 추가한다.
- 아무 카드도 선택하지 않고 보상을 건너뛴다.

일반 적 보상은 일반 카드 보상 풀에서 3장을 뽑는다. 엘리트 적 보상은 제시하는 3장 모두 높은 등급의 카드로 확정한다. 프로토타입은 `battle-reward-design.md`의 명시적 일반·높은 등급 정의 키 목록을 사용하고, 정식 높은 등급 기준은 카드 등급 체계를 확정할 때 교체한다.

최종 보스 보상도 높은 등급 풀에서 서로 다른 카드 3장을 제시한다. 선택하거나 건너뛴 뒤 `RunVictory`로 전환하며, 선택한 카드는 최종 런 덱과 결과 기록에 포함한다. 메타 성장 시스템이 확정되기 전에는 다음 런으로 계승하지 않는다.

보상을 건너뛸 수 있게 해 덱이 불필요하게 비대해지는 것을 방지한다. 카드 제거·강화, 영혼 회복과 유물 획득은 전투 보상에 섞지 않고 상점·무작위 사건 등 별도의 성장 경로에서 제공한다.

### 5.3 골드와 상점 휴식

전투 승리 시 적의 강함에 따른 골드를 지급한다. 엘리트는 일반 적보다 더 많은 골드를 확정 지급하며, 같은 등급 안에서도 더 강한 적이 더 많은 골드를 주는 경향을 갖는다. 정확한 지급량은 적 프로필 데이터로 관리한다.

상점에서는 골드를 사용해 상품과 서비스를 구매한다. 판매 카드는 왼쪽 일반 카드 3장과 오른쪽 악마 카드 2장으로 고정하며 각 카드의 재고는 1개다. 일반 카드는 일반 카드 덱에, 악마 카드는 악마 카드 덱에 추가한다. 구매한 슬롯은 `SOLD OUT` 표지로 교체하고 나머지 상품은 유지하며, 새로고침과 재입고는 제공하지 않는다.

휴식은 독립 노드가 아니라 위스키 오브젝트로 제공하는 상점 서비스이며, 한 상점에서 1회만 골드를 지불해 영혼을 회복한다. 라이터를 통한 일반 카드 제거도 방문당 1회 제공한다. 카드 강화와 유물은 상점 콘텐츠 범위에 따라 추후 제공한다.

영혼 회복은 최대치를 넘을 수 없다. 판매 카드와 모든 서비스의 가격, 휴식 회복량은 아직 확정하지 않으며 골드 지급량 및 계약 비용과 비교해 조정한다. 상점 콘셉트 이미지의 가격과 영혼 수치는 화면 구성을 보여 주는 예시에 불과하고 밸런스 값으로 사용하지 않는다. 현재 골드로 구매할 수 없는 가격은 빨간색으로 표시한다.

### 5.4 카드 강화 원칙

강화는 숫자보다 효과를 개선한다. 숫자를 바꾸면 블랙잭 확률 구조와 카드 정체성이 동시에 변하므로 프로토타입에서는 금지한다.

예시:

- 수정 구슬: 확인 2장 → 3장
- 리볼버: 숫자 1개 선언 → 조건부 2개 선언
- 보위 나이프: 강제 히트 후 추가 카드 버림 제거

## 6. 덱과 카드 시스템

### 6.1 시작 덱

기본안은 숫자 1~10을 2장씩 포함한 20장이다. 에이스 역할의 숫자 1은 자동으로 1 또는 11로 계산한다.

프로토타입 첫 플레이 테스트에서 20장 덱을 우선 사용한다. 평균 전투 시간이 길거나 원하는 효과 카드 재등장 빈도가 낮으면 16장 구성으로 축소한다.

### 6.2 일반 카드 역할

| 숫자 | 대표 카드/역할 | 기획 의도 |
| ---: | --- | --- |
| 1 | 에이스 | 합계 안정성 |
| 2~4 | 기본 카드 및 자동 발동 | 낮은 숫자를 유틸리티와 연계 |
| 5 | 수정 구슬 | 다음 카드 탐색과 덱 조작 |
| 6 | 위협용 해머 | 상대 공개 카드 제거, 스탠드 해제와 비공개 카드 교체 |
| 7~8 | 리볼버 | 비공개 숫자 추측 공격 |
| 9~10 | 보위 나이프 | 낮은 공개 합계의 상대에게 강제 히트 |

### 6.3 카드 사용 공통 규칙

- 사용 가능한 카드는 테두리와 아이콘으로 강조한다.
- 카드를 사용해도 그 카드의 숫자는 효과가 명시하지 않는 한 합계에 남는다.
- `버린다`가 명시된 카드는 합계에서 제거되고 버린 카드 더미로 간다.
- 비공개 카드는 공개하는 순간 상대와 AI의 공개 정보가 된다.
- 한 카드의 효과는 한 번만 사용한다. 사용 여부를 카드 위 아이콘으로 표시한다.
- 대상이 없는 효과는 사용할 수 없다.

### 6.4 자동 발동 카드

- 최초 2장 배분 중에는 발동하지 않는다.
- 히트, 강제 히트, 카드 효과로 공개 영역에 추가되었을 때 발동한다.
- 처리 순서는 `카드 공개 → 자동 효과 → 버스트 확인 → 다음 효과`이다.
- 효과 처리 후 기본적으로 버린다. 카드에 `지속`이 있으면 공개 영역에 남긴다.
- 여러 장이 동시에 추가되면 추가된 순서대로 처리한다.

폴드 삭제에 따라 독극물과 루시퍼는 다음 효과로 개정한다.

- **독극물:** 발동 즉시 `지금 즉시 스탠드한다` 또는 `영혼 3개를 잃는다` 중 하나를 선택한다. 현재 영혼이 3개보다 적다면 영혼 지불 선택 시 보유 영혼을 모두 잃는다. 영혼 지불을 선택한 뒤 해당 라운드에서 승리하면 영혼 5개를 회복하며 최대 영혼을 넘을 수 없다.
- **루시퍼:** 계약 즉시 무작위 악마 카드 5개 중 1개를 선택한다. 루시퍼의 대가로 영혼 1개를 잃고, 선택한 악마 카드의 효과와 대가도 모두 적용한다.

독극물의 즉시 스탠드와 `스탠드할 수 없음` 효과가 충돌할 때의 우선순위, 루시퍼 후보 풀·자기 자신 포함 여부·중복 처리 규칙은 구현 전 공통 규칙으로 확정한다.

### 6.5 카드 문구 표준

모든 카드는 아래 순서로 작성한다.

`발동 조건 → 비용/대상 → 효과 → 처리 후 위치`

예: `상대의 공개 카드 1장을 선택해 버린다. 상대가 스탠드 상태라면 스탠드를 취소하고 비공개 카드 1장을 버리게 한 뒤 새 비공개 카드 1장을 뽑게 한다.`

`카드 소유자` 대신 `자신`, 반대편은 `상대`로 통일한다. 취소·교체·버림의 처리 시점을 카드 데이터에 명시한다.

## 7. 악마 계약

### 7.1 프로토타입 제공 방식 — 기획 확정

- 일반 카드 덱과 악마 카드 덱은 테이블 위에서 서로 구분되는 별도 덱 오브젝트로 둔다.
- 전투 시작 시 악마 덱을 섞는다.
- 플레이어가 악마 카드 덱을 선택해 `계약`을 시작하면 영혼 1을 먼저 지불한다.
- 악마 덱 위 3장을 제시하고 1장을 선택한다. 나머지는 악마 버린 더미로 보낸다.
- 선택한 악마의 계약 즉시 효과를 처리하고, 지속 효과는 해당 스테이지 종료까지 유지한다.
- 하나의 전투 스테이지에서 계약은 원칙적으로 1회만 허용한다.
- 카드·유물·악마 효과가 추가 계약을 명시한 특별한 상황만 횟수 제한의 예외로 처리한다.
- 비용 지불 후 남은 영혼이 0이면 계약할 수 없다.
- 악마의 개별 대가는 영혼을 0까지 감소시킬 수 있으며 0이 되는 순간 즉시 전투 패배로 처리한다.
- `할 수 있다`고 적힌 계약 효과는 소유자가 실행 여부를 선택한다.
- 추가 계약에서는 같은 악마를 다시 선택할 수 있고 각 계약을 별도 인스턴스로 유지한다.
- 적의 계약에도 플레이어와 같은 비용·선택·대가·중첩·정리 규칙을 적용한다.

이 방식은 무작위성과 선택권을 함께 제공하면서 별도의 악마 상점이나 사전 덱 편집 UI 없이 구현할 수 있다.

### 7.2 계약 카드 표시 정보

- 악마 이름과 일러스트
- 계약 즉시 효과
- 지속 능력
- 대가와 발동 시점
- 남은 차례 또는 카운터
- 계약 후 플레이어 영혼

### 7.3 우선 구현 악마

| 악마 | 핵심 역할 | 위험 |
| --- | --- | --- |
| 사탄 | 스탠드 안전성과 종료 카운트다운 | 마지막 차례에 영혼 손실 가능 |
| 벨페고르 | 히트 전 덱 위 카드 확인 | 상대 스탠드 시 자신도 행동 제약 |
| 마몬 | 주사위로 합계를 조작 | 6이 나오면 즉시 버스트 |
| 레비아탄 | 숫자 추측 카드 강화 | 추측 실패 시 영혼 손실 |

악마 계약이 공개 영역에 생성한 특수 카드의 숫자는 **블랙잭 합계에 포함한다**. 사탄의 권능처럼 카드가 뒤집혀 숫자가 바뀌면 바뀐 숫자로 즉시 합계를 다시 계산한다. 숫자가 없는 계약 상태·카운터·표식은 합계에 포함하지 않는다.

### 7.4 계약 임시 카드와 소유권

- 계약이 생성·이동·추방한 카드는 현재 전투의 여러 라운드에 걸쳐 유지하지만 런 덱을 영구 변경하지 않는다.
- 사탄의 권능과 바포메트의 오망성은 전투 종료 시 모든 위치에서 제거한다.
- 파이몬이 추방한 카드는 전투 종료 시 원래 소유권과 전투 시작 덱 구성을 복구한다. 플레이어 런 덱에는 제거를 반영하지 않는다.
- 벨리알이 가져온 카드는 전투 중 계약 소유자의 카드로 취급하고 라운드 종료 시 그 소유자의 버린 카드 더미로 보낸다. 전투 종료 시 원래 소유권으로 복구한다.
- 일반 라운드 정리는 위 임시 상태를 없애지 않으며, 전투 종료 정리만 런 원상복구를 수행한다.

### 7.5 사탄 카운터 임시안

- 계약 직후 종료 카운터 6으로 시작한다.
- 양측 중 누구든 정상 차례를 시작할 때 1 감소한다.
- 추가 행동은 카운터를 감소시키지 않는다.
- 카운터가 0이 된 차례 시작 시 영혼 2를 잃는다.

원문의 `4턴` UI 예시는 실제 밸런스 값이 아닌 표시 예시로 간주한다. 6은 플레이 테스트 우선값이며 평균 라운드 길이에 따라 4~8 사이에서 조정한다.

## 8. 적 AI와 정보 설계

### 8.1 공정성 원칙

AI는 플레이어의 비공개 카드, 덱 순서, 난수 결과를 직접 열람하지 않는다. 다음 공개 정보만 사용한다.

- 공개 카드와 버린 카드
- 공개된 카드 숫자와 플레이어 덱 구성
- 플레이어 행동 이력
- 활성 카드·계약 효과
- 정보 획득 카드의 결과

디버그 빌드에서는 AI가 사용한 입력 목록을 로그로 남겨 부정 정보 접근 여부를 검증한다.

### 8.2 추론 모델

1. 시작 시 플레이어 덱에 남아 있는 각 숫자의 개수를 사전 확률로 삼는다.
2. 공개·버린 카드를 제외해 확률을 갱신한다.
3. 체인지, 히트, 특정 카드 사용 등 관찰된 행동에 적 성향별 가중치를 적용한다.
4. 가능한 비공개 카드별 최종 합계, 버스트 확률, 플레이어 승리 확률을 계산한다.
5. 적은 기대 영혼 손익이 가장 높은 행동을 선택하되 난이도별 무작위성을 적용한다.

### 8.3 행동 평가

AI 행동 후보는 히트, 스탠드, 사용 가능한 카드 효과와 계약이다. 적은 폴드를 사용하지 않는다. 각 행동의 점수는 아래 요소로 계산한다.

`예상 적 피해 - 예상 자기 피해 + 생존 보정 + 성향 보정 + 소량의 무작위값`

일반 적은 계산 깊이 1, 엘리트와 보스는 카드 효과 이후까지 포함한 계산 깊이 2를 사용한다. 프로토타입에서 몬테카를로 탐색은 사용하지 않는다.

### 8.4 적 성향

| 유형 | 영혼 | 행동 특징 |
| --- | ---: | --- |
| 겁쟁이 도박꾼 | 2 | 낮은 합계에서도 빠르게 스탠드, 안전 우선 |
| 총잡이 | 3 | 리볼버으로 비공개 숫자 추측, 튜토리얼 역할 |
| 광신도 | 3 | 계약을 적극 사용하고 자기 영혼을 위험에 노출 |
| 사기꾼 | 4 | 수정 구슬과 거짓말 탐지기로 정보 조작 |
| 집행관(엘리트) | 5 | 스탠드 취소와 강제 히트 중심 |
| 최종 보스 | 7 | 영혼 구간에 따라 성향과 행동 패턴 변화 |

### 8.5 플레이어에게 공개할 추론 정보 — 기획 확정

- 일반 난이도: 가능성이 높은 비공개 숫자 3개와 각각의 확률 표시
- 엘리트: 상위 2개 숫자와 `낮음/보통/높음` 신뢰도 표시
- 보스: 추론 방향과 신뢰도만 표시
- 적의 정확한 다음 행동은 표시하지 않으며, 강력한 고유 행동만 1차례 전에 예고한다.
- 거짓 추론을 보여 주는 보스는 능력 설명과 왜곡 상태 아이콘을 사전에 명시한다.

## 9. 콘텐츠 구성

### 9.1 적 제시

| 구간 | 제시 방식 | 목표 |
| --- | --- | --- |
| 첫 번째 전투 | 일반 적 2명 또는 일반 적 1명+엘리트 1명 | 추론 UI와 카드 사용 학습, 위험 선택 제공 |
| 두 번째 전투 | 일반 적 2명 또는 일반 적 1명+엘리트 1명 | 현재 덱·영혼에 맞는 상대 선택과 빌드 검증 |
| 최종 전투 | 보스 1명 | 전체 빌드와 영혼 관리 검증 |

두 후보는 적 이름, 등급, 최대 영혼, 핵심 성향 또는 특수 규칙과 예상 보상을 전투 전에 공개한다. 같은 후보 2명을 동시에 제시하지 않으며, 엘리트는 한 선택 화면에 최대 1명만 포함한다.

### 9.2 보스 단계안

- 영혼 7~5: 보수적 행동, 플레이어 추론 정보 정상 제공.
- 영혼 4~3: 카드 효과 사용 빈도 상승, 상위 추론 숫자만 제공.
- 영혼 2~1: 다음 강공격을 예고하고 강제 히트 또는 추측 공격 사용.

보스의 별도 승리 조건은 프로토타입에서 제외한다. 기본 규칙 이해를 흐리지 않고 행동 패턴만으로 차별화한다.

### 9.3 유물 예시

- 첫 계약의 영혼 비용 1회 면제
- 정확히 21 승리 시 영혼 1 회복
- 리볼버 사용 시 추론 확률을 상세 표시
- 덱을 다시 섞을 때 맨 위 카드 1장 확인
- 상점 휴식 회복량 +1

## 10. UI/UX 설계

### 10.1 전투 화면 정보 우선순위

1. 양측 영혼과 현재 합계
2. 양측 공개 카드와 플레이어 비공개 카드
3. 적 비공개 카드 및 AI 추론 패널
4. 덱·버린 카드 수와 사용 완료 상태
5. 활성 계약, 대가, 남은 카운터
6. 사용 가능한 행동 버튼
7. 전투 로그와 효과 처리 순서

### 10.2 화면 배치

- 고정 카메라: 플레이어가 서부 시대 술집 테이블에 앉은 시점을 유지한다.
- 상단 월드 영역: 좌측 적 버림패, 중앙 적 공개·비공개 카드, 우측 적 덱
- 중앙 월드 영역: 카드 효과와 선택 연출을 위한 공용 처리 공간
- 하단 월드 영역: 좌측 플레이어 일반 카드 덱·별도 악마 카드 덱·체인지, 중앙 비공개·공개 카드, 우측 플레이어 버림패
- 우측 가장자리: 항상 보이는 리볼버·보위 나이프·위협용 해머 장식과 효과 연출
- 우측: 계약·지속 효과 및 전투 로그
- 최하단: 히트와 스탠드 버튼. 체인지는 비공개 카드, 계약은 악마 카드 덱 오브젝트에서 진입

카드와 카드 더미는 테이블 위 월드 오브젝트로 표현한다. 상세 효과, 영혼·골드, 추론 정보와 행동 버튼은 화면 UI로 분리해 가독성을 확보한다.

### 10.3 정보 표현 규칙

- 실제 합계와 공개 합계를 구분해 플레이어에게는 둘 다, 적 영역에는 공개 합계만 표시한다.
- 버스트 가능성이 있는 히트 버튼에는 위험 아이콘을 표시하되 결과를 확정적으로 예고하지 않는다.
- 영혼을 쓰는 행동은 비용과 행동 후 잔여 영혼을 버튼에 함께 표시한다.
- 체인지에는 현재 비용을 `무료`, `영혼 1`, `영혼 2`처럼 표시하고 같은 전투의 성공 횟수에 따라 즉시 갱신한다.
- 플레이어 비공개 카드는 뒷면 위에 실제 앞면을 반투명하게 겹쳐 플레이어만 확인하게 한다.
- 사용 완료 카드는 현재 위치에 남기되 검게 처리해 숫자만 표시한다.
- 플레이어 덱 클릭 시 실제 잔여 카드 목록을, 적 덱 클릭 시 공개 정보로 추론 가능한 후보 목록만 표시한다.
- 연쇄 효과는 번호가 붙은 처리 대기열로 0.3~0.5초 간격 표시한다.
- 마지막 계약 차례에는 화면 테두리와 경고음으로 위험을 강조한다.

### 10.4 튜토리얼

팝업 설명보다 첫 적의 상황 설계를 통해 학습시킨다.

1. 첫 차례에 히트와 스탠드 설명
2. 사용 가능한 카드 강조
3. 적 비공개 카드 추론 패널 개방
4. 적이 낮은 확률 선택으로 실패하는 모습을 보여 공정성 전달
5. 첫 전투 종료 전 계약 1회 무료 제공
6. 승리 후 카드 보상 선택

튜토리얼 안내는 이미 이해한 플레이어가 즉시 건너뛸 수 있어야 한다.

## 11. 연출 및 아트 방향

- 3D 서부 시대 술집과 테이블을 사용하는 고정 카메라 기반 2.5D 화면
- 카드 앞면·뒷면은 얇은 3D 오브젝트, 덱과 버린 카드 더미는 높이가 있는 3D 오브젝트로 표현
- 남은 카드 수는 덱 높이 단계와 숫자 표시를 함께 사용
- 리볼버, 보위 나이프와 위협용 해머는 효과 발동 시 등장하는 3D 연출 오브젝트로 표현
- 카드 상세 설명과 전투 정보는 화면 UI로 표시하는 월드 오브젝트·UI 혼합 방식
- 전투, 보상과 상점은 같은 술집 테이블을 공유하며 인물·조명·음악·테이블 오브젝트를 교체해 전환
- 상점에서는 적 대신 상인이 테이블 맞은편에 등장해 카드와 서비스 상품을 펼침
- 계약 시 배경 조명, 테이블 문양, UI 테두리를 악마별 색으로 일시 변경
- 적 캐릭터의 정지 일러스트·파츠 애니메이션·3D 여부는 추후 확정
- 생성형 리소스는 콘셉트 초안으로만 사용하고 최종 빌드에서는 색감·선 굵기·광원 방향을 통일

### 11.1 필수 피드백

- 카드 획득: 짧은 뒤집기와 숫자 강조
- 카드 사용: 카드가 중앙으로 이동한 뒤 대상 방향으로 효과 전달
- 영혼 피해: 영혼 아이콘 파괴 및 잔여량 강조
- 정확히 21: 전용 금색 플래시와 2배 피해 표시
- 버스트: 합계 숫자 붉은 파열, 원인 카드 강조
- 계약: 서명/낙인 연출과 대가 재확인

| AI 비공개 카드 추론 정확도 | 난이도에 따라 35~60% |

### 12.2 우선 검증 변수

- 시작 덱 16장과 20장의 체감 차이
- 플레이어 영혼 12와 적 영혼 범위
- 플레이어 합계 비교 패배 1·버스트 2, 적 라운드 패배 1의 비대칭
- 정확히 21의 추가 피해 2가 지나치게 운 의존적인지
- 적별 골드 지급량과 상점 휴식 가격·회복량
- 사탄 카운터 6, 마몬 주사위 6 즉시 버스트의 체감
- 리볼버과 AI 추론 UI의 정확도·가독성

### 12.3 조정 원칙

- 전투가 길면 적 영혼보다 라운드 피해량을 먼저 조정한다.
- 플레이어가 계약을 쓰지 않으면 효과를 강화하기 전에 대가와 기대 이득을 더 명확히 표시한다.
- 특정 숫자 카드만 제거된다면 그 숫자의 효과 효율을 높인다.
- 운 패배가 잦으면 체인지 비용 곡선과 정보 제공량을 먼저 조정한다.

## 14. 테스트 및 완료 조건

### 14.1 규칙 테스트

- 에이스 여러 장이 있을 때 가장 큰 21 이하 합계를 선택한다.
- 양측 모두 21 초과 시 21에 더 가까운 쪽이 승리한다.
- 동점은 플레이어 승리로 처리한다.
- 최초 배분 자동 발동이 발생하지 않는다.
- 카드 효과 처리 중 버스트가 즉시 라운드를 종료한다.
- 버스트·정확히 21의 피해량과 체인지 누적 비용이 표와 일치한다.
- 덱 소진 시 버린 더미가 정확히 한 번 섞인다.
- 영혼 0에서 계약할 수 없고, 영혼 0 도달 시 런이 끝난다.

### 14.2 AI 공정성 테스트

- AI 관측 상태에 플레이어 비공개 카드 값과 덱 순서가 포함되지 않는다.
- 같은 공개 상태와 같은 난수 시드에서 같은 행동을 선택한다.
- 공개·버린 카드 변화에 따라 추론 확률 총합이 100%로 갱신된다.
- 거짓 정보를 표시하는 보스 외에는 UI 확률과 AI 내부 확률이 일치한다.

### 14.3 프로토타입 완료 정의

- 시작부터 보스 승리 또는 패배까지 중단 없이 1런 진행 가능
- 핵심 행동 5종과 악마 4종이 정상 작동
- 일반 3종, 엘리트 1종, 보스 1종의 행동 차이가 체감됨
- 저장이 필요한 런 상태가 아니라면 치명적 오류 없이 재시작 가능
- 5회 이상 완주 테스트에서 진행 불가 버그 0건
- 목표 전투 시간과 핵심 지표를 기록할 수 있는 로그 확보

| 정식 런 경로 | 전투→보상→상점/사건→전투→보상→상점/사건→보스, 지도 없음 | 짧은 런 안에서 전투와 성장 선택에 집중 |
| 일반 전투 상대 | 후보 2명 중 1명 선택, 엘리트 최대 1명 | 지도 없이도 위험·보상 선택 제공 |
| 전투 후 이벤트 | 상점 또는 무작위 사건, 첫 개발은 상점 고정 | 초기 구현 단순화 뒤 비전투 변주 확장 |
| 골드 | 모든 전투 승리에서 획득, 강한 적·엘리트일수록 증가 | 위험한 상대 선택의 추가 보상 |
| 휴식 | 독립 노드 제거, 상점에서 골드로 이용 | 회복과 경제 선택 통합 |
| 상점 판매 카드 | 일반 카드 3장·악마 카드 2장, 슬롯별 재고 1개, 구매 후 `SOLD OUT` | 두 덱의 성장 선택과 재고 변화를 한 화면에서 제공 |
| 상점 갱신 | 새로고침·재입고 없음 | 첫 구현의 경제와 화면 상태를 단순화 |
| 상점 서비스 | 라이터 카드 제거 1회, 위스키 영혼 회복 1회, 가격과 회복량은 미정 | 덱 정리와 회복을 제한된 경제 선택으로 통합 |
| 전투 보상 | 카드 3장 중 1장 선택 또는 건너뛰기 | 덱 비대화를 플레이어가 통제 |
| 엘리트 보상 | 제시되는 3장 모두 높은 등급 카드로 확정 | 엘리트 선택의 위험에 명확한 보상 제공 |
| 보스 특수 승리 조건 | 제외 | 기본 규칙 학습 방해 방지 |
| 일반 합계 승리 피해 | 적 영혼 1 | 전투 진행 규칙 보완 |
| 동시 버스트 | 효과 발생자가 패배 | 연쇄 판정 단순화 |
| 폴드 | 플레이어·적 행동에서 삭제 | 행동 과부하를 줄이고 핵심 선택에 집중 |
| 위협용 해머 대상 | 상대의 공개 카드 1장 | 자신의 카드를 비용으로 버리는 이전 세부 기획을 폐기하고 원래 공격 의도 복원 |
| 독극물 | 즉시 스탠드 또는 영혼 3개·부족 시 전부 지불, 지불 후 라운드 승리 시 영혼 5 회복 | 폴드 없이도 안전 종료와 위험한 영혼 베팅 중 선택 제공 |
| 루시퍼 | 무작위 악마 카드 5개 중 1개 선택, 영혼 1개 상실, 선택한 악마의 효과·대가 모두 적용 | 1회성 선택권의 매력을 유지하면서 최소한의 추가 대가 부여 |
| 체인지 비용 | 같은 전투에서 `0→1→2→3…`, 다음 전투에서 초기화 | 정보전을 반복해서 무효화할수록 영혼 대가 증가 |
| 체인지 최소 잔여 영혼 | 비용 지불 후 1 이상 | 후보 선택 도중 영혼 0으로 런이 종료되는 중간 상태 방지 |
| 플레이어 패배 피해 | 스탠드 비교 패배 1, 버스트 2 | 폴드 없이도 안전한 스탠드와 위험한 히트의 손실 차이를 제공 |

### 15.2 원문에 반영을 권장하는 변경

1. **일반 합계 승리의 피해를 명시한다.** 현재 피해 표에는 적 버스트와 정확히 21만 있어, 일반 비교 승리 시 전투가 진행되는지 모호하다.
2. **독극물과 루시퍼의 새 효과를 구현에 이관한다.** 독극물의 즉시 스탠드 우선순위와 루시퍼의 5개 후보 생성·중복 규칙을 확정한 뒤 카드 풀에 다시 연결한다.
3. **카드 사용 후 숫자가 합계에 남는지 명시한다.** 본 기획서는 `버린다` 문구가 없으면 남는 것으로 정했다.
4. **자동 발동 카드의 처리 후 위치와 동시 발동 순서를 공통 규칙으로 올린다.** 카드별 예외만 개별 문구에 둔다.
5. **사탄의 6카운트와 UI 예시 4턴의 관계를 정리한다.** 플레이 테스트 값과 단순 예시를 구분해야 한다.
6. **동시 버스트와 효과 연쇄 중 전투 종료 우선순위를 추가한다.** 구현과 QA에서 가장 큰 충돌 가능성이 있다.
7. **적 행동 정보의 공개 범위를 적 등급별로 확정한다.** 확률 UI가 핵심 차별점이므로 후순위로 남기지 않는 편이 좋다.

### 15.3 플레이 테스트 후 결정할 항목

- 시작 덱을 16장으로 축소할지 여부
- 바포메트의 모략 카드 분배량과 양측 덱 비율
- 모든 악마의 대가 수치와 동일 악마 중첩 시 효과·대가의 카드별 계산 방식
- 자동 발동 카드가 일부 상황에서 공개 영역에 남아야 하는지
- 보스의 거짓 추론 정보가 재미보다 불공정함을 크게 만드는지
- 상점·무작위 사건 등장 가중치와 연속 등장 제한 여부
- 첫 번째·두 번째 전투의 엘리트 등장 확률
- 높은 등급 카드의 기준과 카드 등급별 보상 등장 확률
- 적별 골드 지급량과 상점 상품·휴식 가격

## 17. 후속 문서

프로토타입 제작 전 아래 문서를 별도로 준비한다.

- 카드 데이터 시트: ID, 숫자, 발동 조건, 대상, 효과, 처리 후 위치, 강화안
- 악마 계약 데이터 시트: 즉시 효과, 지속 효과, 대가, 카운터, 중첩 규칙
- 적 AI 정책표: 성향 가중치, 행동 임계값, 계약 사용 조건
- 전투 상태 전이표: 입력, 처리 순서, 종료 조건, 예외
- 플레이 테스트 기록지: 전투 시간, 계약 횟수, 피해 원인, 승패, 혼동 지점

이 문서는 `rule.md`의 규칙을 대체하지 않는다. 구현 중 충돌이 생기면 본 문서의 임시 결정안을 우선 적용하고, 플레이 테스트 결과를 근거로 두 문서를 함께 갱신한다.

## 1. 문서 목적

이 문서는 일반 적 3종, 엘리트 1종과 최종 보스 1종의 전투 프로필, 행동 성향, 카드 효과 사용 원칙과 실제 전투 생성 경계를 구현 가능한 수준으로 확정한다.

적 전투 프로필은 상대 후보를 만들고 선택하는 시스템과 분리한다. 상대 선택 시스템은 플레이어가 고른 프로필 키만 전달하고, 이 문서의 시스템이 해당 키를 실제 적 영혼·덱·행동 정책·보상 등급으로 변환한다.

기획·전투 구현·검증의 최종 책임자는 이천서다.

- 보스 전용 별도 승리 조건
- 정식 밸런스 수치와 난이도별 가중치 확정

## 5. 공통 AI 공정성 규칙

적 AI는 플레이어가 알 수 없는 정보를 직접 읽지 않는다.

### 5.1 허용 입력

- 적 자신의 손패와 사용 가능한 카드
- 플레이어와 적의 공개 카드
- 공개·버린 카드 기록
- 플레이어가 이번 전투에서 수행한 공개 행동
- 양측 현재·최대 영혼
- 현재 라운드와 공개된 지속 효과
- 규칙에 의해 얻은 추론 결과
- 테스트가 주입한 결정적 난수

### 5.2 금지 입력

- 플레이어 비공개 카드의 실제 숫자·정의·물리 ID
- 플레이어 또는 적 덱의 실제 다음 카드와 전체 순서
- 아직 발생하지 않은 난수 결과
- 화면에 공개되지 않은 상대 선택 후보 내부 정보
- 테스트 편의를 위해 만든 정답 값

정책 API에는 금지 정보가 포함된 전투 객체 전체를 전달하지 않는다. 필요한 공개 정보만 복사한 관측 모델을 제공한다.

## 6. 공통 행동 결정 원칙

적은 현재 규칙에서 가능한 행동만 후보로 만든 다음 점수를 비교한다.

`예상 상대 영혼 피해 - 예상 자기 영혼 피해 + 생존 보정 + 적 성향 보정 + 결정적 변동값`

- 일반 적은 현재 행동의 직접 결과까지만 평가한다.
- 엘리트와 보스는 카드 효과 뒤의 히트·스탠드 가능 상태까지 한 단계 더 평가할 수 있다.
- 프로토타입에서는 완전 탐색과 몬테카를로 시뮬레이션을 사용하지 않는다.
- 같은 관측 상태와 같은 시드는 같은 행동을 선택해야 한다.
- 동점 우선순위는 정책에 명시하고 테스트로 고정한다.
- 실행할 수 없는 카드·계약을 점수 후보에 넣지 않는다.
- 정책 결정은 실행 직전에 다시 확인한다. 무효 결정은 전투를 변경하지 않고 한 번만 재평가하며, 이후에는 유효한 안전 후보를 결정적으로 선택한다.
- 적 폴드는 플레이어 폴드와 대칭으로 자기 영혼 1을 소모하고 즉시 라운드를 끝낸다. 원문에 적 비용이 별도로 없으므로 첫 밸런스 테스트 뒤 재검토하는 임시안이다.

## 7. 적별 행동 기획

### 7.1 총잡이

- 첫 일반 전투에서 적 AI와 추론 시스템을 학습시키는 기준 적이다.
- 자동 권총을 사용할 수 있으면 공개 카드·버린 카드 분포로 플레이어 비공개 숫자를 추정한다.
- 추정 확률이 낮으면 무리한 추측보다 기본 히트·스탠드를 우선한다.
- 플레이어에게 가능성이 높은 숫자 3개와 각 확률을 표시할 수 있다.
- 실제 비공개 숫자는 정책 입력과 공개 결과에 포함하지 않는다.
- 공개되지 않은 후보군에서 가장 높은 확률이 50% 이상일 때만 자동 권총을 사용한다. 동률이면 낮은 숫자를 우선하는 프로토타입 임시안이다.

### 7.2 광신도

- 낮은 생존 보정보다 공격 기대값을 크게 평가한다.
- 남은 영혼이 있어도 위험한 히트나 비용 행동을 비교 후보에 적극 포함한다.
- 악마 계약 시스템이 구현되기 전에는 가짜 계약 결과를 만들지 않는다.
- EP 1차 범위에서는 기존 행동과 구현된 카드 효과 안에서 공격적 가중치만 제공한다.
- 계약 API가 완성되면 같은 정책 경계에 계약 후보를 추가한다.
- 프로토타입에서는 합계 18까지 히트를 우선하고 19부터 스탠드를 우선한다.

### 7.3 사기꾼

- 직접 피해보다 덱·정보 우위를 만드는 행동을 선호한다.
- 수정 구슬이 사용 가능하면 다음 행동의 선택지를 개선하는 정도를 평가한다.
- 거짓말 탐지기는 현재 카드 카탈로그와 처리기에 없으므로 EP 1차 범위에서 제외한다.
- 상대 비공개 값을 실제로 읽는 방식으로 속임수를 구현하지 않는다.
- 정보 조작 결과는 플레이어가 규칙적으로 확인할 수 있는 상태와 로그를 남긴다.
- 수정 구슬이 공개한 두 장 중 버스트하지 않으면서 합계가 가장 높은 카드를 가져오고, 둘 다 위험하면 가져오지 않는다.

### 7.4 집행관

- 엘리트 적이며 한 상대 선택 화면에 최대 한 명만 등장한다.
- 위협용 해머와 군용 나이프를 중심으로 플레이어의 안전 행동을 압박한다.
- 카드 사용 뒤 강제 히트 또는 스탠드 방해가 가능한 경우 그 후속 결과까지 평가한다.
- 플레이어가 스탠드한 경우 해머로 스탠드를 해제하고, 비용 선택에서는 적이 실제 보유한 공개 카드 중 숫자가 가장 낮은 카드를 우선한다.
- 군용 나이프는 공개 손패와 공개 구성 추론으로 비공개 카드·강제 드로우 뒤 버스트 가능성을 한 단계 더 추정해 점수에 반영한다.
- 플레이어에게 상위 추론 숫자 2개와 낮음·보통·높음 신뢰도만 표시한다.
- 신뢰도는 최상위 추론 확률 35% 미만 `낮음`, 35~59% `보통`, 60% 이상 `높음`을 프로토타입 값으로 사용한다.
- 승리 보상은 기존 `HighGrade` 보상 경계로 전달한다.

### 7.5 최종 보스

- 최대 영혼은 프로토타입 값 7을 사용한다.
- 영혼 7~5에서는 생존을 우선하고 정상 추론 정보를 제공한다.
- 영혼 4~3에서는 구현된 카드 효과 사용 가중치를 높이고 정보 공개량을 줄인다.
- 영혼 2~1에서는 강한 행동을 한 차례 전에 예고하고 추측 공격 또는 강제 히트를 우선한다.
- 별도 승리 조건은 만들지 않고 기존 영혼 0 전투 종료 규칙을 사용한다.
- 승리 뒤에는 기존 높은 등급 보상을 선택하거나 건너뛴 다음 `RunVictory`가 된다.

## 8. 적 카드와 덱 원칙

- 프로필은 카드 정의 키 목록과 결정적 셔플 시드를 가진다.
- 물리 카드 ID는 전투 덱을 만들 때 충돌 없이 발급한다.
- 카드 효과는 기존 공통 효과 처리 명령을 재사용하되 행위자와 대상이 올바른지 검증한다.
- 플레이어용 입력 API를 적이 우회 호출하지 않는다.
- 적 카드 사용은 적 차례의 승인된 행동 하나로 처리한다.
- 선택 대기 효과가 있다면 적 정책이 유효한 옵션을 결정하고 같은 효과 상태 기계를 완료한다.
- 구현되지 않은 카드나 처리기는 프로필 덱에 넣지 않는다.

## 9. 상대 선택 시스템과의 계약

상대 선택 담당은 프로필의 내부 전투 객체를 직접 만들지 않는다.

```text
상대 선택 후보
  → BattleProfileKey 선택
  → 적 프로필 카탈로그 조회
  → 적 영혼·덱·행동 정책 구성
  → CoreLoopBattle 생성
  → 전투 결과
  → Normal 또는 HighGrade 보상
```

상대 선택 화면에 필요한 이름, 등급, 최대 영혼, 핵심 성향과 예상 보상은 전투 프로필에서 만든 읽기 전용 미리보기로 제공한다. 표시 정보와 실제 전투 설정을 별도 장소에 중복 저장하지 않는다.

## 10. 실패·안전 규칙

- 알 수 없는 프로필 키면 전투를 생성하지 않는다.
- 프로필의 카드 정의 키가 카탈로그에 없으면 시작 전에 실패한다.
- 등급·최대 영혼·정책·덱이 유효하지 않으면 부분 전투를 만들지 않는다.
- AI가 선택한 행동이 실행 시점에 무효면 임의 행동으로 강행하지 않고 안전한 재평가를 한 번 수행한다.
- 재평가 뒤에도 후보가 없으면 규칙상 가능한 기본 스탠드를 사용한다.
- 정책 예외는 전투 상태와 카드 소유권을 부분 변경하지 않아야 한다.
- 최종 보스가 아닌 적이 `Boss` 등급 또는 보스 완료 목적지를 요청하면 거절한다.

## 11. 플레이어 표시 정보

상대 선택 담당이 표시할 수 있는 프로필 미리보기는 다음으로 제한한다.

- 이름
- 일반·엘리트·보스 등급
- 최대 영혼
- 핵심 행동 성향 또는 특수 규칙 요약
- 예상 보상 등급

전투 중 추론 UI는 난이도에 따라 다음을 제공한다.

- 일반: 상위 추론 숫자 3개와 확률
- 엘리트: 상위 숫자 2개와 신뢰도 구간
- 보스: 추론 방향과 신뢰도, 예고된 강행동

적의 정확한 다음 행동과 비공개 카드 값은 표시하지 않는다.

## 12. 완료 조건

- 다섯 프로필 키와 등급·최대 영혼·덱·정책이 검증된다.
- 기존 `SimpleEnemyPolicy` 기반 전투가 회귀 없이 유지된다.
- 적 정책 입력에 플레이어 비공개 값과 덱 순서가 없다.
- 일반 적 3종이 같은 공개 상태에서 서로 다른 행동 성향을 보인다.
- 집행관은 엘리트 정책과 높은 등급 보상으로 연결된다.
- 보스는 영혼 3구간 정책과 높은 등급 보상 뒤 런 승리 흐름을 사용한다.
- 선택된 프로필 키로 실제 전투가 생성된다.
- 적 카드 효과의 행위자·대상·카드 소유권이 테스트로 검증된다.
- 전체 EditMode, 실제 전투, 씬과 Console 검증이 통과한다.
- AI 활용과 이천서 실제 담당 기록이 단계마다 갱신된다.

> 최종 갱신: 2026-07-22

## 1. 계획 목적

악마 계약 시스템을 데이터·덱, 계약 행동, 첫 수직 악마, 나머지 효과, 화면·런, 적 AI와 최종 검증으로 나눠 구현한다. 각 단계는 문서 확인 → 회귀 테스트 또는 새 테스트 작성 → 최소 구현 → 대상·전체 검증 → 기록 → 커밋 순서로 독립 완료한다.

한 단계의 완료 증거가 없으면 다음 단계의 코드를 미리 합치지 않는다. 사탄은 규칙 충돌이 있으므로 결정 게이트를 통과하기 전 구현하지 않는다.

## 2. 실행 원칙

1. 단계별로 작은 커밋을 만든다.
2. 행동 변경 전 기존 회귀 테스트로 현재 규칙을 고정한다.
3. 순수 C# 규칙과 테스트를 UI보다 먼저 구현한다.
4. 조회·표시 경계는 상태를 변경하지 않는다.
5. 난수와 셔플은 결정적으로 주입해 재현한다.
6. 일반 카드 효과를 복제하지 않고 필요한 계약 훅만 연결한다.
7. 단계 완료 뒤 AI 활용·팀 역할·진행 기록을 함께 갱신한다.
8. AI는 사용자의 명시적 요청 없이 스테이징·커밋하지 않는다.

- 사탄·레비아탄 등 충돌하거나 사람이 확인할 항목을 결정 대장으로 분리한다.

- README, AI 활용과 이천서 역할 기록을 갱신한다.

- 이천서를 기획·개발 책임자로, AI를 구조 대조·초안 보조로 기록했다.

## 5. DC-01 — 계약 데이터와 런·전투 덱 기반

### 목표

화면과 효과 없이도 네 악마를 안정된 키와 물리 카드로 보존하고, 런 덱에서 전투마다 독립적인 계약 덱을 만들 수 있게 한다.

### 작업

- `DemonContractDefinition`, 종류와 카탈로그
- 사탄·벨페고르·마몬·레비아탄 정의
- `RunDemonDefinition`과 플레이어 최초·현재 악마 덱
- 전투용 `DemonContractCard`, 드로우·버림 덱
- 후보 3장 확보와 버린 더미 보충 명령
- 일반 카드와 분리된 `StageBattleFactory` 변환
- 셔플 시드 독립과 전투·재시작 상태 격리
- 정의·덱·카드 총수·변환 단위 테스트

### 완료 게이트

- 네 정의 키, 표시명과 기본 비용이 카탈로그에서 조회된다.
- 알 수 없는 키와 중복 물리 ID를 거절한다.
- 후보 이동 전후 카드 총수와 ID가 보존된다.
- 일반 카드 덱 순서는 악마 덱 셔플 때문에 바뀌지 않는다.
- 전투마다 새 악마 덱을 사용하고 런 재시작 시 초기 4장이 복구된다.
- 대상 테스트와 전체 EditMode가 통과한다.

### 권장 커밋 제목

`악마 카드가 일반 덱을 침범하지 않고 런과 전투 사이에서 보존되게 하다`

### 완료 기록

- 네 계약 정의와 카탈로그, 런 물리 카드, 전투 카드·드로우·버림 덱을 구현했다.
- 기본 `PlayerRunState`는 사탄·벨페고르·마몬·레비아탄 각 1장의 최초/현재 덱을 분리해 보존한다.
- `StageBattleFactory`는 일반 카드와 별도로 악마 덱을 변환하고, 전투마다 새 덱과 독립 시드를 사용한다.
- 후보 3장, 부족 시 무변경 거절, 버린 더미 보충, 활성 카드 제외와 물리 ID 총수 보존을 테스트했다.
- DC-01 대상 19/19, 전체 EditMode 329/329와 컴파일 오류 0을 확인했다. 테스트 뒤 Console에는 Unity Test Framework 기반 시설 경고 4건과 결과 저장 `Exception` 2건만 남았으며 게임 코드 오류와 구분했다.
- 계약 비용·횟수·선택 상태·개별 효과·UI는 계획대로 DC-02 이후에 남겼다.

## 6. DC-02 — 계약 행동과 후보 선택

### 목표

영혼 비용과 전투당 제한을 지키며 후보 3장 중 하나를 원자적으로 활성화하는 공통 계약 행동을 완성한다.

### 작업

- 계약 가능 여부·불가 사유·계약 후 영혼 모델
- `PlayerResolvingDemonContract` 상태
- 보류 상호작용·옵션·증가형 `InteractionId`
- `TryBeginPlayerDemonContract`
- `TryResolvePlayerDemonContract`
- 비용 1, `현재 영혼 > 비용`, 전투당 1회
- 선택 1장 활성, 미선택 2장 버림
- 계약 결과와 차례 소비
- 개별 대가 영혼 0 즉시 패배와 후속 처리 중단
- 동일 악마 추가 계약을 위한 물리 인스턴스 분리
- CoreLoopSession·StageProgressionSession 전달
- 정상·거절·오래된 입력·중복 입력 테스트

### 완료 게이트

- 영혼 1에서는 무변경 거절하고 영혼 2에서는 1을 지불해 후보를 연다.
- 후보 공개 뒤 취소, 히트·스탠드·체인지·카드·추가 계약을 거절한다.
- 잘못된 상호작용 ID와 옵션은 카드 위치·영혼·차례를 바꾸지 않는다.
- 정확히 한 장만 활성화되고 두 장은 버린다.
- 두 번째 기본 계약을 거절한다.
- 효과가 없는 테스트 처리기로 행동이 적 차례까지 정확히 한 번 이어진다.
- 같은 악마 계약 두 개가 서로 다른 ID·런타임 상태를 유지한다.
- 개별 대가로 영혼이 0이 되면 버스트 처리 없이 즉시 전투가 끝난다.
- 대상·전체 EditMode가 통과한다.

### 권장 커밋 제목

`영혼을 건 계약 선택이 취소 악용과 중복 입력 없이 한 번만 성립하게 하다`

### 완료 기록

- `DemonContractAvailability`가 비용 1, 계약 후 영혼, 남은 기본 횟수와 상태별 불가 사유를 무변경 조회한다.
- `PlayerResolvingDemonContract`, 세 옵션의 `PendingDemonContractInteraction`과 증가형 `InteractionId`를 추가했다.
- 시작 전 후보 수를 확인하고, 승인 시 영혼 1·기본 횟수 1을 적용한 뒤 후보 3장을 보류 위치로 이동한다.
- 올바른 ID와 옵션만 선택 카드 하나를 `ActiveDemonContract`로 등록하고 나머지 둘을 버리며, 중복·오래된 입력은 무변경 거절한다.
- `DemonContractResolver`는 기본 악마 효과를 등록하지 않은 채 주입식 활성화 경계만 제공한다. 이를 통해 개별 대가 영혼 0이 버스트 없이 즉시 패배하고 적 AI가 실행되지 않는 경로를 검증했다.
- `CoreLoopSession`과 `StageProgressionSession`이 두 계약 입력을 전달하고, 계약 대가 패배의 지속 영혼·런 패배를 한 번만 동기화한다.
- DC-02 대상 13/13과 전체 EditMode 342/342, 컴파일 오류 0을 확인했다. UI·Game View는 DC-05 범위이며 씬·프리팹·패키지·외부 에셋을 변경하지 않았다.

## 7. DC-03 — 벨페고르 수직 기능

### 목표

계약 공통 기반이 실제 지속 효과, 전용 선택, 덱 이동과 차례 제약까지 연결되는 첫 세로 기능을 완성한다.

### 작업

- 벨페고르 처리기와 활성 런타임 상태
- 히트 입력 전 덱 위 카드 미리보기
- 그대로 공개 히트 / 덱 아래 이동 후 차례 넘김
- 상대 스탠드 뒤 다음 플레이어 차례 종료 자동 스탠드
- 정보 은닉 표시 모델
- 카드 부족·버스트·전투 종료 중 후속 처리 차단
- 독립·런 전투 통합 테스트

### 완료 게이트

- 미리보기 카드와 실제 히트·덱 아래 이동 카드가 같은 물리 ID다.
- 미리보기 값이 적 관측과 공용 로그에 노출되지 않는다.
- 히트 외 행동은 미리보기를 발동시키지 않는다.
- 자동 스탠드가 한 번만 적용되고 전투 종료 뒤 남지 않는다.
- 공개 합 중간 버스트와 전체 합 최종 승부 규칙이 유지된다.
- 대상·전체 EditMode가 통과한다.

### 권장 커밋 제목

`첫 계약이 덱 정보와 차례 제약을 숨은 정보 유출 없이 바꾸게 하다`

### 완료 기록

- 기본 Resolver에 확정된 벨페고르 처리기만 등록하고 물리 계약 인스턴스별 `BelphegorRuntimeState`를 생성했다.
- 히트 전에 실제 덱 위 카드를 이동 없이 확인하고, 공개 옵션과 분리된 `PlayerDemonContractPreview`에만 카드 정보를 보존한다.
- 그대로 히트와 덱 아래 이동은 미리 본 카드 ID를 다시 확인한다. 히트는 기존 공개 드로우·공개 합 버스트를 재사용하고 이동은 드로우 없이 차례를 종료한다.
- 상대 스탠드가 확인된 플레이어 정상 차례에 자동 스탠드를 예약하고, 카드·체인지·계약·벨페고르 선택을 포함한 행동 완료 뒤 한 번만 소비한다. 라운드 종료와 전투 종료에서는 예약을 초기화한다.
- 미리보기 전후 적 관측·공용 로그 불변, 카드 사용·체인지 비발동, 독립·런 전투를 DC-03 대상 9/9로 확인했다.
- Unity 6000.3.10f1에서 전체 EditMode 351/351과 컴파일 오류 0을 확인했다. `GameScene`·다른 UI·씬·프리팹·패키지·외부 에셋은 변경하지 않았다.

## 8. DC-04 — 마몬과 레비아탄

### 목표

난수·최종 합계 선택과 기존 카드 효과 후속 훅이라는 서로 다른 계약 확장 경계를 검증한다.

### 작업

- 주입 가능한 6면체 난수와 마몬 런타임 상태
- 계약 즉시 굴림, 차례 시작 재굴림, 6 즉시 버스트
- 최종 비교 직전 주사위 적용·미적용 선택
- 레비아탄의 리볼버 완료 후 추가 판정
- 추가 성공과 최종 실패 영혼 1 대가
- `ContractEffectBust` 종료 원인과 안전한 결과 모델
- 숨은 전체 합·주사위 미래값 미노출 테스트

### 완료 게이트

- 고정 난수 1~6이 모두 재현된다.
- 마몬 재굴림은 차례당 한 번이며 6은 즉시 라운드를 끝낸다.
- 마몬 값을 더하지 않은 경로와 더해 승리·버스트하는 경로가 구분된다.
- 리볼버 성공 시 레비아탄 대가가 없고, 추가 성공 시 상대만 버스트한다.
- 완전 실패 시 영혼 1을 정확히 한 번 잃고 고갈 시 전투가 끝난다.
- 상대 숨은 합이 결과·UI·AI 관측에 노출되지 않는다.
- 대상·전체 EditMode가 통과한다.

### 권장 커밋 제목

`계약의 난수와 카드 연계 위험이 같은 종료 규칙 아래 판정되게 하다`

### 완료 기록

- `IDemonDieRoller`와 물리 계약별 `MammonRuntimeState`를 추가해 계약 즉시 굴림, 정상 차례 전 유지·재굴림, 새 6의 계약 버스트를 재현 가능하게 만들었다.
- 양쪽 스탠드 뒤 `MammonApplyDie`를 보류하고, 적용하지 않음·적용 승리·적용 숫자 버스트를 기존 라운드 해석과 구분했다. 주사위는 중간 공개 합과 카드 객체를 바꾸지 않는다.
- `ContractEffectBust`를 숫자·카드 버스트와 분리하고 최종 보정 합이 에이스 가치까지 함께 재계산되도록 했다.
- 레비아탄은 실패한 리볼버 완료 결과만 후처리한다. 원 효과 성공은 그대로 종료하고, 내부 전체 합 22 이상은 상대 계약 버스트, 그 외는 플레이어 영혼 1 대가와 영혼 0 즉시 패배로 연결했다.
- 안전한 결과 모델은 발동 여부·버스트 대상·지불 영혼만 공개하고 상대 전체 합과 비공개 숫자를 노출하지 않는다.
- DC-04 대상 11/11과 Unity 전체 EditMode 362/362, 최종 컴파일 오류 0을 확인했다. 현재 Codex 세션의 Unity MCP 도구가 노출되지 않아 열린 Editor의 공식 Test Runner API를 사용했으며 씬·UI·프리팹·패키지·외부 에셋은 변경하지 않았다.

## 9. DC-05 — 플레이어 계약 화면과 런 연결

### 목표

플레이어가 실제 전투 화면에서 비용을 확인하고 계약·후속 선택을 완료하며 결과를 런까지 이어갈 수 있게 한다.

### 작업

- 계약 가능·불가 사유와 계약 후 영혼 표시
- 비용 지불 전 확인과 취소
- 후보 3장, 능력·대가·선택 입력
- 활성 계약, 주사위·카운터 자리와 최근 결과 표시
- 계약 선택 중 다른 입력 잠금
- Controller의 독립·런 세션 전달
- 계약 효과 승패·영혼·보상 동기화
- 사탄 권능·오망성 제거와 파이몬 추방·벨리알 소유권 원상복구 표시 결과
- GameScene 스크립트 표시 연결
- 두 해상도·두 씬·Console 검증

### 완료 게이트

- 비용 지불 전에는 취소할 수 있고 지불 뒤에는 후보 하나를 반드시 선택한다.
- 후보 3장의 능력과 대가가 잘리지 않고 구분된다.
- 선택 중 중복 클릭과 일반 행동이 실행되지 않는다.
- 독립·런 전투 결과와 지속 영혼이 일치한다.
- 양측 비공개 카드 최좌측 표시와 상대 정보 은닉이 유지된다.
- 계약 임시 카드는 라운드 전환 뒤 유지되고 전투 종료 뒤 런 덱에 남지 않는다.
- `GameScene.unity` 직렬화 변경 없이 우선 연결하며 필요 시 별도 시각 작업으로 기록한다.
- 대상·전체 EditMode, 1280×720·1920×1080, 두 씬과 Console이 통과한다.

### 권장 커밋 제목

`계약의 이득과 대가를 확인한 선택이 실제 런 결과까지 이어지게 하다`

## 10. DC-06 — 사탄 결정과 구현

### 착수 게이트

이천서가 DC-D05·D06을 먼저 확정한다.

- 카운터 증가/감소 방식과 4/6 값
- 0 또는 6 도달 뒤 계약 종료/반복 여부
- 화염·괴력이 하나의 물리 양면 카드인지 여부
- 뒤집힌 면의 사용 완료·사용 가능 상태 전이

결정은 기획서, 개발 명세와 진행 기록에 먼저 반영한다.

### 작업

- 스탠드 불가와 표시 사유
- 숫자·카드·계약 버스트 방지
- 정상 차례 전용 카운터 감소/증가
- 영혼 2 대가와 전투 종료
- `사탄의 권능` 물리 카드·양면 효과·사용 상태
- 권능 숫자 변경 직후 합계 재계산과 전투 종료 제거
- 추가 행동·카드 효과·양측 스탠드 불가 교착 테스트

### 완료 게이트

- 확정된 카운터 문구와 코드·UI 값이 일치한다.
- 모든 버스트 원인에서 방지 여부와 카드 이동이 정의대로 처리된다.
- 영혼 대가가 정확한 시점에 한 번 적용된다.
- 사탄 때문에 라운드가 영구 교착되지 않는다.
- 권능 카드가 복제·유실되지 않고 계약 종료 규칙을 따른다.
- 권능 카드가 라운드 사이에는 유지되고 전투 종료 뒤 런 덱에는 남지 않는다.
- 대상·전체 EditMode와 실제 전투 화면이 통과한다.

### 권장 커밋 제목

`사탄의 무적과 대가가 교착 없이 끝나는 하나의 규칙으로 작동하게 하다`

## 11. DC-07 — 적 AI 계약과 최종 검증

### 목표

광신도부터 공개 정보만으로 계약을 평가·선택하게 하고 플레이어 계약 전체를 반복 검증해 1차 작업을 마감한다.

### 작업

- 적 유효 행동 후보의 계약 종류
- 공개 정보 전용 계약 평가 입력
- 광신도 계약 가중치와 실행 직전 재검증
- 적 후보 선택·후속 선택 정책
- 네 악마 정상·거절·위험 경로 반복 테스트
- 동일 악마 중복 계약의 인스턴스·대가 격리 테스트
- 사탄 권능·오망성·파이몬 추방·벨리알 탈취의 전투 종료 원상복구 테스트
- 독립·런·상대 선택·보상·재시작 통합 회귀
- 카드 총수·상태 격리·난수 재현 50회
- 두 씬·두 해상도·Console 최종 검증
- 진행·AI 활용·팀 역할·프로젝트 구조 문서 마감

### 완료 게이트

- 적 AI가 플레이어 비공개 값, 덱 순서와 미래 난수를 읽지 않는다.
- 영혼·횟수·후보 조건이 플레이어와 같은 방식으로 재검증된다.
- 광신도가 계약을 사용할 수 있지만 무조건 자멸하지 않는다.
- 네 악마의 정상·위험·종료·재시작 경로가 각 10회 통과한다.
- 전체 EditMode, 실제 흐름, 화면과 Console에 알려진 오류가 없다.
- 이천서의 실제 구현 파일과 검증만 완료 기여로 기록된다.

### 권장 커밋 제목

`계약이 플레이어와 적의 공정한 선택으로 반복 전투에서도 안정되게 하다`

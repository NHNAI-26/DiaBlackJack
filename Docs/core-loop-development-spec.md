# 데블랙잭 코어 루프 개발 명세서

> 프로젝트: DiaBlackJack  
> 개발 문서 책임자: 이천서  
> 버전: v0.1  
> 상태: 구현 전 설계  
> 대상 환경: Unity 6000.3.10f1 / C#  
> 최종 갱신: 2026-07-19

## 1. 개발 목표

`CoreLoopTest` 씬에서 한 전투의 시작, 라운드 반복, 히트·스탠드, 결과 판정, 영혼 피해, 전투 종료와 재시작을 검증 가능한 형태로 연결한다.

이번 단계는 실제 코드 구현을 포함하지 않는다. 아래 내용은 다음 구현 작업의 기준이다.

## 2. 설계 원칙

- 규칙 계산은 화면과 입력 코드에서 분리한다.
- 한 시점의 진행 상태는 하나만 존재하도록 명시적인 상태 전이를 사용한다.
- 영혼 피해, 카드 이동과 합계 계산은 각자 한 곳에서만 처리한다.
- 무작위 드로우는 저장소의 `DeterministicRng`처럼 시드를 지정할 수 있는 방식을 우선 검토한다.
- 카드 특수 효과와 계약을 미리 구현하지 않는다.
- Unity 오브젝트는 표시와 입력 연결을 담당하고, 규칙은 일반 C# 단위 테스트가 가능하게 유지한다.

## 3. 시스템 경계

### 3.1 코어 루프가 담당하는 것

- 기본 전투 상태 생성
- 라운드 시작과 종료
- 덱·손패·버린 카드 이동
- 합계와 에이스 계산
- 플레이어 행동 수신
- 단순 적 행동 결정
- 버스트·합계 비교와 피해 적용
- 승리·패배 판정
- UI에 읽기 전용 상태 제공

### 3.2 담당하지 않는 것

- 카드 데이터 에셋과 특수 효과 실행
- 계약·보상·런 지도
- 정식 AI 추론
- 저장 시스템 연결
- 연출 순서 제어와 사운드

## 4. 권장 구성 요소

명칭은 구현 중 프로젝트 규칙에 맞게 조정할 수 있으나 책임은 유지한다.

| 구성 요소 | 책임 |
| --- | --- |
| `CoreLoopController` | 전투 생성, 전투 종료, 재시작과 최상위 상태 전이 |
| `BattleController` | 양측 영혼, 라운드 반복, 승리·패배 판정 |
| `RoundController` | 배분, 차례, 히트·스탠드, 라운드 결과 |
| `Deck` | 뽑을 카드와 버린 카드 관리, 재섞기 |
| `Hand` | 현재 카드 보유와 공개 상태 |
| `HandValueCalculator` | 에이스를 포함한 최적 합계와 버스트 판정 |
| `RoundResolver` | 버스트 또는 양측 스탠드 결과와 피해 값 계산 |
| `SimpleEnemyPolicy` | 합계 기준 히트·스탠드 반환 |
| `CoreLoopView` | 상태 표시와 버튼 활성화 |
| `CoreLoopInput` | 히트·스탠드·재시작 입력 전달 |

컨트롤러가 UI 텍스트를 직접 수정하거나, View가 영혼과 카드 상태를 직접 변경하지 않도록 한다.

## 5. 상태 모델

### 5.1 최상위 상태

| 상태 | 진입 작업 | 허용 입력 | 다음 상태 |
| --- | --- | --- | --- |
| `Initializing` | 덱, 영혼, UI 초기화 | 없음 | `StartingRound` |
| `StartingRound` | 양측 카드 2장 배분 | 없음 | `PlayerTurn` 또는 `ResolvingRound` |
| `PlayerTurn` | 플레이어 행동 대기 | 히트, 스탠드 | `PlayerTurn`, `EnemyTurn`, `ResolvingRound` |
| `EnemyTurn` | 단순 AI 행동 실행 | 없음 | `EnemyTurn`, `PlayerTurn`, `ResolvingRound` |
| `ResolvingRound` | 승패·피해·카드 정리 | 없음 | `StartingRound` 또는 `BattleEnded` |
| `BattleEnded` | 승리·패배 표시 | 재시작 | `Initializing` |

### 5.2 전이 규칙

- 카드 드로우 직후 항상 버스트를 확인한다.
- 플레이어가 스탠드하면 이후 플레이어 차례를 다시 열지 않는다.
- 한쪽만 스탠드한 경우 반대쪽이 버스트하거나 스탠드할 때까지 행동한다.
- 양쪽이 스탠드하면 한 번만 결과를 계산한다.
- `ResolvingRound` 중에는 모든 전투 입력을 막는다.
- 피해 적용 후 영혼 0 여부를 확인한 뒤 카드 정리를 실행한다.
- 재시작은 기존 상태를 재사용하지 않고 새로운 전투 상태를 만든다.

## 6. 최소 데이터 정의

### 6.1 카드

| 필드 | 형식 | 설명 |
| --- | --- | --- |
| `Id` | 정수 또는 문자열 | 동일 숫자 카드도 구분 가능한 식별자 |
| `Rank` | 1~10 정수 | 합계 계산에 사용하는 숫자 |
| `IsFaceUp` | 불리언 | 공개 여부 |

코어 루프에서는 카드 이름, 효과, 등급, 강화 정보가 필요하지 않다.

### 6.2 전투 참가자

| 필드 | 설명 |
| --- | --- |
| 현재 영혼 | 0 이상 정수 |
| 최대 영혼 | 플레이어 12, 적 3 |
| 덱 | 현재 뽑을 카드 |
| 버린 카드 | 라운드 종료 카드 |
| 손패 | 현재 라운드 카드 |
| 스탠드 여부 | 이번 라운드 행동 종료 상태 |

### 6.3 라운드 결과

| 값 | 설명 |
| --- | --- |
| `PlayerBust` | 플레이어 버스트, 플레이어 피해 2 |
| `EnemyBust` | 적 버스트, 적 피해 1 |
| `PlayerWin` | 합계 비교 또는 동점 승리, 적 피해 1 |
| `PlayerBlackjackWin` | 정확히 21 승리, 적 피해 2 |
| `EnemyWin` | 합계 비교 적 승리, 플레이어 피해 1 |

피해량은 결과 객체에 포함해 UI가 규칙을 다시 계산하지 않게 한다.

## 7. 핵심 처리 순서

### 7.1 전투 시작

1. 기존 전투 상태와 이벤트 구독을 정리한다.
2. 양측 기본 덱을 생성하고 시드로 섞는다.
3. 영혼과 상태를 초기화한다.
4. 새 라운드를 시작한다.

### 7.2 라운드 시작

1. 양측 손패와 스탠드 상태가 비어 있는지 확인한다.
2. 플레이어, 적 순서로 각 2장을 뽑는다.
3. 각자 1장은 공개, 1장은 비공개로 지정한다. 플레이어는 자기 비공개 카드를 확인할 수 있다.
4. 합계를 계산하고 예외 상태를 확인한다.
5. 플레이어 입력을 연다.

### 7.3 히트

1. 현재 차례와 입력 가능 여부를 검증한다.
2. 덱에서 카드 1장을 손패에 추가한다.
3. 덱이 비었다면 버린 카드 더미를 재섞는다.
4. 새 합계를 계산한다.
5. 버스트면 라운드 판정으로 이동한다.
6. 버스트가 아니면 현재 정책에 맞는 다음 차례로 이동한다.

### 7.4 스탠드

1. 현재 참가자의 스탠드 상태를 설정한다.
2. 양쪽 모두 스탠드면 라운드 판정으로 이동한다.
3. 아니면 스탠드하지 않은 참가자의 차례로 이동한다.

### 7.5 라운드 판정

1. 결과를 한 번 계산한다.
2. 결과에 따른 영혼 피해를 적용한다.
3. UI에 결과와 피해를 알린다.
4. 손패 전체를 버린 카드 더미로 이동한다.
5. 영혼이 0이면 전투 종료, 아니면 새 라운드를 시작한다.

## 8. 화면 연결 계약

View는 아래 정보만 읽고 표시한다.

- 현재 상태와 차례
- 양측 영혼과 최대 영혼
- 표시 가능한 카드 목록
- 표시용 합계
- 덱과 버린 카드 수
- 최근 라운드 결과와 피해
- 히트·스탠드·재시작 가능 여부

버튼 활성화 기준은 상태에서 계산한다.

| 버튼 | 활성 조건 |
| --- | --- |
| 히트 | `PlayerTurn`이고 플레이어가 스탠드하지 않음 |
| 스탠드 | `PlayerTurn`이고 플레이어가 스탠드하지 않음 |
| 재시작 | `BattleEnded` |

연속 클릭으로 행동이 중복 처리되지 않도록 입력을 받은 즉시 해당 입력을 잠근다.

## 9. 오류 및 경계 조건

- 뽑을 덱과 버린 더미가 모두 비면 명시적인 오류로 처리하고 진행을 중단한다.
- 영혼은 0 미만으로 표시하지 않는다.
- 같은 라운드 결과가 두 번 적용되지 않도록 라운드 식별자 또는 처리 완료 플래그를 둔다.
- 재시작 시 이전 이벤트 구독, 코루틴과 지연 연출이 새 상태를 수정하지 못하게 한다.
- 에이스가 여러 장이어도 21 이하 최대 합계를 계산한다.
- 양측 모두 21 초과 상태의 비교 규칙은 원본 문서와 전체 기획서 기준을 따른다.

- 카드와 합계가 상태 변화 직후 일치한다.

- 라운드 결과와 영혼 감소 원인을 플레이어가 알 수 있다.
- 승리·패배 후 이전 라운드 표시가 남지 않는다.

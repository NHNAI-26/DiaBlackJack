> 프로젝트: DiaBlackJack

> 버전: v0.1
> 상태: RW-01 보상 데이터·런 덱 기반 구현 완료
> 최종 갱신: 2026-07-20

## 1. 문서 목적

이 문서는 전투 승리 후 카드 보상 3장을 제시하고, 플레이어가 1장을 현재 런 덱에 추가하거나 보상을 건너뛰는 과정을 구현 가능한 수준으로 확정한다.

전투 보상은 완료된 코어 루프, 런·스테이지 진행 SP-00~SP-04, 전투 행동과 카드 사용 시스템을 재작성하지 않는 별도 `RW`(Reward) 작업이다. 일반·엘리트·최종 보스 전투의 승리 결과와 다음 진행 상태 사이에 보상 선택을 추가한다.

최종 기획 승인과 임시 결정의 변경 책임자는 이천서다.

## 4. 플레이 흐름

### 4.1 일반·엘리트 전투 승리

```text
InBattle
→ 전투 승리와 남은 영혼 동기화
→ RewardSelection
→ 카드 1장 선택 또는 SKIP
→ StageCleared
→ 결과 확인 후 다음 이벤트 또는 스테이지 진행
```

### 4.2 최종 보스 승리

```text
InBattle
→ 최종 보스 승리와 남은 영혼 동기화
→ RewardSelection
→ 높은 등급 카드 1장 선택 또는 SKIP
→ RunVictory
→ 최종 런 덱과 보상 결과 표시
```

### 4.3 전투 패배

```text
InBattle
→ 플레이어 영혼 0
→ RunDefeat
```

패배에는 보상 후보를 생성하지 않는다.

## 5. 공통 전투 보상 규칙

1. 보상 후보는 전투 승리가 확정되고 지속 영혼이 동기화된 뒤 한 번만 생성한다.
2. 한 보상 화면에는 정확히 3개의 서로 다른 카드 정의를 제시한다.
3. 플레이어는 후보 하나를 선택하거나 보상을 건너뛸 수 있다.
4. 선택한 카드만 현재 런 덱에 새 물리 카드 인스턴스로 추가한다.
5. 선택하지 않은 후보는 덱·버린 더미·별도 보관 영역에 남기지 않는다.
6. 보상 선택은 영혼을 회복하거나 소비하지 않는다.
7. 선택과 건너뛰기는 각각 한 번만 승인된다.
8. 잘못된 후보 ID, 중복 입력과 보상 외 상태의 입력은 아무 상태도 바꾸지 않는다.
9. 보상 생성과 선택은 전투 카드의 공개 여부·사용 상태와 무관하다.
10. 보상 처리 중에는 다음 스테이지, 런 재시작과 전투 행동 입력을 모두 차단한다.

## 6. 보상 풀과 카드 등급

### 6.1 일반 보상 풀

일반 적 승리에는 일반 보상 풀을 사용한다. RW 프로토타입의 일반 풀은 현재 출시 카탈로그 전체를 명시적으로 등록한다.

| 카드 정의 키 | 표시 이름 | 숫자 |
| --- | --- | ---: |
| `standard-ace-1` | 에이스 | 1 |
| `standard-plain-2` | 기본 카드 | 2 |
| `standard-plain-3` | 기본 카드 | 3 |
| `standard-plain-4` | 기본 카드 | 4 |
| `crystal-orb-5` | 수정 구슬 | 5 |
| `threat-hammer-6` | 위협용 해머 | 6 |
| `auto-pistol-7` | 자동 권총 | 7 |
| `auto-pistol-8` | 자동 권총 | 8 |
| `military-knife-9` | 군용 나이프 | 9 |
| `military-knife-10` | 군용 나이프 | 10 |

### 6.2 높은 등급 보상 풀

엘리트와 최종 보스 승리에는 높은 등급 보상 풀을 사용한다. RW 프로토타입에서는 실제 사용 효과가 구현된 다음 6개 정의를 명시적으로 등록한다.

| 카드 정의 키 | 표시 이름 | 숫자 |
| --- | --- | ---: |
| `crystal-orb-5` | 수정 구슬 | 5 |
| `threat-hammer-6` | 위협용 해머 | 6 |
| `auto-pistol-7` | 자동 권총 | 7 |
| `auto-pistol-8` | 자동 권총 | 8 |
| `military-knife-9` | 군용 나이프 | 9 |
| `military-knife-10` | 군용 나이프 | 10 |

이 목록은 카드 등급 체계가 생기기 전의 프로토타입 결정이다. 높은 등급을 숫자 7 이상처럼 숫자만으로 판단하지 않는다. 숫자는 블랙잭 확률과 직접 연결되므로 등급과 분리한다.

### 6.3 후보 부족

해당 보상 풀에 서로 다른 카드 정의가 3개 미만이면 보상 생성을 승인하지 않는다. 후보를 중복하거나 다른 등급 풀로 조용히 대체하지 않는다. 출시용 보상 카탈로그는 각 풀에 최소 3개의 유효한 정의를 제공해야 한다.

## 7. 최종 보스 보상 의미

최종 보스 보상은 선택 완료 뒤 추가 전투에 사용되지 않는다. 프로토타입에서는 다음 목적으로 유지한다.

- 최종 런 덱과 완주 결과 기록
- 높은 등급 보상 경험의 일관성
- 향후 컬렉션·해금·메타 성장 연결점 확보

현재 범위에서는 보상 카드를 다음 런으로 계승하거나 영구 컬렉션에 저장하지 않는다. 메타 성장 요구사항이 확정되기 전에 임시 저장 구조를 만들지 않는다.

## 8. 런 덱 추가 규칙

- 보상 후보는 카드 정의 키만 참조하며 아직 런 덱 카드 ID를 소유하지 않는다.
- 선택이 승인될 때 새 `RunCardDefinition`을 생성한다.
- 새 카드 ID는 현재 런에서 사용한 ID와 중복되지 않는 증가값을 사용한다.
- 삭제 기능이 나중에 추가돼도 같은 런에서 사용했던 ID를 재사용하지 않는 것을 권장한다.
- 같은 정의 키를 여러 번 획득하는 것은 허용하지만 물리 카드 ID는 항상 다르다.
- 런 재시작은 최초 덱으로 복구하며 이전 런에서 획득한 보상 카드를 제거한다.

## 9. 화면 요구사항

`StageTest`의 진행 화면에 보상 상태를 표시한다.

- 승리한 적과 보상 등급
- 후보 카드 3장의 숫자, 이름과 효과 요약
- 카드별 `SELECT` 입력
- `SKIP REWARD` 입력
- 선택 후 추가된 카드와 현재 덱 장수
- 최종 보스 보상인 경우 `보상 완료 후 런 승리` 안내

보상 화면에서는 `NEXT STAGE`, `RESTART RUN`과 전투 행동을 표시하거나 활성화하지 않는다. 최종 카드 레이아웃·애니메이션·사운드·현지화는 별도 UI 작업이다.

## 10. 포함 범위

- 전투 승리 후 보상 후보 3장 생성
- 일반·높은 등급 보상 풀 구분
- 카드 1장 선택과 런 덱 추가
- 보상 건너뛰기
- 일반·엘리트·최종 보스 완료 상태 연결
- 최종 보스 보상 완료 후 `RunVictory`
- 진행 화면 표시와 입력
- 결정적 생성과 전체 회귀 테스트

## 11. 제외 범위

- 카드 제거와 강화
- 영혼 회복
- 유물과 악마 카드 보상
- 휴식·상점·무작위 사건 구현
- 상대 후보 선택과 엘리트 등장 확률
- 카드 등급별 최종 밸런스와 가중 드롭 확률
- 메타 성장, 영구 해금과 저장·불러오기
- 보상 교체, 새로고침과 가중치 조작
- 최종 아트·연출·사운드

## 12. 실패·경계 조건

- 전투 중이거나 패배 상태에서는 보상을 시작할 수 없다.
- 이미 보상 선택 중이면 새 보상을 생성할 수 없다.
- 후보 수가 3개가 아니거나 정의 키가 중복되면 보상을 시작하지 않는다.
- 카탈로그에 없는 정의 키를 선택할 수 없다.
- 선택 완료 뒤 같은 후보를 다시 선택할 수 없다.
- 건너뛴 뒤 카드를 추가할 수 없다.
- 일반 전투 보상 뒤에만 `StageCleared`가 된다.
- 최종 보스 보상 뒤에만 `RunVictory`가 된다.
- 보상 중 런 재시작과 다음 스테이지 입력을 거절한다.
- 런 재시작 후 보류 보상과 획득 카드가 남지 않는다.

## 13. 완료 조건

- 일반 전투 승리 뒤 정확히 3개의 일반 보상 후보가 표시된다.
- 엘리트와 최종 보스는 높은 등급 후보만 표시한다.
- 선택한 카드 하나만 고유 ID로 런 덱에 추가된다.
- 건너뛰기는 덱을 바꾸지 않는다.
- 일반 전투 보상 완료 뒤 `StageCleared`가 된다.
- 최종 보스 보상 완료 뒤 `RunVictory`가 된다.
- 패배에는 보상이 생성되지 않는다.
- 중복·잘못된 입력은 진행과 덱을 바꾸지 않는다.
- 재시작은 최초 덱과 첫 진행 상태를 복구한다.
- 신규·기존 EditMode 테스트와 실제 진행 화면 흐름이 통과한다.
- AI 활용 기술서와 이천서 역할 기록을 단계마다 갱신한다.

| 결정 | 현재안 | 재검토 시점 |

| 보스 보상 카드 용도 | 최종 덱·결과 기록만 | 메타 성장 명세 작성 시 |

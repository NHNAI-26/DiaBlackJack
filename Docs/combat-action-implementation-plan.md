# 전투 행동 확장 구현 계획서

> 프로젝트: DiaBlackJack  
> 문서 책임자: 이천서  
> 버전: v0.1  
> 상태: BA-02 완료·BA-03 착수 대기
> 최종 갱신: 2026-07-19

## 1. 계획 목적

폴드와 체인지를 기존 전투에 작은 검증 단위로 추가하고, 단독 전투와 런 전투에서 모두 안전하게 작동하는지 단계별로 확인한다.

## 2. 실행 방식

이번 작업은 **전체 단계의 문서를 먼저 작성하고, 구현은 한 단계씩 완료·검증·기록·커밋하는 방식**으로 진행한다.

```text
BA-00 전체 기준 문서화
→ BA-01 상태·카드 이동 기반
→ BA-02 폴드
→ BA-03 체인지
→ BA-04 화면·입력
→ BA-05 런 연동·전체 검증
```

한꺼번에 모든 코드를 작성한 뒤 한 번에 검증하지 않는다. 각 단계는 독립적인 회귀 지점과 커밋 후보가 되어야 한다.

## 3. 단계 현황

| 단계 | 목표 | 상태 | 선행 조건 |
| --- | --- | --- | --- |
| BA-00 | 기획·기술·계획·기록 기준 확정 | 완료 | 기존 규칙·코드 확인 |
| BA-01 | 선택형 행동을 위한 상태·카드 이동 기반 | 완료 | BA-00 |
| BA-02 | 폴드 규칙과 라운드 종료 | 완료 | BA-01 |
| BA-03 | 2단계 체인지와 라운드당 제한 | 대기 | BA-01, BA-02 회귀 통과 |
| BA-04 | 표시 모델·버튼·후보 선택 입력 | 대기 | BA-02, BA-03 |
| BA-05 | 런 전투 전달·전체 회귀·문서 마감 | 대기 | BA-04 |

- `PlayerChoosingChangeCard` 상태 추가
- 손패의 비공개 카드 확인·인출 기능 추가

- 체인지 진행 중 데이터와 라운드별 사용 상태의 최소 구조 추가
- 카드 위치·총량·잘못된 상태의 불변 조건 테스트 작성

### 제외

- UI 버튼
- 런 진행 연결
- 카드 효과용 범용 선택 시스템

### 실제 구현 결과

- `PlayerChoosingChangeCard` 상태를 추가했다.
- 손패에서 비공개 카드가 정확히 한 장일 때만 원자적으로 꺼내는 기능을 추가했다.

## 6. BA-02 — 폴드 구현

### 목표

플레이어가 영혼 1을 지불하고 현재 라운드를 즉시 포기할 수 있게 한다.

### 작업

- `RoundOutcome.PlayerFold`와 피해 1 정의
- `CoreLoopBattle.TryPlayerFold()` 구현
- 공통 라운드 종료·카드 정리 경로 재사용
- `CoreLoopSession` 전달
- 정상 폴드, 영혼 1 폴드, 중복·잘못된 상태 테스트

### 완료 게이트

- [x] 폴드 결과당 피해가 정확히 1회 적용
- [x] 적 차례와 합계 판정이 실행되지 않음
- [x] 전투 계속 시 다음 라운드, 영혼 0이면 전투 종료
- [x] CoreLoop 세션 전달과 최근 결과 표시 호환
- [x] 기존 히트·스탠드와 전체 진행 회귀 통과

### 실제 구현 결과

- `RoundOutcome.PlayerFold`와 피해 1의 명시적 판정 경로를 추가했다.
- `CoreLoopBattle.TryPlayerFold()`가 기존 행동 조건을 재사용한다.
- 기존 판정과 폴드가 같은 `CompleteRound` 정리 경로를 사용한다.
- 폴드는 적 행동 없이 양쪽 손패를 버리고, 전투가 계속되면 다음 라운드를 시작한다.
- 영혼 1에서 폴드하면 새 라운드 없이 `PlayerDefeat`가 된다.
- `CoreLoopSession` 전달과 `CoreLoopPresenter` 최근 결과 호환을 추가했다.
- 폴드 버튼·Controller와 런 진행 세션 연결은 계획대로 BA-04·BA-05에 남겼다.

## 10. 단계별 공통 작업 절차

1. 진행 기록에서 해당 단계를 `진행 중`으로 바꾼다.
2. 해당 단계의 실패 테스트 또는 검증 조건을 먼저 고정한다.
3. 계획 범위의 최소 코드만 수정한다.
4. 신규 테스트를 통과시킨다.
5. 전체 EditMode 회귀 테스트를 실행한다.
6. 필요 단계에서는 실제 씬과 Console을 확인한다.
7. 진행 기록, AI 활용 기록과 이천서의 실제 기여를 갱신한다.
8. 변경 범위를 검토한 뒤 단계 단위로 커밋한다.

# 데블랙잭 코어 루프 기획서

> 프로젝트: DiaBlackJack  
> 기획 책임자: 이천서  
> 버전: v0.1  
> 상태: 구현 전 확정안  
> 최종 갱신: 2026-07-19

## 1. 목적

전체 게임을 만들기 전에 데블랙잭의 가장 작은 반복 플레이 단위를 구현해 재미와 규칙을 검증한다. 이번 단계에서는 한 명의 적과 블랙잭 라운드를 반복해 영혼을 모두 깎고, 승리 또는 패배 후 다시 시작할 수 있는 흐름만 다룬다.

## 2. 코어 루프 한 문장

> 카드를 받고 히트 또는 스탠드를 선택해 라운드 결과를 만들고, 결과에 따라 영혼 피해를 주고받으며 한쪽 영혼이 0이 될 때까지 반복한다.

## 3. 플레이 흐름

1. 테스트 씬에서 게임을 시작한다.
2. 플레이어 영혼 12, 적 영혼 3과 각자의 기본 덱을 준비한다.
3. 양측이 공개 카드 1장과 비공개 카드 1장씩 받고 라운드를 시작한다.
4. 플레이어는 히트 또는 스탠드를 선택한다.
5. 적은 단순 규칙에 따라 히트 또는 스탠드한다.
6. 버스트가 발생하거나 양측이 스탠드하면 승패를 판정한다.
7. 패자에게 영혼 피해를 적용한다.
8. 양측 영혼이 남아 있으면 사용 카드를 정리하고 다음 라운드를 시작한다.
9. 한쪽 영혼이 0이면 승리 또는 패배 화면을 표시한다.
10. 재시작을 선택하면 2번 상태로 돌아간다.

## 4. 이번 단계의 규칙

### 4.1 시작 상태

| 항목 | 값 |
| --- | ---: |
| 플레이어 영혼 | 12 |
| 적 영혼 | 3 |
| 양측 덱 | 숫자 1~10 각 2장, 총 20장 |
| 시작 카드 | 각 2장 |
| 선공 | 플레이어 |

숫자 1은 에이스로 취급해 1 또는 11 중 21 이하에서 가장 큰 합계를 자동 선택한다.
플레이어는 자기 카드 2장을 모두 확인하며, 적의 비공개 카드는 라운드 판정 전까지 볼 수 없다.

### 4.2 가능한 행동

- **히트**: 덱에서 카드 1장을 공개로 추가한다.
- **스탠드**: 이번 라운드의 행동을 끝낸다.

폴드, 카드 사용, 체인지, 계약은 이번 단계에서 제공하지 않는다.

### 4.3 적 행동

- 합계 16 이하: 히트
- 합계 17 이상: 스탠드

적은 자신의 정상적인 카드 정보만 사용하며 플레이어 비공개 정보는 읽지 않는다.

### 4.4 라운드 종료와 피해

- 플레이어 버스트: 플레이어 영혼 2 감소
- 적 버스트: 적 영혼 1 감소
- 양측 스탠드: 21 이하에서 합계가 더 높은 쪽 승리, 패자 영혼 1 감소
- 정확히 21로 플레이어 승리: 적 영혼 2 감소
- 동점: 원본 규칙에 따라 플레이어 승리, 적 영혼 1 감소

한쪽이 버스트하면 즉시 라운드를 종료한다. 양측 스탠드 비교에서는 비공개 카드를 공개한다.

### 4.5 카드 순환

- 라운드 종료 시 양측 카드 전체를 버린 카드 더미로 이동한다.
- 뽑을 덱이 비면 버린 카드 더미를 섞어 새 덱으로 사용한다.
- 새 전투 또는 재시작 시 20장 기본 덱으로 초기화한다.

## 5. 최소 화면 요소

- 플레이어와 적의 현재 영혼
- 플레이어 카드와 현재 합계
- 적 카드와 공개된 합계
- 덱에 남은 카드 수
- 히트 버튼과 스탠드 버튼
- 현재 차례 또는 처리 상태
- 라운드 결과 메시지
- 전투 승리·패배 메시지와 재시작 버튼

임시 도형, 텍스트와 버튼을 사용해도 된다. 아트 완성도와 애니메이션은 평가 대상이 아니다.

## 6. 포함 범위

- 한 전투의 시작과 종료
- 여러 라운드 반복
- 기본 덱 생성, 드로우, 버림과 재섞기
- 에이스 포함 합계 계산
- 플레이어 히트·스탠드 입력
- 단순 적 AI
- 버스트와 스탠드 비교
- 영혼 피해와 승리·패배
- 즉시 재시작

## 7. 제외 범위

- 특수 카드 효과와 카드 사용
- 폴드, 체인지, 악마 계약
- AI 확률 추론 및 적 성향
- 보상, 덱 빌딩, 휴식, 상점, 지도
- 여러 적, 엘리트, 보스 패턴
- 저장·불러오기와 메타 성장
- 완성 아트, 사운드, 연출과 튜토리얼

제외 기능을 위해 미리 복잡한 확장 구조를 만들지 않는다. 다만 상태와 계산 로직을 UI에서 분리해 다음 단계 확장을 막지 않도록 한다.

## 8. 사용자 경험 목표

- 시작 후 10초 안에 첫 선택을 할 수 있다.
- 버튼을 누르면 카드, 합계, 상태 변화의 원인을 이해할 수 있다.
- 한 라운드가 불필요한 대기 없이 진행된다.
- 영혼 피해와 다음 라운드 진입이 명확하게 구분된다.
- 승리와 패배 후 한 번의 입력으로 같은 조건을 다시 시험할 수 있다.

## 9. 완료 조건

- 테스트 씬에서 별도 설정 없이 플레이 가능하다.
- 플레이어와 적이 히트·스탠드를 통해 정상적으로 라운드를 끝낸다.
- 에이스, 버스트, 동점, 정확히 21의 결과가 문서와 일치한다.
- 영혼이 남으면 새 라운드, 영혼이 0이면 전투 결과로 이동한다.
- 덱 소진 후 재섞기와 재시작이 정상 작동한다.
- 최소 10회 연속 재시작해도 진행 불가 상태가 발생하지 않는다.

## 10. 코어 루프 외 별도 확장 작업

아래 항목은 코어 루프의 추가 단계로 부르지 않고 각각 별도 기능 작업으로 관리한다.

1. 폴드와 체인지
2. 일반 카드 효과와 카드 사용
3. 공개·비공개 정보 모델과 AI 추론 UI
4. 악마 계약과 영혼 소비
5. 전투 보상과 덱 빌딩
6. 적 종류와 보스
7. 런·스테이지 진행, 휴식·상점·사건

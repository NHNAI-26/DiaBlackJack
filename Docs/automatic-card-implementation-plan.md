## 1. 구현 전략

자동 발동 카드 5종을 한 번에 효과 처리기에 추가하지 않는다. 먼저 공개 카드 유입과 바깥 처리 재개를 고정한 뒤, 서로 다른 위험을 가진 카드를 순서대로 수직 구현한다.

```text
AC-00 문서 기준
  → AC-01 공개 유입·자동 선택 기반
  → AC-02 독극물과 라운드 승리 후속 효과
  → AC-03 거짓말 탐지기와 소유자 전용 정보
  → AC-04 화염 방사기·회중시계의 다단 선택
  → AC-05 부활초의 승패 없는 라운드 초기화
  → AC-06 UI·런·보상·적 AI·반복 검증
```

각 단계는 기준 확인 → 회귀 테스트 → 최소 구현 → 대상·전체 검증 → 기록 → 사용자 커밋 순서로 완료한다. 구현하지 않은 다음 단계 코드를 미리 섞지 않는다.

## 2. 실행 원칙

1. 한 단계마다 독립적으로 검토 가능한 작은 변경을 만든다.
2. 기존 공개 유입과 버스트 동작을 회귀 테스트로 고정한 뒤 구조를 바꾼다.
3. 순수 C# 규칙과 EditMode 테스트를 UI보다 먼저 구현한다.
4. 자동 효과와 수동 효과의 실행기를 분리하고 데이터·표시 기반만 재사용한다.
5. 공개 카드 유입을 문자열·이벤트 추측이 아닌 명시적 API로 연결한다.
6. 입력 보류는 증가형 상호작용 ID로 중복·오래된 입력을 거절한다.
7. 적 자동 선택에는 플레이어 비공개 값과 실제 덱 순서를 전달하지 않는다.
8. 상점·골드·정식 런 진행과 새 외부 에셋은 변경하지 않는다.
9. 단계 완료 뒤 진행 기록, AI 활용 문서와 이천서 역할 기록을 갱신한다.
10. AI는 사용자의 명시적 요청 없이 스테이징·커밋·푸시하지 않는다.

## 4. AC-00 — 문서 기준 수립

### 작업

- `rule.md`, 전체 기획과 카드 사용 문서의 자동 발동 규칙을 대조한다.
- 현재 카드 정의·수동 효과·전투·진행·적 AI·UI 코드를 확인한다.
- 최초 배분 예외, 공개 유입, 효과 뒤 위치와 버스트 순서를 명시한다.
- 독극물·부활초·탐지기·화염 방사기·회중시계의 모호한 규칙을 프로토타입 결정으로 분리한다.
- 기획서, 개발 명세서, 구현 계획서와 진행 기록서를 작성한다.
- README, AI 활용과 이천서 역할 기록을 갱신한다.

### 완료 게이트

- 네 문서의 작업 ID, 책임자, 현재·다음 단계가 일치한다.
- 최초 배분 비발동과 `자동 효과 → 원본 위치 → 공개 합 버스트` 순서가 일치한다.
- 부활초 영혼 1, 독극물 스탠드 금지, 탐지기 정보 공개, 회중시계 순환 방지 결정이 기록돼 있다.
- 구현하지 않은 기능이 완료로 기록되지 않는다.
- 코드·테스트·씬·패키지·외부 에셋 변경과 Unity 재실행이 없다.

### 완료 결과

- 자동 발동 카드 문서 4종을 작성했다.
- 이천서를 기획·개발 책임자로, AI를 규칙·구조 대조와 문서 초안 보조로 기록했다.
- 직전 DC-08 전체 397/397은 선행 기준으로만 기록하고 AC-00 테스트 결과로 사용하지 않았다.
- 다음 구현 대상을 AC-01 공통 기반으로 확정했다.

### 권장 커밋 제목

`자동 카드가 전투 흐름을 끊지 않도록 발동 순서와 재개 경계를 고정하다`

## 5. AC-01 — 공개 유입과 자동 선택 기반

### 목표

실제 카드 효과 없이도 최초 배분 예외, 플레이어·적 공개 유입, 선택 보류와 원래 처리 재개를 검증할 수 있는 공통 기반을 만든다.

### 선행 작업

- AC-00 문서 네 종 검토
- 현재 전체 EditMode와 컴파일 기준선 재확인
- 직접 공개 드로우·추가 호출부 목록 고정

### 작업

- `CardEffectKind` 자동 효과 5종과 카드 정의 키 5개 추가
- 자동 카드 Resolver와 처리기 등록 경계
- 공개 카드 유입 조정자와 유입 원인
- 자동 해결 프레임과 열거형 기반 연속 처리
- 증가형 상호작용 ID와 선택 주체를 가진 보류 모델
- `ResolvingAutomaticCardEffect` 상태와 입력 잠금
- 최초 배분·비공개 유입 제외
- 테스트 처리기를 사용한 즉시 완료·선택 대기·버스트·재개 검증
- 일반 히트, 적 히트, 수정 구슬, 강제 드로우의 공통 유입 이관

### 수정 범위

- CoreLoop 카드 정의·상태·전투·참가자
- 공개 카드를 추가하는 기존 수동 카드 처리기
- 신규 `AutomaticCards` 순수 C# 영역
- AC-01 EditMode 테스트

UI, 진행 세션, 보상 풀과 적 프로필은 변경하지 않는다.

### 완료 게이트

- 최초 배분 자동 카드는 발동하지 않는다.
- 히트·효과·강제 드로우의 자동 카드는 같은 순서로 처리된다.
- 비공개 체인지·해머 교체는 발동하지 않는다.
- 자동 선택 중 다른 모든 입력이 무변경 거절된다.
- 오래된 상호작용과 중복 입력이 무변경 거절된다.
- 자동 효과 완료 뒤 공개 합 버스트 또는 원래 행동 중 하나만 정확히 한 번 실행된다.
- 카드 총수와 물리 ID가 보존된다.
- AC-01 대상·전체 EditMode와 컴파일이 통과한다.

### 권장 커밋 제목

`공개 카드의 자동 효과가 모든 드로우 경로에서 한 번만 해결되게 하다`

## 6. AC-02 — 독극물 수직 기능

### 목표

자동 카드 소유자의 선택, 계약 제한, 영혼 0 즉시 패배와 라운드 승리 후속 효과를 첫 실제 카드로 검증한다.

### 작업

- `PoisonEffectHandler`
- 즉시 스탠드·영혼 지불 선택
- 계약을 포함한 실제 스탠드 가능 여부 반영
- 영혼 `min(3, 현재)` 손실과 영혼 0 즉시 종료
- 물리 카드별 현재 라운드 승리 회복 예약
- 라운드 승패·부활초 초기화·전투 종료 시 예약 정리
- 플레이어와 적 양측 규칙 단위 테스트
- 적 독극물 선택 정책의 순수 기반

### 완료 게이트

- 스탠드 금지 상태에서 스탠드 선택지가 없다.
- 영혼 3 이상·미만·정확히 1의 결과가 정확하다.
- 영혼 0이면 원래 행동과 적 후속 처리가 실행되지 않는다.
- 승리만 영혼 5를 회복하고 최대 영혼을 넘지 않는다.
- 여러 예약이 카드별로 한 번씩 해결되고 라운드 밖에 남지 않는다.
- 원본 독극물은 해결 후 버린다.
- AC-02 대상·전체 EditMode와 컴파일이 통과한다.

### 권장 커밋 제목

`독극물의 생존 선택이 계약 제한과 영혼 위험을 정확히 감수하게 하다`

## 7. AC-03 — 거짓말 탐지기와 정보 경계

### 목표

카드가 허용한 숨은 정보만 효과 소유자에게 제공하고, 공용 결과와 적 관측에는 실제 숫자가 새지 않게 한다.

### 작업

- `LieDetectorEffectHandler`
- 1~10 숫자 선언과 정확히 한 장인 상대 비공개 카드 검사
- 공개 선언 결과와 소유자 전용 이상·미만 결과 분리
- 실제 비공개 숫자가 없는 비교 지식 모델
- 숨은 카드 ID 교체·새 라운드·전투 종료 시 지식 폐기
- 적 선언 값 선택을 위한 공개 추론 분포 경계
- 플레이어·적 양측 정보 은닉 테스트

### 완료 게이트

- 범위 밖 선언과 오래된 선택을 거절한다.
- 소유자는 정확한 이상·미만 결과를 받는다.
- 상대용 표시, 공용 로그와 결과에 실제 숫자와 비교 결과가 없다.
- 적 관측에는 허용된 비교 조건만 존재한다.
- 체인지·해머·새 라운드 뒤 이전 지식을 사용할 수 없다.
- 숨은 카드 불변식 위반은 판정 불가로 안전하게 끝난다.
- AC-03 대상·전체 EditMode와 컴파일이 통과한다.

### 권장 커밋 제목

`탐지기가 허용한 비교 정보만 소유자에게 남기도록 숨은 값 경계를 지키다`

## 8. AC-04 — 화염 방사기와 회중시계

### 목표

소유자와 상대가 순서대로 선택하는 효과, 카드 폐기와 사용 상태 재활성화를 무한 연쇄 없이 구현한다.

### 작업

- `FlamethrowerEffectHandler`
- 발동 시점 스탠드 여부와 공개 카드 후보 생성
- 소유자→상대의 단계형 폐기 선택
- 원본 제외, 건너뛰기와 선택 시점 재검증
- `PocketWatchEffectHandler`
- 사용 완료 수동 카드 후보와 재활성화
- 자동 카드·자기 자신·사용 가능 카드 제외
- 회중시계 원본 유지·폐기 선택
- 적 공개 카드 폐기·재사용 우선순위

### 완료 게이트

- 화염 방사기 선택 순서와 선택 주체가 바뀌지 않는다.
- 스탠드 참가자와 후보 없는 단계는 교착 없이 넘어간다.
- 선택 카드가 정확한 소유자의 버린 더미로 이동한다.
- 양측 선택·원본 폐기 뒤 공개 합을 계산한다.
- 회중시계는 사용 완료 수동 카드만 `Available`로 되돌린다.
- 유지한 회중시계가 같은 손에서 다시 발동하지 않는다.
- 자동 카드끼리 재활성화하거나 효과를 즉시 실행하지 않는다.
- AC-04 대상·전체 EditMode와 컴파일이 통과한다.

### 권장 커밋 제목

`양측 카드 폐기와 재사용 선택이 순서와 소유권을 잃지 않게 하다`

## 9. AC-05 — 부활초 라운드 초기화

### 목표

승자·일반 피해 없이 현재 라운드를 끝내고 바깥 효과를 취소한 뒤 새 라운드를 한 번만 시작한다.

### 작업

- `ResurrectionHerbEffectHandler`
- 양측 영혼 2 이상 조건과 재시작·거절 선택
- 승패 없는 전용 라운드 전이 결과
- 양측 영혼 동시 1 손실
- 손패 폐기, 스탠드·자동 상태·탐지기 지식·독극물 예약 정리
- 계약 라운드 종료 훅 한 번 호출
- 바깥 수동 효과와 자동 연속 처리 취소
- 새 라운드 최초 배분 예외 보존
- 중첩·강제 드로우·거절 경로 테스트

### 완료 게이트

- 어느 한쪽이라도 영혼 1이면 재시작 옵션이 없다.
- 활성화가 영혼·손패·라운드 번호를 정확히 한 번 변경한다.
- 일반 피해, 21 보너스와 독극물 회복이 발생하지 않는다.
- 바깥 카드 효과와 남은 자동 대기열이 재개되지 않는다.
- 계약 라운드 훅과 새 라운드 시작이 각각 한 번이다.
- 새 최초 배분의 자동 카드가 발동하지 않는다.
- 거절은 원본 기본 폐기 외 영혼·라운드 상태를 바꾸지 않는다.
- AC-05 대상·전체 EditMode와 컴파일이 통과한다.

### 권장 커밋 제목

`부활초가 승패와 후속 효과를 중복시키지 않고 라운드만 되돌리게 하다`

## 10. AC-06 — UI·런·보상·적 AI 통합과 마감

### 목표

플레이어가 실제 독립·런 전투 화면에서 자동 효과를 해결하고, 적이 정보 경계를 지키며 사용하고, 보상으로 얻은 카드가 다음 전투에서 동작하도록 1차 범위를 마감한다.

### 작업

- 두 세션의 자동 선택 입력 전달과 종료 동기화
- 자동 상호작용·공개 결과·소유자 전용 결과 표시 모델
- 기존 View·Controller·GameManager 입력 연결
- 플레이어 차례와 적 차례 양쪽의 선택 대기 화면
- 독극물 회복 예약·탐지기 결과·회중시계 상태 표시
- 적 자동 선택 정책과 결정 검증
- 일반 보상 풀에 자동 카드 5종 추가
- 사기꾼 덱에 거짓말 탐지기 연결
- 독립·런·보상·다음 전투·재시작 통합 테스트
- 각 대표 흐름 10회 반복 회귀
- 두 씬, 1280×720·1920×1080과 Console 검증
- 문서 4종, README, AI 활용·팀 역할·구조 기록 마감

### 수정 금지·보호 범위

- HONG의 골드·상점·정식 런 진행 구조
- Shim0Hwan의 아트 오브젝트·재질·연출 자산
- 시작 덱 구성
- 높은 등급 보상 풀
- 패키지와 외부 에셋

### 완료 게이트

- 플레이어 자동 선택이 독립·런 세션에서 동일하게 동작한다.
- 적 차례 중 플레이어 대상 선택도 UI 교착 없이 끝난다.
- 적은 실제 플레이어 비공개 값 없이 5종 선택을 완료한다.
- 일반 보상으로 획득한 자동 카드가 다음 전투에서 발동한다.
- 사기꾼 탐지기 결과가 플레이어 화면에 노출되지 않는다.
- 10회 반복에서 카드 총수, ID, 예약·지식·상호작용과 전투 인스턴스가 격리된다.
- 대상·전체 EditMode, 두 씬·두 해상도·Console이 통과한다.
- 계획과 실제 기여, AI 보조, 외부 에셋 없음이 문서에 기록된다.

## 15. 최종 완료 체크리스트

- [ ] AC-01~AC-06 단계별 완료 증거가 있다.
- [ ] 자동 카드 5종이 플레이어와 적 양측에서 동작한다.
- [ ] 최초 배분·비공개 유입 예외가 유지된다.
- [ ] 공개 효과·버스트·연속 처리 순서가 일관된다.
- [ ] 독극물 회복과 부활초 초기화가 중복 처리되지 않는다.
- [ ] 거짓말 탐지기 실제 숨은 값이 노출되지 않는다.
- [ ] 회중시계 무한 연쇄가 차단된다.
- [ ] 일반 보상 획득과 다음 전투 사용이 연결된다.
- [ ] 적 AI가 유효 선택을 정보 경계 안에서 완료한다.
- [ ] 전체 EditMode, 두 씬, 두 해상도와 Console 검증이 통과한다.
- [ ] 이천서의 실제 기여와 AI 보조가 구분돼 기록된다.
- [ ] 새 외부 에셋·패키지·오픈소스가 없거나 출처가 기록된다.

- 계획, 착수, 구현, 검증과 완료를 구분한다.
- 실행하지 않은 테스트를 통과로 기록하지 않는다.
- 단계별 코드·씬·문서 변경과 이천서의 실제 담당 범위를 남긴다.

### BA-01 — 상태·카드 이동 기반

**상태:** 완료
**담당:** 이천서
**AI 역할:** 현재 코드·테스트 구조 분석, 기반 코드·테스트 초안, Unity MCP 컴파일·회귀 검증과 기록 보조

#### 수행 내용

- `CoreLoopState.PlayerChoosingChangeCard`를 추가했다.
- `BlackjackHand`가 비공개 카드 수를 제공하고, 정확히 한 장일 때만 상태 변화 없이 검증 후 인출하도록 했다.
- `BlackjackDeck`이 뽑을 더미와 버린 더미에서 확보 가능한 카드 수를 계산하고 `CanDraw`로 검사하도록 했다.
- `PlayerChangeSelection`이 기존 비공개 카드와 후보 2장의 ID 중복을 거부하고, 후보 선택을 한 번만 완료하도록 했다.
- 잘못된 후보 인덱스, 비공개 카드 0장·복수 장, 후보 부족 검사의 무변경 조건을 테스트했다.
- 범용 행동 디스패처, UI, 전투 행동 API와 진행 세션 연결은 추가하지 않았다.

### BA-02 — 폴드

**상태:** 완료
**담당:** 이천서
**AI 역할:** 폴드 규칙 경계 분석, 판정·공통 종료·세션 코드와 테스트 초안, Unity MCP 회귀 검증 및 기록 보조

#### 수행 내용

- `RoundOutcome.PlayerFold`와 플레이어 피해 1의 명시적 결과를 추가했다.
- `TryPlayerFold`가 플레이어 차례에서만 승인되고 적 행동·합계 비교 없이 라운드를 끝내도록 했다.
- 기존 합계 판정과 폴드가 `CompleteRound`에서 피해·손패 정리·전투 종료·다음 라운드 처리를 공유하도록 했다.
- 영혼 1에서 폴드하면 `PlayerDefeat`로 종료하고 새 라운드를 만들지 않도록 했다.
- `CoreLoopSession`에 폴드 전달을 추가했다.
- 새 `RoundOutcome`이 최근 결과 표시에서 예외를 만들지 않도록 폴드 문구를 추가했다.
- UI 버튼·Controller와 `StageProgressionSession` 연결은 아직 추가하지 않았다.

### BA-03 — 체인지

**상태:** 완료
**담당:** 이천서
**AI 역할:** 체인지 상태·카드 이동 경계 분석, 전투·참가자·세션 코드와 테스트 초안, Unity MCP 회귀 검증 및 기록 보조

#### 수행 내용

- `CoreLoopBattle`에 체인지 시작 가능 여부, 진행 중 후보, 선택 가능 여부와 라운드당 사용 상태를 추가했다.
- 체인지 시작 전 플레이어 차례, 미사용, 비공개 카드 1장과 후보 2장 확보 조건을 먼저 검사한다.
- 기존 비공개 카드 1장과 후보 2장을 보류하고 후보만 공개한 뒤 `PlayerChoosingChangeCard` 상태로 전환한다.
- 잘못된 후보 번호는 카드·상태를 바꾸지 않으며, 선택 중 일반 행동과 체인지 시작 반복을 거부한다.
- 유효한 선택 카드만 비공개 손패로 넣고 기존 비공개 카드와 미선택 후보는 버린 더미로 이동한다.
- 선택 완료 뒤 기존 적 차례를 실행하며, 같은 라운드 재사용은 거부하고 새 라운드에서 제한을 초기화한다.
- `CoreLoopSession`에 체인지 시작·후보 선택 전달을 추가했다.
- UI와 `StageProgressionSession` 연결은 계획대로 BA-04·BA-05에 남겼다.

#### 검증

- 구현 전 전체 EditMode 64/64 통과, job `ebe3c93a2e8e48448b50dc09f07de4a3`
- Unity 최종 재컴파일 성공, 컴파일 오류 0
- CoreLoop 49/49 통과, job `7ed1855082ee4e898787e3dbabd65f63`
- 전체 EditMode 72/72 통과, job `ed5f5b4c18844c348f74b139592ecb01`

- Console에는 MCP WebSocket 초기화 경고 1개와 Test Framework 기반 시설 메시지 6개만 있었으며, 게임·컴파일 오류는 없었다. 분리 확인 후 Console을 초기화했다.

### BA-04 — 화면·입력

**상태:** 완료
**담당:** 이천서

#### 수행 내용

- `CoreLoopBattle`이 폴드 가능 여부를 읽기 전용으로 제공하도록 했다.
- 표시 모델에 폴드·체인지 문구와 가능 여부, 체인지 선택 상태와 후보 목록을 추가했다.

- Unity EditMode에서는 예약 호출 대신 입력 잠금을 즉시 해제하고 실제 Play Mode의 다음 프레임 해제는 유지했다.
- `StageProgressionSession`을 통한 행동 전달·종료 동기화는 BA-05 범위로 남겼다.

- 런 전투의 폴드·체인지는 BA-05에서 `StageProgressionSession`을 경유하게 하고 종료·영혼 동기화를 함께 검증한다.

- 적의 폴드·체인지 사용 여부는 적 AI 작업에서 별도로 결정한다.
- 카드 사용과 계약의 공통 선택 처리 구조는 각 기획 문서 작성 전까지 만들지 않는다.

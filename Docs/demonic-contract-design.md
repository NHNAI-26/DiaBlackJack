## 1. 문서 목적

이 문서는 `rule.md`의 악마 계약을 현재 전투·카드·런 구조에서 단계별로 구현할 수 있도록 구체화한다. 계약은 영혼을 직접 지불해 전투 규칙을 바꾸는 고위험·고효율 행동이며, 일반 카드 사용과 구분되는 DiaBlackJack의 핵심 차별점이다.

문서에 없는 세부 처리는 다음 세 종류로 구분한다.

- **현행 확정:** 원본 규칙과 전체 기획이 일치하거나 이천서가 이미 명시한 사항
- **프로토타입 결정:** 첫 구현과 테스트를 위해 이 문서에서 임시로 고정한 사항
- **기획 확인 게이트:** 구현 전에 이천서가 최종 승인하거나 플레이 테스트로 조정해야 하는 사항

AI는 규칙·코드 구조 대조와 문서 초안을 보조했다. 기획 승인과 변경 결정 책임자는 이천서다.

## 3. 핵심 기획 결정

| 항목 | 현행 기준 |
| --- | --- |
| 계약 진입 | 플레이어 차례에 테이블의 별도 악마 카드 덱을 선택한다. |
| 기본 비용 | 영혼 1 |
| 비용 조건 | 현재 영혼이 비용보다 많아야 한다. 기본 비용 1일 때 영혼 2 이상이어야 한다. |
| 기본 횟수 | 전투 스테이지당 1회 |
| 후보 | 악마 덱 위 3장 |
| 선택 | 3장 중 정확히 1장을 반드시 선택한다. |
| 미선택 카드 | 해당 전투의 악마 버린 더미로 보낸다. |
| 지속 범위 | 선택한 상대와의 전투 전체, 즉 여러 라운드에 걸쳐 유지하고 전투 종료 시 해제한다. |
| 행동 소비 | 계약 선택과 즉시 효과 처리는 플레이어의 정상 행동 1회를 소비한다. |
| 취소 | 비용 지불 전 확인창은 닫을 수 있다. 비용 지불 후 후보가 공개되면 취소·환불할 수 없다. |
| 개별 대가 | 영혼을 0까지 감소시킬 수 있고, 0이 되는 순간 즉시 전투 패배한다. |
| 선택형 문구 | `할 수 있다`는 계약 소유자가 실행 여부를 선택한다. |
| 버스트 범위 | 숫자·카드·악마·계약 효과와 특수 패배 조건의 버스트를 모두 포함한다. |
| 생성 카드 합계 | 숫자가 적힌 계약 생성 특수 카드는 합계에 포함한다. |
| 동일 악마 중첩 | 추가 계약에서 같은 악마를 다시 선택할 수 있으며 각 계약은 별도 인스턴스다. |
| 임시 카드 수명 | 생성·이동·추방 상태는 현재 전투의 여러 라운드에 유지하고 전투 종료 시 원상복구한다. |
| 적 계약 | 플레이어와 같은 비용·선택·대가·중첩·정리 규칙을 사용한다. |
| 1차 구현 악마 | 사탄, 벨페고르, 마몬, 레비아탄 |

`현재 영혼 > 비용`은 체인지와 동일한 엄격 조건이다. 계약 시작 비용 자체가 영혼을 0으로 만들 수는 없지만 이후 악마의 개별 대가는 영혼을 0까지 감소시킬 수 있다. 영혼이 0이 되면 버스트 판정과 구분되는 즉시 전투 패배로 처리하고 남은 효과와 차례를 중단한다.

## 4. 계약 플레이 흐름

1. 플레이어가 자신의 차례에 악마 덱을 선택한다.
2. 화면에 비용, 계약 후 남는 영혼, 전투당 남은 계약 횟수를 표시한다.
3. 플레이어가 확인하면 계약 가능 조건을 다시 검사한다.
4. 영혼 1을 지불한다.
5. 악마 덱 위 3장을 공개하고 효과와 대가를 함께 표시한다.
6. 플레이어가 하나를 선택한다.
7. 선택한 악마를 활성 계약 영역으로 옮기고 나머지 둘은 악마 버린 더미로 보낸다.
8. 선택한 악마의 즉시 효과를 처리한다.
9. 버스트·영혼 고갈·전투 종료를 확인한다.
10. 전투가 계속되면 지속 효과와 대가를 등록하고 적 차례로 넘긴다.
11. 이후 명시된 발동 시점마다 능력과 대가를 처리한다.
12. 전투가 끝나면 활성 계약과 전투용 악마 덱·버린 더미를 제거한다.

## 5. 계약 가능 조건

다음 조건을 모두 만족해야 계약을 시작할 수 있다.

1. 전투가 진행 중이고 플레이어가 정상 행동을 할 수 있다.
2. 카드 효과, 체인지, 다른 계약 선택을 처리 중이지 않다.
3. 이번 전투의 기본 계약 횟수를 아직 사용하지 않았다.
4. 플레이어의 현재 영혼이 계약 비용보다 많다.
5. 악마 덱에서 서로 다른 물리 카드 3장을 제시할 수 있다.
6. 전투·런 상태가 계약 입력을 받을 수 있는 유효한 인스턴스다.

검사 실패는 영혼, 덱 순서, 사용 횟수, 차례와 UI 상태를 변경하지 않는다. 추가 계약을 허용하는 효과는 횟수 조건만 예외 처리하며 영혼 조건과 후보 3장 조건은 그대로 적용한다.

## 6. 악마 덱과 수명

### 6.1 시작 덱 프로토타입 결정

현재 원본 규칙에는 시작 악마 덱의 장수와 구성이 없다. 첫 구현에서는 아래 네 악마를 한 장씩 가진 **4장 시작 악마 덱**으로 고정한다.

- 사탄 1장
- 벨페고르 1장
- 마몬 1장
- 레비아탄 1장

이는 MVP 네 악마를 모두 플레이 테스트할 수 있게 하는 임시 구성이다. 중복 카드, 등장 가중치와 시작 덱 장수는 상점에서 악마 카드를 획득하는 흐름이 구현된 뒤 다시 조정한다.

### 6.2 런 덱과 전투 덱

- 런 상태는 보유한 악마 카드의 안정된 정의 키와 물리 ID를 보존한다.
- 전투 시작 시 런 악마 덱 전체를 복사해 전투용 드로우 더미를 만들고 섞는다.
- 선택한 카드는 활성 영역, 선택하지 않은 카드는 전투용 악마 버린 더미로 이동한다.
- 추가 계약 시 드로우 더미가 부족하면 활성 계약을 제외한 버린 더미를 섞어 보충한다.
- 서로 다른 물리 카드가 총 3장 미만이면 계약을 시작할 수 없다.
- 전투 종료 시 전투용 위치만 폐기하고 런 악마 덱의 소유권은 유지한다.

이 구조에서는 첫 전투에서 버린 악마도 다음 전투에 다시 등장할 수 있다. 전투 중 카드 이동을 런 영구 제거로 해석하지 않는다.

### 6.3 계약 임시 카드와 소유권

- 사탄의 권능과 바포메트의 오망성은 숫자를 합계에 포함하고 현재 전투의 덱·손·버린 더미를 순환한다. 전투 종료 시 모든 위치에서 제거한다.
- 파이몬이 추방한 카드는 현재 전투 동안 덱·손·버린 더미에서 제외하고 전투 종료 시 원래 소유권과 전투 시작 덱 구성을 복구한다. 플레이어 런 덱에는 제거를 반영하지 않는다.
- 벨리알이 가져온 카드는 현재 전투 동안 벨리알 계약 소유자의 카드다. 라운드 종료 시 그 소유자의 버린 더미로 보내고 전투 종료 시 원래 소유권으로 복구한다.
- 라운드 종료는 위 임시 상태를 제거하지 않는다. 하나의 상대와 벌이는 전투가 끝났을 때만 원상복구한다.
- 적이 같은 효과를 사용해도 방향만 반대로 적용하고 동일한 수명·복구 규칙을 따른다.

## 7. 공통 처리 우선순위

계약 효과와 카드 효과가 연쇄될 때 다음 순서를 사용한다.

1. 현재 실행 중인 단일 카드 이동·수치 변경을 끝낸다.
2. 명시적인 계약 대가와 직접 버스트를 처리한다.
3. 대가로 영혼이 0이 되면 즉시 전투 패배로 확정하고 남은 처리를 중단한다.
4. 행동 도중 숫자 버스트는 공개 카드 합으로 판정한다.
5. 라운드·전투가 끝났으면 남은 후속 효과를 중단한다.
6. 전투가 계속될 때만 다음 계약 훅이나 카드 효과를 처리한다.
7. 모든 효과와 선택이 끝났을 때만 차례를 넘긴다.

양측이 같은 효과 명령으로 동시에 버스트하면 전체 기획의 임시안대로 효과를 발생시킨 쪽이 패배한다. 계약 효과가 기존 카드 판정을 바꿀 때는 카드 이동을 복제하지 않고 카드 처리 결과 전후의 명시된 계약 훅만 사용한다.

## 8. 우선 악마 4종

### 8.1 벨페고르 — 첫 수직 구현 대상

#### 능력

플레이어가 자신의 정상 차례에 히트를 선택하면 드로우 전에 덱 위 카드 한 장을 자신만 확인한다. 다음 중 하나를 고른다.

- 확인한 카드를 공개 상태로 히트한다.
- 확인한 카드를 덱 맨 아래로 보내고 이번 차례를 넘긴다.

#### 대가

상대가 스탠드한 상태로 플레이어의 다음 정상 차례가 시작되면 `차례 종료 시 자동 스탠드`를 예약한다. 플레이어는 그 차례에 허용된 행동 하나를 수행할 수 있지만 행동 완료 뒤 자동으로 스탠드한다. 덱 아래로 보내고 넘기기를 골라도 자동 스탠드한다.

#### 프로토타입 결정

- 덱 위 카드 정보는 플레이어에게만 표시한다.
- 히트 외 카드 사용·체인지는 미리보기를 발동시키지 않는다.
- 덱이 비어 정상 히트를 할 수 없으면 기존 히트 불가 규칙을 따른다.

벨페고르는 계약 공통 기반, 전용 선택, 덱 이동, 지속 대가와 정보 은닉을 한 번에 검증할 수 있어 첫 수직 구현 대상으로 정한다.

#### DC-03 구현 상태

- 히트 입력은 실제 드로우 전에 `BelphegorTopCard` 선택으로 보류되며, 공개 옵션에는 카드 ID·숫자를 담지 않는다.
- 실제 카드 ID·정의·숫자는 소유자 전용 `PlayerDemonContractPreview`에만 제공한다.
- 그대로 히트와 덱 아래 이동 모두 미리 본 물리 ID를 다시 확인한 뒤 처리한다.
- 상대 스탠드 뒤 시작된 다음 정상 차례에는 행동 완료 시 자동 스탠드를 한 번 예약하고 라운드 종료 시 초기화한다.
- 카드 사용·체인지는 미리보기를 만들지 않으며, 공개 합 중간 버스트와 전체 합 최종 승부 규칙은 기존 경계를 유지한다.

### 8.2 마몬

#### 능력과 대가

- 계약 즉시 6면체 주사위를 한 번 굴리고 결과를 공개한다.
- 플레이어 정상 차례 시작 시 행동 전에 현재 값을 유지하거나 한 번 다시 굴릴 수 있다.
- 결과가 6이면 즉시 계약 효과 버스트로 처리한다.
- 양측 스탠드 후 최종 합계 비교 직전에 1~5 값을 자신의 최종 합계에 더할지 선택한다.
- 더한 결과가 21을 넘으면 플레이어가 숫자 버스트한다.

#### 프로토타입 결정

- 주사위 값은 중간 공개 카드 합에 포함하지 않는다.
- 한 차례에 재굴림은 최대 한 번이며 재굴림 결과를 되돌릴 수 없다.
- 값 적용 여부는 라운드마다 선택하고, 현재 주사위 값은 계약이 끝날 때까지 유지한다.
- 난수는 테스트 가능한 주입식 난수원을 사용한다.

#### DC-04 구현 상태

- 물리 계약마다 `MammonRuntimeState`가 현재 주사위 값과 차례·최종 선택 완료 여부를 보존한다.
- 계약 직후 굴림과 재굴림은 주입된 6면체 난수원을 사용하며, 새로 나온 6은 `ContractEffectBust`로 즉시 라운드를 끝낸다.
- 차례 시작의 유지·재굴림 선택은 정상 행동을 소비하지 않고, 양쪽 스탠드 뒤 적용·미적용 선택만 최종 비교를 보류한다.
- 적용 값은 카드 객체나 중간 공개 합을 바꾸지 않고 에이스 보정을 포함한 최종 합 계산에만 전달한다.

### 8.3 레비아탄

#### 능력과 대가

- 플레이어가 리볼버를 사용할 때 기존 숫자 추측 효과를 먼저 처리한다.
- 기존 효과가 상대를 버스트시키지 못했다면 상대 전체 손패 합이 22 이상인지 내부적으로 확인한다.
- 22 이상이면 레비아탄의 직접 효과로 상대를 버스트시킨다.
- 두 조건 모두 상대를 버스트시키지 못하면 플레이어 영혼 1을 잃는다.

#### 프로토타입 결정

- 레비아탄 판정은 카드 문구가 명시한 직접 효과이므로 비공개 카드를 포함한 전체 합을 내부에서 사용할 수 있다.
- UI는 상대 전체 합이나 비공개 숫자를 노출하지 않고 `레비아탄 발동 성공/실패`만 표시한다.
- 대가로 영혼이 0이 되면 즉시 전투 패배다.
- 리볼버 사용 자체가 승인되지 않았다면 능력과 대가 모두 발동하지 않는다.

이 결정은 일반적인 중간 숫자 버스트가 공개 합만 사용한다는 규칙을 바꾸지 않는다. 플레이 테스트에서 불공정하게 느껴지면 `상대 공개 합` 또는 별도 임계 조건으로 다시 설계한다.

#### DC-04 구현 상태

- 기존 리볼버 결과가 성공해 라운드가 끝나면 레비아탄은 추가 판정이나 대가를 실행하지 않는다.
- 실패 결과 뒤에만 상대 전체 합을 내부 조회한다. 22 이상이면 상대를 `ContractEffectBust`시키고, 아니면 플레이어 영혼을 정확히 1 감소시킨다.
- 영혼 1 대가가 0을 만들면 적 차례 없이 즉시 전투 패배로 종료한다.
- 공개 결과는 `Triggered`, `BustedTarget`, `PaidSoulCost`만 제공하며 상대 전체 합과 비공개 숫자를 저장하지 않는다.

### 8.4 사탄 — 기획 확인 뒤 구현

#### 원문 공통 부분

- 계약 중 플레이어는 스탠드할 수 없다.
- 계약 중 플레이어는 숫자·카드·계약 효과로 버스트되지 않는다.
- 계약 즉시 `사탄의 권능: 화염`을 자신의 공개 영역에 추가한다.
- 권능 카드의 숫자는 합계에 포함하고 현재 전투의 여러 라운드에서 순환한 뒤 전투 종료 시 제거한다.
- 카운터 종료 시 영혼 2를 잃는 위험이 있다.

#### 구현 전 확인 게이트

`rule.md`는 종말 카운트 증가와 6 이상 대가를, 전체 기획은 6에서 감소하는 종료 카운터를 사용한다. UI 예시에는 4턴이 적혀 있어 서로 일치하지 않는다. DC-06 전에 다음 중 하나를 이천서가 확정한다.

1. 6에서 0으로 감소하고 0에서 영혼 2를 잃은 뒤 계약 종료
2. 6에서 0으로 감소하고 영혼 2를 잃은 뒤 6으로 재설정해 전투 종료까지 반복
3. 플레이어 차례 종료마다 증가하고 6 이상부터 플레이어 차례 시작마다 영혼 2 손실

`사탄의 권능`은 숫자를 합계에 포함하고 전투 종료 시 제거하는 것으로 확정했다. 다만 화염·괴력이 한 물리 양면 카드인지, 뒤집힌 면의 사용 가능 상태가 어떻게 바뀌는지는 DC-06 전에 확정해야 한다. 이 결정 전에는 사탄을 부분 구현하지 않는다.

## 9. 화면과 피드백

### 9.1 계약 전 확인

- 현재 영혼 / 계약 비용 / 계약 후 영혼
- 이번 전투 계약 사용 여부
- 후보 3장을 제시할 수 없는 경우의 불가 사유
- `확인`과 `취소`

### 9.2 후보 선택

- 악마 이름과 일러스트 자리
- 즉시 효과
- 지속 능력
- 대가와 정확한 발동 시점
- 계약 후 남은 영혼
- 선택 버튼

후보가 공개된 뒤에는 취소 버튼을 제공하지 않는다. 다른 전투 행동, 카드 사용, 체인지와 재중복 입력은 잠근다.

### 9.3 활성 계약

- 활성 악마 이름
- 지속 효과와 대가 요약
- 사탄 카운터 또는 마몬 주사위처럼 변하는 값
- 최근 발동 결과
- 전투당 계약 사용 완료 표시

UI는 규칙을 다시 계산하지 않고 전투가 제공한 안전한 표시 스냅샷만 사용한다. 상대 비공개 카드와 덱 순서는 계약 UI에 포함하지 않는다.

## 10. 1차 범위와 제외 범위

### 10.1 포함

- 별도 런 악마 덱과 전투용 악마 덱·버린 더미
- 엄격한 영혼 비용 조건과 전투당 1회 제한
- 후보 3장과 필수 1장 선택
- 활성 계약 수명과 전투 종료 초기화
- 동일 악마의 별도 계약 인스턴스와 전투 종료 원상복구 기반
- 벨페고르, 마몬, 레비아탄, 사탄
- 카드 효과·버스트·영혼·차례와의 명시적 연동
- 독립 전투와 런 전투의 입력·표시·영혼 동기화
- 정보 은닉, 반복 재시작과 카드 소유권 테스트

### 10.2 제외

- 나머지 악마 8종과 오망성 전체 기능
- 루시퍼가 만드는 추가 계약
- 유물·강화에 의한 계약 횟수·비용 변경
- 상점 악마 카드 구매의 실제 구현
- 저장 파일과 덱 편집 UI
- 최종 일러스트·애니메이션·사운드·현지화
- 추가 계약을 실제로 발생시키는 루시퍼·유물과 악마별 세부 중첩 계산

적의 자율 계약은 DC-07의 2차 연결로 계획하되, 플레이어 계약 1차 완료를 막지 않는다.

## 11. 플레이 테스트 지표

| 지표 | 1차 목표 |
| --- | ---: |
| 계약 사용 런 비율 | 70% 이상 |
| 평균 계약 횟수 | 전투당 0.5~1.0회 |
| 계약 선택 시간 | 20초 이하 |
| 계약 대가로 인한 패배 | 원인 이해 가능 여부를 우선 기록 |
| 우선 악마 선택률 | 한 악마가 60%를 넘으면 후보 가치 재검토 |

계약 사용률이 낮으면 효과 수치부터 올리지 않고 비용, 계약 후 남은 영혼, 기대 이득과 대가 발동 시점을 먼저 더 명확히 표시한다.

## 13. 완료 조건

- 계약 비용과 전투당 제한을 우회할 수 없다.
- 후보 3장과 선택·버림·활성 영역에서 물리 카드가 복제되거나 유실되지 않는다.
- 네 우선 악마의 능력과 대가가 표시 문구와 같은 순서로 처리된다.
- 일반 중간 버스트는 공개 합, 최종 승부는 전체 합이라는 현행 규칙이 유지된다.
- 상대 비공개 정보와 실제 덱 순서가 화면·로그·적 AI에 새로 노출되지 않는다.
- 전투 종료·다음 전투·런 재시작에서 계약 상태가 남지 않는다.
- 사탄 권능·오망성·파이몬 추방·벨리알 탈취가 라운드를 넘어 유지되면서 런 덱에는 남지 않는다.
- 같은 악마 계약이 서로 다른 물리 ID와 런타임 상태를 가진다.
- 자동·수동 검증 결과와 실제 담당 파일을 진행 기록·AI 활용·팀 역할 문서에 남긴다.

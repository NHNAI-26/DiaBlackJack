## 4. 현재 코드와 미구현 영역

### 4.1 이미 가능한 것

- 프로필 안전 미리보기 조회
- 키 하나로 실제 전용 적 덱·영혼·정책 생성
- 엘리트·보스 높은 등급 보상
- 보스 구간·예고 표시 모델 생성
- 진행 씬과 전투 씬 왕복

### 4.2 아직 구현되지 않은 것

- 일반 전투 전 후보 2명 생성
- `OpponentSelection` 진행 상태
- 후보 집중·확정 입력과 선택 화면
- 선택된 프로필을 현재 스테이지에서 확정하는 세션 API
- 일반 적 추론 확률 표시 모델
- 기존 엘리트·보스 표시 모델의 실제 전투 View 연결
- 상대 선택→전투→보상→다음 선택 반복 흐름

### 계획과 실제 차이

- 최초 분업안에서는 HONG이 상대 선택·런 진행을 함께 담당할 예정이었다.
- HONG이 해당 게임 코드에 착수하지 않은 상태에서 이천서가 상대 선택·전투 UI 통합까지 담당하기로 합의했다.
- 랜덤 이벤트와 정식 런 진행 확장은 HONG에게 남겨 작업 집중을 방지했다.
- 전투 정보 UI는 EP-06 전투 변환과 분리된 별도 EUI 작업으로 관리한다.

### 남은 위험

- `OpponentSelection`을 기존 `RunProgress` 상태와 연결할 때 진행 테스트 수정 범위가 커질 수 있다.
- 일반 추론 확률 UI는 현재 공개 표시 타입이 없어 안전한 스냅샷이 추가로 필요하다.
- 보스 `CurrentDisplay`를 읽기 위해 정책 객체를 UI에 직접 노출하면 결합도가 높아질 수 있으므로 Factory 경계를 우선한다.
- 두 후보 카드와 기존 보상 UI가 같은 View를 사용하므로 작은 해상도에서 세로 공간을 검증해야 한다.
- 엘리트 35%는 밸런스 로그 전의 임시 값이다.

## 1. 문서 목적

지도 없이 `전투 → 이벤트 → 전투 → 이벤트 → 보스`로 진행하는 정식 런을 구현할 기획 기준과 분업 경계를 확정한다. 최신 `rule.md`에 따라 이 문서에서 말하는 이벤트는 첫 개발에서는 **상점으로 고정**한다. 무작위 사건은 첫 정식 런이 안정된 뒤 확장한다.

기획 결정과 최종 통합 승인은 이천서가 담당한다. HONG은 이 문서를 기준으로 런 골드, 전투 후 상점, 정식 순서 연결과 진행 화면을 구현한다. 계획 역할은 실제 구현 완료 기록과 구분한다.

## 3. 정식 플레이 순서

사용자가 말한 `전투 → 이벤트 → 전투 → 이벤트 → 보스`를 실제 화면 단위로 풀면 다음과 같다.

```text
런 시작(영혼 12, 골드 0)
→ 상대 후보 2명 중 1명 선택
→ 첫 번째 전투
→ 카드 보상 선택 또는 건너뛰기
→ 선택한 적에 따른 골드 지급
→ 첫 번째 상점
→ 상대 후보 2명 중 1명 선택
→ 두 번째 전투
→ 카드 보상 선택 또는 건너뛰기
→ 선택한 적에 따른 골드 지급
→ 두 번째 상점
→ 고정 최종 보스 전투
→ 높은 등급 카드 보상 선택 또는 건너뛰기
→ 보스 골드 지급과 최종 결과 기록
→ 런 승리
```

- 일반 전투 후보는 `일반+일반` 또는 `일반+엘리트`다.
- 보스는 후보 선택 없이 고정 진입한다.
- 카드를 건너뛰어도 승리 골드는 지급한다.
- 전투 사이에 자동 영혼 회복은 없다. 상점에서 골드를 지불한 휴식만 회복한다.
- 플레이어 영혼이 전투 중 0이 되면 카드·골드 보상과 상점 없이 런 패배다.
- 보스 보상 뒤에는 상점이 없다.
- 분기 지도는 만들지 않는다.

## 4. 포함·제외 범위

### 4.1 포함

- 런 골드 보유·지급·소비·재시작 초기화
- 적 프로필별 승리 골드
- 카드 보상 선택/건너뛰기 뒤 골드 1회 지급
- 두 일반 전투 뒤 상점 1회씩, 총 2회
- 상점의 카드 구매, 카드 제거, 유료 휴식과 무료 나가기
- 상점 완료 뒤 다음 상대 선택 또는 고정 보스 진입
- 기존 `StageTest` 진행 화면과 `CoreLoopTest` 공용 전투 화면 재사용
- 잘못된 ID·중복 구매·중복 지급·오래된 입력의 무변경 거부
- 런 승리·패배·재시작과 반복 검증

### 4.2 제외

- 무작위 사건과 상점/사건 가중치
- 독립 휴식 노드
- 카드 강화와 유물 구매
- 상점 재입고·새로고침과 플레이어 카드 판매
- 분기 지도와 경로 선택
- 저장·불러오기와 메타 성장
- 전투 규칙·적 AI·카드 효과·보상 풀 변경
- 새 이벤트 전용 씬과 최종 아트·사운드·연출

카드 강화는 숫자가 아니라 효과 강화가 필요해 현재 카드 정의·효과 계약의 별도 작업이므로 RF에서 가짜 단계나 플래그를 만들지 않는다.

## 5. 골드 규칙

### 5.1 공통

- 런 시작 골드는 0이다.
- 골드는 현재 런 동안 유지하고 상점에서만 사용한다.
- 승리한 실제 `BattleProfileKey` 하나로 지급량을 조회한다.
- 골드는 카드 보상 선택 또는 건너뛰기가 성공한 순간 정확히 한 번 지급한다.
- 패배, 잘못된 보상 입력, 중복 입력에는 지급하지 않는다.
- 보스 골드는 최종 런 결과에는 포함하지만 사용할 상점은 열지 않는다.
- 재시작하면 골드 0과 최초 상점 상태로 돌아간다.

### 5.2 프로토타입 지급량

| 프로필 | 등급·강함 | 골드 |
| --- | --- | ---: |
| `gunslinger` | 일반, 최대 영혼 3 | 3 |
| `cultist` | 일반, 최대 영혼 3 | 3 |
| `trickster` | 일반, 최대 영혼 4 | 4 |
| `enforcer` | 엘리트, 최대 영혼 5 | 6 |
| `final-boss` | 보스, 최대 영혼 7 | 10 |

이는 첫 경제 검증용 임시값이다. 엘리트가 모든 일반 적보다 많이 지급하고, 같은 등급에서는 강한 적이 같거나 더 많이 지급한다.

## 6. 상점 규칙

### 6.1 상점 생성과 유지

- 상점은 첫째·둘째 일반 전투의 카드·골드 보상 완료 뒤 정확히 한 번 연다.
- 한 번 만든 상품과 가격은 화면 재진입으로 바뀌지 않는다.
- 상점은 안정적인 `ShopOfferId`와 방문 번호 0 또는 1을 가진다.
- 판매 카드는 서로 다른 상품 슬롯 5개로 생성하며 각 슬롯의 재고는 1개다.
- 플레이어는 가능한 서비스를 원하는 순서로 이용한 뒤 `상점 나가기`를 선택한다.
- 카드를 구매하면 해당 상품만 사라지고 구매하지 않은 카드와 서비스 상태는 유지된다.
- 상품 새로고침과 재입고는 제공하지 않는다.
- 나가기는 무료이며 항상 가능하다.
- 나간 상점에는 다시 들어갈 수 없다.
- 두 번째 상점에서 나가면 상대 선택 없이 보스 전투로 간다.

### 6.2 프로토타입 상품·서비스

| 항목 | 가격 | 효과 | 방문당 제한 |
| --- | --- | --- | --- |
| 판매 카드 | 미정 | 서로 다른 카드 5장을 개별 상품으로 제시하고 선택한 카드를 런 덱에 추가 | 상품당 재고 1개 |
| 카드 제거 | 미정 | 현재 런 덱에서 선택한 카드 1장 제거 | 1회 |
| 휴식 | 미정 | 미정 수치만큼 영혼 회복, 최대 영혼 초과 불가 | 1회 |
| 상점 나가기 | 0골드 | 상점을 끝내고 다음 진행으로 이동 | 항상 |

- 판매 카드 5장은 상점 입장 시 한 번 생성하고 재추첨하지 않는다.
- 구매한 카드 슬롯은 즉시 빈 재고로 바뀌며 다시 채우지 않는다.
- 가격은 판매 카드, 카드 제거, 휴식과 향후 서비스 전체를 대상으로 추후 확정한다.
- 카드 제거 뒤에도 런 덱은 최소 1장을 유지한다.
- 제거한 카드 ID는 같은 런에서 다시 사용하지 않는다.
- 회복할 영혼이 0이어도 휴식은 비활성화해 골드를 낭비하지 않는다.
- 각 서비스는 골드·대상·방문당 제한을 모두 검증한 뒤 비용과 효과를 한 번에 적용한다.
- 골드가 부족하거나 대상이 잘못되면 아무것도 바뀌지 않는다.

## 7. 화면과 입력

| 상황 | 화면 | 허용 입력 | 다음 흐름 |
| --- | --- | --- | --- |
| 상대 선택 | `StageTest` | 후보 집중·확정 | `CoreLoopTest` |
| 전투 | `CoreLoopTest` | 기존 전투 행동 | `StageTest` 보상 또는 패배 |
| 카드 보상 | `StageTest` | 카드 선택·건너뛰기 | 골드 결과 뒤 같은 씬 상점 |
| 상점 | `StageTest` | 구매·제거·휴식·나가기 | 다음 상대 선택 또는 보스 |
| 보스 보상 | `StageTest` | 카드 선택·건너뛰기 | 런 승리 |
| 런 종료 | `StageTest` | 재시작 | 첫 상대 선택 |

상점마다 새 공간으로 이동하지 않는다. 정식 화면에서는 전투와 상점이 고정 카메라의 같은 서부 시대 술집 테이블을 공유한다. 전투가 끝나면 적과 전투 오브젝트를 정리하고, 적이 있던 자리에 상인을 배치해 판매 카드와 서비스 상품을 테이블에 펼친다. 상점 중에는 상대·보상·`NEXT STAGE`·재시작 입력을 숨기거나 비활성화하며 현재 골드, 가격, 이용 가능 여부와 최근 거래 결과를 표시한다.

현재 `StageTest`와 `CoreLoopTest` 분리는 개발·검증용 씬 경계다. 최종 프레젠테이션에서는 2.5D 월드 오브젝트와 화면 UI를 조합하되, 런·전투 규칙 계층은 기존 공개 API를 통해 연결한다.

## 9. 완료 조건

- 실제 순서가 `전투 보상·골드 → 상점 → 전투 보상·골드 → 상점 → 보스 보상·골드 → RunVictory`다.
- 카드 건너뛰기에도 골드가 한 번 지급된다.
- 두 상점에서 판매 카드 5장, 구매 후 개별 재고 제거, 새로고침 없음, 1회 휴식과 나가기가 동작하고 결과가 다음 전투에 반영된다.
- 두 일반 전투에는 상대 선택이 있고 보스에는 선택이 없다.
- 패배 시 보상·골드·상점·다음 전투가 생성되지 않는다.
- 재시작 시 영혼·덱·골드·상대·상점 상태가 초기화된다.
- 신규 RF 테스트와 기존 CoreLoop·StageProgression 전체 테스트가 통과한다.
- 실제 두 씬 왕복과 1280×720·1920×1080 상점 화면이 통과한다.
- 게임 관련 Unity Console Error·Exception이 없다.

## 1. 계획 목적

적 전투 프로필을 데이터·정책 경계, 공개 관측, 일반 적, 엘리트, 보스, 상대 선택·진행 통합과 전체 검증 순서로 구현한다.

이천서가 CoreLoop의 적 프로필·행동 정책·카드 효과·전투 생성과 검증을 담당한다. 상대 후보 생성·선택 UI와 아트 리소스는 다른 담당자의 작업으로 유지한다.

## 2. 단계 운영 원칙

1. 단계 시작 전에 기획서와 개발 명세의 범위를 확인한다.
2. 기존 행동과 공정성 불변 조건을 테스트로 먼저 고정한다.
3. 한 단계에서는 한 책임만 확장한다.
4. 구현되지 않은 계약·카드를 임시 성공 처리하지 않는다.
5. 관련 CoreLoop·StageProgression 테스트 뒤 전체 EditMode를 실행한다.
6. 통합 단계 전까지 상대 선택 담당 파일과 씬을 수정하지 않는다.
7. 단계마다 AI 활용 기술서와 이천서 역할 기록을 갱신한다.
8. 사용자가 명시적으로 요청한 경우에만 스테이징·커밋·푸시한다.

## 4. EP-00 — 구현 기준 확정

### 목표

이천서의 전투 담당 범위와 상대 선택·아트 담당자의 경계를 확정하고, 다섯 적의 프로필·공정성·행동·카드·전투 생성·보상 연결 기준을 작성한다.

### 완료 내용

- 일반 적 3종, 엘리트 1종과 보스 1종의 키·영혼·성향을 확정했다.
- 적 AI의 허용·금지 입력과 결정적 행동 평가 원칙을 명시했다.
- 현재 구현 카드와 미구현 계약·거짓말 탐지기를 구분했다.
- 상대 선택은 `BattleProfileKey`만 전달하고 실제 전투 설정은 프로필 카탈로그가 소유하도록 정했다.
- 기존 `SimpleEnemyPolicy` 호환, 정책 인터페이스와 적 카드 실행 원자성을 명세했다.
- EP-01~EP-06 구현·검증 순서와 완료 게이트를 작성했다.
- 코드·테스트·씬·패키지·외부 에셋은 변경하지 않았다.

### 검증

- `rule.md`와 전체 게임 기획의 AI·적·보스·보상 규칙 대조
- `CoreLoopBattle`, `SimpleEnemyPolicy`, 카드 카탈로그·효과 처리기와 `StageBattleFactory` 구조 대조
- 문서 4종의 작업 ID·책임·단계·완료 게이트 대조
- Markdown 링크와 Git diff 공백 검사
- 문서 전용 단계이므로 Unity 테스트 미실행

### 추천 커밋 제목

## 5. EP-01 — 프로필·정책 기반

### 목표

다섯 적을 한 카탈로그에서 조회하고 기존 단순 정책을 깨지 않는 교체 가능 정책 경계를 만든다.

### 완료 내용

1. `EnemyGrade`, `EnemyCombatProfile`, 읽기 전용 미리보기를 작성한다.
2. 다섯 프로필 키·영혼·덱·정책 키를 명시적으로 등록한다.
3. `IEnemyBehaviorPolicy`와 결정 결과의 최소 타입을 작성한다.
4. `SimpleEnemyPolicy`의 16/17 규칙을 새 경계에 맞추되 기존 생성자를 보존한다.
5. 프로필 기반 적 덱·정책 설정을 결정적으로 생성한다.
6. EP01 단위 테스트와 기존 CoreLoop 회귀를 실행한다.

프로필별 특수 행동을 앞당겨 구현하지 않기 위해 다섯 프로필의 정책 키는 EP-01에서 모두 기존 16/17 단순 정책을 가리킨다. `EnemyObservation`은 인터페이스 호환에 필요한 적 자신의 손패 합계만 가진 최소 형태이며, 플레이어 공개 정보·행동 후보·적 카드 가용성은 EP-02에서 확장한다.

### 완료 게이트

- 다섯 프로필의 값과 모든 생성 실패 조건이 테스트된다.
- 상대 선택용 미리보기가 내부 정책·덱 순서를 노출하지 않는다.
- 기존 단순 적 전투 결과와 공개 API가 유지된다.
- CoreLoop와 전체 EditMode가 통과한다.

### 검증

- EP-01 신규 테스트 12/12 통과
- CoreLoop EditMode 134/134 통과
- 전체 EditMode 199/199 통과
- Unity 컴파일 오류와 최종 Console Error 0
- 씬·프리팹·패키지·외부 에셋 변경 없음

### 협업 해제 조건

EP-01의 프로필 키와 미리보기 계약이 확정되면 상대 선택 담당자는 해당 읽기 모델을 기준으로 후보·선택 구현을 병렬 진행할 수 있다.

### 추천 커밋 제목

`적 프로필을 안정적인 키와 교체 가능한 행동 정책으로 구성할 기반을 마련`

## 6. EP-02 — 공개 관측과 공통 행동

### 목표

적 AI가 볼 수 있는 정보와 실행 가능한 행동을 분리하고 적 카드 사용을 CoreLoop의 승인된 행동으로 연결한다.

### 완료 내용

1. 플레이어 비공개 카드 객체·값·물리 ID와 덱 순서를 담지 않는 `EnemyObservation`을 작성했다.
2. 히트·스탠드·폴드·카드 사용과 선택형 효과 옵션을 `EnemyActionCandidate`로 만들고 정책 점수·사유를 결정 결과에 보존했다.
3. 라운드와 결정 순번으로 만든 시드와 고정 동점 규칙으로 같은 관측에서 같은 결정을 반환하게 했다.
4. `EnemyDecisionValidator`로 정책 반환값을 실행 직전에 다시 검증하고, 무효 결정은 상태 변경 없이 한 번 재평가한 뒤 안전 후보로 대체했다.
5. 카드 효과 컨텍스트를 플레이어 전용에서 행위자·상대 구조로 일반화해 적이 자기 카드와 덱을 사용하고 플레이어를 올바른 대상으로 삼게 했다.
6. 적 선택형 카드 효과는 별도 UI 상태를 만들지 않고 적 차례 안에서 유효 옵션 후보로 결정·완료하게 했다.
7. 폴드는 양측 공통 행동으로 보고 적 폴드도 적 영혼 1을 소모하고 즉시 라운드를 끝내는 임시 규칙으로 구현했다.

### 완료 게이트

- 비공개 정보 접근이 타입과 테스트로 차단된다.
- 잘못된 결정이 상태·손패·덱·영혼을 바꾸지 않는다.
- 카드 사용 뒤 라운드·전투 종료가 중복 처리되지 않는다.
- 기존 플레이어 카드 사용 회귀가 유지된다.

### 검증

- EP-02 신규 EditMode 8/8 통과
- CoreLoop EditMode 142/142 통과
- 전체 EditMode 207/207 통과
- 전체 에셋 강제 갱신 뒤 Unity 컴파일 오류와 Console Error 0
- 기존 카드 처리기를 복제하지 않고 행위자·상대 컨텍스트로 공용화
- 씬·프리팹·패키지·외부 에셋과 다른 담당 파일 변경 없음

### 추천 커밋 제목

`적 AI가 공개 정보만으로 유효한 전투 행동을 결정하도록 경계를 분리`

## 7. EP-03 — 일반 적 3종

### 목표

총잡이·광신도·사기꾼이 같은 공통 정책 경계를 사용하면서 다른 전투 성향을 보이게 한다.

### 완료 내용

1. 플레이어 덱의 전체 공개 구성에서 현재 공개 손패·버림패를 제외해 비공개 숫자 확률을 계산하고, 실제 비공개 객체와 덱 순서는 사용하지 않게 했다.
2. 총잡이가 최상위 확률 50% 이상에서만 리볼버를 사용하고 가장 가능성 높은 숫자를 선언하게 했다.
3. 광신도가 합계 18까지 위험한 히트를 우선하고 19부터 스탠드를 우선하게 했다.
4. 미구현 악마 계약은 카드 카탈로그·프로필 덱·행동 후보에 추가하지 않았다.
5. 사기꾼이 수정 구슬을 일반 행동보다 우선하고 버스트하지 않는 가장 높은 합계의 카드를 가져오게 했다.
6. 미구현 거짓말 탐지기는 카드 카탈로그·프로필 덱·행동 후보에 추가하지 않았다.
7. 일반 적 세 프로필을 각 전용 정책 키에 연결하고 엘리트·보스는 기존 단순 정책을 유지했다.

### 완료 게이트

- 세 프로필이 의도된 차이를 보인다.
- 리볼버 결과에 실제 비공개 값이 유출되지 않는다.
- 수정 구슬의 덱 순서·카드 소유권이 보존된다.
- 계약·거짓말 탐지기를 구현 완료로 오기록하지 않는다.

### 검증

- EP-03 신규 EditMode 10/10 통과
- CoreLoop EditMode 152/152 통과
- 전체 EditMode 217/217 통과
- 전체 에셋 강제 갱신 뒤 Unity 컴파일 오류 0
- 공개 구성 추론 확률 합계 100%, 확률·숫자 정렬과 실제 비공개 값 비노출 확인
- 총잡이 리볼버 성공, 사기꾼 수정 구슬 소유권·반환 순서, 같은 관측의 세 정책 차이 확인
- UI·씬·프리팹·패키지·외부 에셋과 다른 담당 파일 변경 없음

### 추천 커밋 제목

`일반 적 세 종이 같은 규칙 안에서 서로 다른 위험과 정보 전략을 사용하게 구현`

## 8. EP-04 — 집행관 엘리트

### 목표

집행관의 방해 카드와 한 단계 깊은 행동 평가를 구현하고 엘리트 승리를 높은 등급 보상에 연결한다.

### 완료 내용

1. 집행관 최대 영혼 5를 유지하면서 전용 `EnforcerEnemyPolicy`와 정책 키를 등록하고 보스는 단순 정책을 유지했다.
2. 위협용 해머의 선택 옵션이 플레이어 공개 카드만 안전하게 관측하도록 이관하고 가장 높은 숫자 대상을 선택하게 했다.
3. 플레이어 스탠드 시 해머를 우선하고, 보위 나이프는 공개 손패와 구성 추론으로 비공개 카드·강제 드로우 뒤 버스트 가능성을 결정적으로 근사했다.
4. 확률과 비공개 값을 노출하지 않는 엘리트용 상위 숫자 2개·낮음/보통/높음 신뢰도 모델을 제공했다.
5. 기존 전투 보상 경계를 재사용해 집행관 승리를 `HighGrade` 선택 대기와 `StageCleared`로 연결하는 통합 테스트를 작성했다.

### 검증

- EP-04 신규 EditMode 11/11 통과
- CoreLoop EditMode 163/163 통과
- 전체 EditMode 228/228 통과
- Unity 컴파일 오류와 Console Error/Warning 0
- UI·씬·프리팹·패키지·외부 에셋 변경 없음

### 완료 게이트

- 카드 대상 방향과 상태 종료가 정확하다.
- 플레이어 비공개 정보가 표시 모델에 없다.
- 엘리트 전투 결과와 보상 등급이 자동·통합 테스트를 통과한다.

### 추천 커밋 제목

`집행관의 방해 행동이 엘리트 전투와 높은 등급 보상으로 이어지게 연결`

## 9. EP-05 — 최종 보스

### 목표

보스가 최대 영혼 7을 기준으로 세 구간의 행동 성향을 바꾸되 기존 전투·런 승리 규칙을 유지하게 한다.

### 완료 내용

1. 보스 프로필만 `FinalBossEnemyPolicy`와 전용 정책 키에 연결하고 최대 영혼 7을 강제했다.
2. 7~5 생존, 4~3 압박, 2~1 처형 구간 판정과 구간별 히트·카드 점수를 구현했다.
3. 처형 구간은 확신 50% 이상의 리볼버를 우선하고, 그 외에는 보위 나이프를 골라 플레이어 행동 한 번 전에 범주만 예고한다.
4. `BossCombatDisplayModel`에는 현재 구간·추론 방향·신뢰도·예고 범주만 두고 실제 비공개 값·원시 확률·카드 ID·선언 숫자를 제외했다.
5. 기존 선택형 카드 처리기를 재사용해 수정 구슬 최고 안전 카드, 해머 최고 공개 대상과 리볼버 최상위 숫자 선택을 구현했다.
6. 별도 승리 조건 없이 영혼 0 종료, `HighGrade` 보상 선택·건너뛰기 뒤 `RunVictory`와 재시작 영혼 7 복구를 통합 검증했다.

### 검증

- EP-05 신규 EditMode 16/16 통과
- CoreLoop EditMode 179/179 통과
- 전체 EditMode 244/244 통과
- Unity 전체 에셋 갱신·컴파일 뒤 Console Error 0
- UI·씬·프리팹·패키지·외부 에셋 변경 없음

### 완료 게이트

- 세 구간 경계가 정확하고 상태 고착이 없다.
- 별도 승리 조건 없이 기존 전투 종료 규칙을 사용한다.
- 보스 선택·건너뛰기와 재시작 회귀가 유지된다.

### 추천 커밋 제목

`최종 보스가 영혼 구간에 따라 압박을 높이면서 기존 런 승리 규칙을 유지하게 구현`

## 10. EP-06 — 선택 전투 통합과 마감

### 목표

상대 선택 담당의 프로필 키를 실제 전투와 보상으로 연결하고 다섯 프로필의 반복 안정성을 검증한다.

### 작업

1. 선택된 `BattleProfileKey`를 전투 설정으로 변환한다.
2. 표시 미리보기와 실제 적 등급·영혼·보상이 일치하는지 확인한다.
3. 다섯 프로필의 생성·전투·종료·재시작을 반복한다.
4. CoreLoop, StageProgression, 카드 사용, 전투 보상과 전체 EditMode를 실행한다.
5. 실제 일반·엘리트·보스 흐름, 양쪽 씬과 Console을 확인한다.
6. 문서 4종, README, AI 활용과 이천서 역할 기록을 마감한다.

### 완료 게이트

- 알 수 없는·중복·오염된 프로필 입력이 원자적으로 거절된다.
- 선택한 적과 실제 전투·보상 결과가 일치한다.
- 반복에서 카드·덱·영혼·정책 상태가 다음 전투에 누출되지 않는다.
- 전체 자동·실제 검증이 통과한다.

### 추천 커밋 제목

`선택된 상대의 전투 성향과 보상 결과를 반복 검증해 적 프로필 1차 범위를 마감`

### 실제 완료 결과

- `StageDefinition.CreateForEnemyProfile(...)`로 프로필 키에서 스테이지 최대 영혼을 파생하고 등급과 스테이지 종류를 검증했다.
- `StageBattleFactory`가 선택 키를 적 전용 10장 덱·최대 영혼·새 정책 인스턴스로 변환하도록 연결했다.
- `StageProgressionSession`의 기본 보상 등급이 같은 프로필 미리보기에서 파생되도록 했고, 기존 최종 보스 보상 뒤 런 승리 규칙은 유지했다.
- 키 없는 기존 스테이지의 표준 20장 덱·`SimpleEnemyPolicy` 경로를 보존했다.
- 다섯 프로필 생성과 50회 상태 격리, 엘리트 보상, 보스 보상·재시작을 신규 16개 테스트로 검증했다.
- StageProgression 81/81, CoreLoop 179/179, 전체 EditMode 260/260, 실제 `StageTest` 프로필 런과 양쪽 씬·Console을 검증했다.
- 상대 후보 생성·선택 UI는 다른 담당 범위로 남기고, 해당 UI가 호출할 공개 변환 경계까지만 완료했다.

## 14. 커밋 원칙

- 단계별로 구현·테스트·문서 기록을 함께 커밋한다.
- 제목은 변경 파일보다 해결한 목적을 설명한다.
- Lore 본문에 제약, 거절한 대안, 검증과 미검증 항목을 기록한다.
- Unity `.meta`는 대응 C# 파일과 함께 관리한다.
- 다른 담당자의 아트·상대 선택 파일을 한 EP 커밋에 섞지 않는다.
- 사용자가 요청하기 전에는 스테이징하거나 커밋하지 않는다.

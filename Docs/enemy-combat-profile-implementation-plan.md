# 적 전투 프로필 시스템 구현 계획서

> 프로젝트: DiaBlackJack  
> 계획·구현 책임자: 이천서  
> 버전: v0.1  
> 현재 단계: EP-00 문서화 완료  
> 다음 단계: EP-01 프로필·정책 기반  
> 최종 갱신: 2026-07-20

## 1. 계획 목적

적 전투 프로필을 데이터·정책 경계, 공개 관측, 일반 적, 엘리트, 보스, 상대 선택·진행 통합과 전체 검증 순서로 구현한다.

이천서가 CoreLoop의 적 프로필·행동 정책·카드 효과·전투 생성과 검증을 담당한다. 상대 후보 생성·선택 UI와 아트 리소스는 다른 담당자의 작업으로 유지한다.

## 2. 단계 운영 원칙

1. 단계 시작 전에 기획서와 개발 명세의 범위를 확인한다.
2. 기존 행동과 공정성 불변 조건을 테스트로 먼저 고정한다.
3. 한 단계에서는 한 책임만 확장한다.
4. 구현되지 않은 계약·카드를 임시 성공 처리하지 않는다.
5. 관련 CoreLoop·StageProgression 테스트 뒤 전체 EditMode를 실행한다.
6. 통합 단계 전까지 상대 선택 담당 파일과 씬을 수정하지 않는다.
7. 단계마다 AI 활용 기술서와 이천서 역할 기록을 갱신한다.
8. 사용자가 명시적으로 요청한 경우에만 스테이징·커밋·푸시한다.

## 4. EP-00 — 구현 기준 확정

### 목표

이천서의 전투 담당 범위와 상대 선택·아트 담당자의 경계를 확정하고, 다섯 적의 프로필·공정성·행동·카드·전투 생성·보상 연결 기준을 작성한다.

### 완료 내용

- 일반 적 3종, 엘리트 1종과 보스 1종의 키·영혼·성향을 확정했다.
- 적 AI의 허용·금지 입력과 결정적 행동 평가 원칙을 명시했다.
- 현재 구현 카드와 미구현 계약·거짓말 탐지기를 구분했다.
- 상대 선택은 `BattleProfileKey`만 전달하고 실제 전투 설정은 프로필 카탈로그가 소유하도록 정했다.
- 기존 `SimpleEnemyPolicy` 호환, 정책 인터페이스와 적 카드 실행 원자성을 명세했다.
- EP-01~EP-06 구현·검증 순서와 완료 게이트를 작성했다.
- 코드·테스트·씬·패키지·외부 에셋은 변경하지 않았다.

### 검증

- `rule.md`와 전체 게임 기획의 AI·적·보스·보상 규칙 대조
- `CoreLoopBattle`, `SimpleEnemyPolicy`, 카드 카탈로그·효과 처리기와 `StageBattleFactory` 구조 대조
- 문서 4종의 작업 ID·책임·단계·완료 게이트 대조
- Markdown 링크와 Git diff 공백 검사
- 문서 전용 단계이므로 Unity 테스트 미실행

### 추천 커밋 제목

`다섯 적의 전투 성향과 상대 선택 연동 경계를 구현 기준으로 확정`

## 5. EP-01 — 프로필·정책 기반

### 목표

다섯 적을 한 카탈로그에서 조회하고 기존 단순 정책을 깨지 않는 교체 가능 정책 경계를 만든다.

### 작업

1. `EnemyGrade`, `EnemyCombatProfile`, 읽기 전용 미리보기를 작성한다.
2. 다섯 프로필 키·영혼·덱·정책 키를 명시적으로 등록한다.
3. `IEnemyBehaviorPolicy`와 결정 결과의 최소 타입을 작성한다.
4. `SimpleEnemyPolicy`의 16/17 규칙을 새 경계에 맞추되 기존 생성자를 보존한다.
5. 프로필 기반 적 덱·정책 설정을 결정적으로 생성한다.
6. EP01 단위 테스트와 기존 CoreLoop 회귀를 실행한다.

### 완료 게이트

- 다섯 프로필의 값과 모든 생성 실패 조건이 테스트된다.
- 상대 선택용 미리보기가 내부 정책·덱 순서를 노출하지 않는다.
- 기존 단순 적 전투 결과와 공개 API가 유지된다.
- CoreLoop와 전체 EditMode가 통과한다.

### 협업 해제 조건

EP-01의 프로필 키와 미리보기 계약이 확정되면 상대 선택 담당자는 해당 읽기 모델을 기준으로 후보·선택 구현을 병렬 진행할 수 있다.

### 추천 커밋 제목

`적 프로필을 안정적인 키와 교체 가능한 행동 정책으로 구성할 기반을 마련`

## 6. EP-02 — 공개 관측과 공통 행동

### 목표

적 AI가 볼 수 있는 정보와 실행 가능한 행동을 분리하고 적 카드 사용을 CoreLoop의 승인된 행동으로 연결한다.

### 작업

1. 플레이어 비공개 값을 담을 수 없는 `EnemyObservation`을 작성한다.
2. 히트·스탠드·폴드·카드 사용 후보와 점수 결과를 정의한다.
3. 같은 관측·시드에서 같은 결정을 반환하게 한다.
4. 결정과 실행 사이에 재검증 경계를 둔다.
5. 적 카드의 행위자·대상·소유권과 효과 종료를 처리한다.
6. 디버그용 후보 점수·사유 로그를 추가한다.

### 완료 게이트

- 비공개 정보 접근이 타입과 테스트로 차단된다.
- 잘못된 결정이 상태·손패·덱·영혼을 바꾸지 않는다.
- 카드 사용 뒤 라운드·전투 종료가 중복 처리되지 않는다.
- 기존 플레이어 카드 사용 회귀가 유지된다.

### 추천 커밋 제목

`적 AI가 공개 정보만으로 유효한 전투 행동을 결정하도록 경계를 분리`

## 7. EP-03 — 일반 적 3종

### 목표

총잡이·광신도·사기꾼이 같은 공통 정책 경계를 사용하면서 다른 전투 성향을 보이게 한다.

### 작업

1. 총잡이의 자동 권총 추론·확신 임계값을 구현한다.
2. 광신도의 공격·위험 선호 가중치를 구현한다.
3. 미구현 계약 행동은 후보에서 제외한다.
4. 사기꾼의 수정 구슬 선택과 정보 우위 가중치를 구현한다.
5. 미구현 거짓말 탐지기는 덱과 후보에 넣지 않는다.
6. 적별 고정 관측 시나리오와 전투 카드 효과를 검증한다.

### 완료 게이트

- 세 프로필이 의도된 차이를 보인다.
- 자동 권총 결과에 실제 비공개 값이 유출되지 않는다.
- 수정 구슬의 덱 순서·카드 소유권이 보존된다.
- 계약·거짓말 탐지기를 구현 완료로 오기록하지 않는다.

### 추천 커밋 제목

`일반 적 세 종이 같은 규칙 안에서 서로 다른 위험과 정보 전략을 사용하게 구현`

## 8. EP-04 — 집행관 엘리트

### 목표

집행관의 방해 카드와 한 단계 깊은 행동 평가를 구현하고 엘리트 승리를 높은 등급 보상에 연결한다.

### 작업

1. 집행관 최대 영혼 5와 엘리트 정책을 등록한다.
2. 위협용 해머·군용 나이프의 적 행위자 경로를 검증한다.
3. 플레이어 스탠드·강제 히트 효과와 후속 상태를 평가한다.
4. 엘리트 추론 표시용 상위 2개·신뢰도 모델을 제공한다.
5. 승리 결과가 `HighGrade`와 `StageCleared`로 연결되게 한다.

### 완료 게이트

- 카드 대상 방향과 상태 종료가 정확하다.
- 플레이어 비공개 정보가 표시 모델에 없다.
- 엘리트 전투 결과와 보상 등급이 자동·통합 테스트를 통과한다.

### 추천 커밋 제목

`집행관의 방해 행동이 엘리트 전투와 높은 등급 보상으로 이어지게 연결`

## 9. EP-05 — 최종 보스

### 목표

보스가 최대 영혼 7을 기준으로 세 구간의 행동 성향을 바꾸되 기존 전투·런 승리 규칙을 유지하게 한다.

### 작업

1. 7~5·4~3·2~1 구간 판정을 작성한다.
2. 구간별 생존·카드·추측 공격 가중치를 적용한다.
3. 낮은 영혼 구간의 강행동 예고 데이터를 제공한다.
4. 보스 표시 모델에는 추론 방향과 신뢰도만 제공한다.
5. 영혼 0 전투 종료와 높은 등급 보상 뒤 `RunVictory`를 검증한다.

### 완료 게이트

- 세 구간 경계가 정확하고 상태 고착이 없다.
- 별도 승리 조건 없이 기존 전투 종료 규칙을 사용한다.
- 보스 선택·건너뛰기와 재시작 회귀가 유지된다.

### 추천 커밋 제목

`최종 보스가 영혼 구간에 따라 압박을 높이면서 기존 런 승리 규칙을 유지하게 구현`

## 10. EP-06 — 선택 전투 통합과 마감

### 목표

상대 선택 담당의 프로필 키를 실제 전투와 보상으로 연결하고 다섯 프로필의 반복 안정성을 검증한다.

### 작업

1. 선택된 `BattleProfileKey`를 전투 설정으로 변환한다.
2. 표시 미리보기와 실제 적 등급·영혼·보상이 일치하는지 확인한다.
3. 다섯 프로필의 생성·전투·종료·재시작을 반복한다.
4. CoreLoop, StageProgression, 카드 사용, 전투 보상과 전체 EditMode를 실행한다.
5. 실제 일반·엘리트·보스 흐름, 양쪽 씬과 Console을 확인한다.
6. 문서 4종, README, AI 활용과 이천서 역할 기록을 마감한다.

### 완료 게이트

- 알 수 없는·중복·오염된 프로필 입력이 원자적으로 거절된다.
- 선택한 적과 실제 전투·보상 결과가 일치한다.
- 반복에서 카드·덱·영혼·정책 상태가 다음 전투에 누출되지 않는다.
- 전체 자동·실제 검증이 통과한다.

### 추천 커밋 제목

`선택된 상대의 전투 성향과 보상 결과를 반복 검증해 적 프로필 1차 범위를 마감`

## 11. 의존 관계

```text
EP-00 문서 기준
  → EP-01 프로필·정책 계약 ──→ 상대 선택 담당 병렬 구현 가능
      → EP-02 공개 관측·공통 행동
          → EP-03 일반 적 3종
          → EP-04 엘리트
          → EP-05 보스
              → EP-06 상대 선택·진행 통합
```

EP-03~EP-05는 EP-02의 공정성·행동 실행 경계를 공통으로 사용한다. 상대 선택 담당은 EP-01 이후 프로필 미리보기와 키만 소비하며 CoreLoop 정책을 수정하지 않는다.

## 14. 커밋 원칙

- 단계별로 구현·테스트·문서 기록을 함께 커밋한다.
- 제목은 변경 파일보다 해결한 목적을 설명한다.
- Lore 본문에 제약, 거절한 대안, 검증과 미검증 항목을 기록한다.
- Unity `.meta`는 대응 C# 파일과 함께 관리한다.
- 다른 담당자의 아트·상대 선택 파일을 한 EP 커밋에 섞지 않는다.
- 사용자가 요청하기 전에는 스테이징하거나 커밋하지 않는다.
